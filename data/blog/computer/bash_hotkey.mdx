## 引言

Bash（Bourne Again Shell）是 Linux 和类 Unix 系统中广泛使用的命令行解释器，也是 macOS 和许多其他系统的默认 shell。掌握 Bash 的常用命令和快捷键不仅可以显著提升命令行操作的效率，更能让您体验到"命令行即生产力"的流畅工作流。与图形界面不同，Bash 通过组合简单的命令和快捷键来实现复杂的操作，这种设计哲学使得熟练的用户可以仅用键盘就能完成几乎所有系统管理任务，无需频繁使用鼠标。

Bash 的学习曲线虽然相对平缓，但一旦掌握了其核心命令和快捷键，您会发现工作效率的提升是显著的。许多开发者和系统管理员将 Bash 视为提高生产力的重要工具，甚至有人表示"一旦习惯了命令行，就很难再回到图形界面"。

本文档旨在提供一份全面而实用的 Bash 常用命令和快捷键参考指南，从基础的文件操作到高级的文本处理，从简单的光标移动到复杂的命令组合，帮助您系统性地学习和掌握 Bash 的强大功能。无论您是刚开始接触命令行的新手，还是希望深入理解 Bash 高级特性的资深用户，都能从本文中找到有价值的内容。

**学习建议**

- **循序渐进**：先掌握基础的文件操作和目录导航，再逐步学习高级功能
- **实践为主**：命令和快捷键的记忆需要通过实际使用来巩固，建议在日常工作中刻意练习
- **理解原理**：理解 Bash 的命令组合模式和管道重定向机制，可以事半功倍
- **建立习惯**：将 Bash 快捷键融入日常工作流，形成肌肉记忆

## 第一部分：Bash 基础命令 - 文件与目录操作

文件与目录操作是 Bash 中最基础也是最重要的功能。掌握这些命令是高效使用命令行的前提。

### 目录导航

| 命令    | 功能             | 说明                                     |
| ------- | ---------------- | ---------------------------------------- |
| `cd`    | 切换目录         | 更改当前工作目录                         |
| `cd ~`  | 切换到用户主目录 | 快速返回主目录                           |
| `cd -`  | 切换到上一个目录 | 返回到之前所在的目录                     |
| `cd ..` | 切换到上级目录   | 向上一级目录移动                         |
| `pwd`   | 显示当前目录     | 打印当前工作目录的完整路径               |
| `ls`    | 列出目录内容     | 显示当前目录的文件和子目录               |
| `ls -l` | 详细列表         | 显示文件的详细信息（权限、大小、时间等） |
| `ls -a` | 显示所有文件     | 包括隐藏文件（以点开头的文件）           |
| `ls -h` | 人类可读格式     | 文件大小以 KB、MB 等单位显示             |
| `ls -t` | 按时间排序       | 按修改时间排序显示                       |
| `ls -r` | 反向排序         | 反向排序显示                             |
| `ls -R` | 递归列出         | 递归显示子目录内容                       |

**目录导航技巧**：

- 使用 `cd` 不加参数等同于 `cd ~`，都会返回主目录
- `cd -` 是一个非常有用的命令，可以在两个目录间快速切换
- `ls` 命令支持多个选项组合，如 `ls -lah` 表示详细列出所有文件（包括隐藏文件）并以人类可读格式显示

### 文件操作

| 命令         | 功能                   | 说明                                   |
| ------------ | ---------------------- | -------------------------------------- |
| `touch`      | 创建空文件或更新时间戳 | 如果文件不存在则创建，存在则更新时间戳 |
| `cat`        | 显示文件内容           | 连接并显示文件内容                     |
| `cat -n`     | 显示行号               | 显示文件内容并标注行号                 |
| `less`       | 分页查看文件           | 分页显示文件内容，支持上下滚动         |
| `more`       | 分页查看文件           | 类似 less，但功能较少                  |
| `head`       | 显示文件开头           | 默认显示文件前 10 行                   |
| `head -n 20` | 显示前 n 行            | 显示文件前 20 行                       |
| `tail`       | 显示文件结尾           | 默认显示文件后 10 行                   |
| `tail -n 20` | 显示后 n 行            | 显示文件后 20 行                       |
| `tail -f`    | 实时跟踪文件           | 实时显示文件新增内容（常用于查看日志） |

**文件操作技巧**：

- `cat` 可以同时显示多个文件：`cat file1 file2`
- `less` 比 `more` 功能更强大，支持向前向后滚动，按 `q` 退出
- `tail -f` 非常适合监控日志文件，按 `Ctrl+C` 退出跟踪

### 文件管理

| 命令       | 功能           | 说明                             |
| ---------- | -------------- | -------------------------------- |
| `cp`       | 复制文件或目录 | 复制文件或目录到指定位置         |
| `cp -r`    | 递归复制目录   | 复制目录及其所有内容             |
| `cp -p`    | 保留属性       | 保留文件的权限、时间戳等属性     |
| `mv`       | 移动或重命名   | 移动文件或目录，也可用于重命名   |
| `rm`       | 删除文件       | 删除指定的文件                   |
| `rm -r`    | 递归删除目录   | 删除目录及其所有内容             |
| `rm -f`    | 强制删除       | 强制删除，不提示确认             |
| `rm -i`    | 交互式删除     | 删除前提示确认                   |
| `mkdir`    | 创建目录       | 创建新目录                       |
| `mkdir -p` | 创建多级目录   | 创建目录及其父目录（如果不存在） |
| `rmdir`    | 删除空目录     | 只能删除空目录                   |

**文件管理技巧**：

- `cp` 和 `mv` 都支持通配符，如 `cp *.txt backup/`
- `rm -rf` 组合非常危险，删除前务必确认路径正确
- `mkdir -p` 可以一次性创建多级目录，如 `mkdir -p project/src/main/java`

### 文件查找

| 命令                   | 功能         | 说明                                       |
| ---------------------- | ------------ | ------------------------------------------ |
| `find`                 | 查找文件     | 在目录树中查找文件                         |
| `find . -name "*.txt"` | 按名称查找   | 在当前目录查找所有 .txt 文件               |
| `find . -type f`       | 查找文件类型 | 查找普通文件（不包括目录）                 |
| `find . -type d`       | 查找目录     | 只查找目录                                 |
| `find . -size +100M`   | 按大小查找   | 查找大于 100MB 的文件                      |
| `find . -mtime -7`     | 按时间查找   | 查找最近 7 天内修改的文件                  |
| `locate`               | 快速查找文件 | 使用数据库快速查找文件（需要先更新数据库） |
| `which`                | 查找命令路径 | 显示命令的完整路径                         |
| `whereis`              | 查找命令位置 | 显示命令的二进制文件、源码和手册页位置     |

**文件查找技巧**：

- `find` 功能强大但速度较慢，`locate` 速度快但需要定期更新数据库（`updatedb`）
- `find` 可以结合 `-exec` 或 `-delete` 执行操作，如 `find . -name "*.tmp" -delete`
- `which` 只查找 PATH 中的命令，`whereis` 查找范围更广

### 权限管理

| 命令                    | 功能             | 说明                                           |
| ----------------------- | ---------------- | ---------------------------------------------- |
| `chmod`                 | 更改文件权限     | 修改文件或目录的权限                           |
| `chmod 755 file`        | 数字方式设置权限 | 755 表示所有者可读写执行，组和其他用户可读执行 |
| `chmod +x file`         | 添加执行权限     | 给文件添加执行权限                             |
| `chmod -w file`         | 移除写权限       | 移除文件的写权限                               |
| `chmod u+x file`        | 用户权限         | 给所有者添加执行权限                           |
| `chmod g+w file`        | 组权限           | 给组添加写权限                                 |
| `chmod o-r file`        | 其他用户权限     | 移除其他用户的读权限                           |
| `chown`                 | 更改文件所有者   | 修改文件或目录的所有者                         |
| `chown user:group file` | 更改所有者和组   | 同时更改所有者和组                             |
| `chgrp`                 | 更改文件组       | 修改文件或目录的组                             |

**权限说明**：

- 权限分为三种：读（r=4）、写（w=2）、执行（x=1）
- 权限分为三类用户：所有者（u）、组（g）、其他用户（o）
- 数字权限：755 = rwxr-xr-x，644 = rw-r--r--

## 第二部分：文本处理命令

文本处理是 Bash 的强项之一，通过组合各种文本处理命令可以实现强大的数据处理功能。

### 搜索命令

| 命令                  | 功能       | 说明                             |
| --------------------- | ---------- | -------------------------------- |
| `grep`                | 搜索文本   | 在文件中搜索匹配的文本模式       |
| `grep "pattern" file` | 基本搜索   | 在文件中搜索包含 pattern 的行    |
| `grep -i`             | 忽略大小写 | 搜索时不区分大小写               |
| `grep -r`             | 递归搜索   | 递归搜索目录中的所有文件         |
| `grep -n`             | 显示行号   | 显示匹配行的行号                 |
| `grep -v`             | 反向匹配   | 显示不匹配的行                   |
| `grep -E`             | 扩展正则   | 使用扩展正则表达式               |
| `egrep`               | 扩展 grep  | 等同于 `grep -E`                 |
| `fgrep`               | 固定字符串 | 搜索固定字符串，不使用正则表达式 |

**搜索技巧**：

- `grep` 支持正则表达式，如 `grep "^[0-9]" file` 查找以数字开头的行
- `grep -r` 可以结合管道使用，如 `grep -r "error" . | grep -v ".log"`
- `grep` 可以搜索多个文件：`grep "pattern" file1 file2 file3`

### 编辑命令

| 命令                      | 功能         | 说明                      |
| ------------------------- | ------------ | ------------------------- |
| `sed`                     | 流编辑器     | 对文本进行流式编辑        |
| `sed 's/old/new/g' file`  | 替换文本     | 将文件中的 old 替换为 new |
| `sed -i`                  | 原地编辑     | 直接修改文件（谨慎使用）  |
| `sed -n '5p'`             | 打印指定行   | 打印第 5 行               |
| `sed '5d'`                | 删除指定行   | 删除第 5 行               |
| `awk`                     | 文本处理工具 | 强大的文本分析和处理工具  |
| `awk '{print $1}'`        | 打印第一列   | 打印每行的第一个字段      |
| `awk -F: '{print $1}'`    | 指定分隔符   | 使用冒号作为字段分隔符    |
| `awk '/pattern/ {print}'` | 模式匹配     | 打印匹配模式的行          |

**编辑技巧**：

- `sed` 非常适合批量替换，如 `sed 's/old/new/g' file > newfile`
- `awk` 功能强大，可以处理复杂的文本分析任务
- `sed` 和 `awk` 都支持正则表达式，可以组合使用

### 排序与去重

| 命令        | 功能       | 说明               |
| ----------- | ---------- | ------------------ |
| `sort`      | 排序       | 对文本行进行排序   |
| `sort -n`   | 数字排序   | 按数字大小排序     |
| `sort -r`   | 反向排序   | 降序排序           |
| `sort -u`   | 去重排序   | 排序并去除重复行   |
| `sort -k 2` | 按字段排序 | 按第二个字段排序   |
| `uniq`      | 去重       | 去除相邻的重复行   |
| `uniq -c`   | 计数       | 显示每行重复的次数 |
| `uniq -d`   | 显示重复行 | 只显示重复的行     |

**排序技巧**：

- `sort` 通常需要先排序再使用 `uniq`，因为 `uniq` 只去除相邻重复行
- `sort -u` 等同于 `sort | uniq`，但更高效
- `sort` 可以按多个字段排序：`sort -k1,1 -k2,2n`

### 文本提取

| 命令          | 功能       | 说明                           |
| ------------- | ---------- | ------------------------------ |
| `cut`         | 提取字段   | 从每行中提取指定字段           |
| `cut -d: -f1` | 指定分隔符 | 使用冒号分隔，提取第一个字段   |
| `cut -c1-10`  | 按字符提取 | 提取第 1 到 10 个字符          |
| `wc`          | 统计       | 统计文件的行数、单词数、字符数 |
| `wc -l`       | 统计行数   | 只统计行数                     |
| `wc -w`       | 统计单词数 | 只统计单词数                   |
| `wc -c`       | 统计字符数 | 只统计字符数                   |

**提取技巧**：

- `cut` 适合处理结构化文本，如 CSV 文件
- `wc -l` 常用于统计文件行数或命令输出行数
- `cut` 和 `awk` 功能有重叠，`awk` 更灵活但 `cut` 更简单

### 比较命令

| 命令      | 功能       | 说明               |
| --------- | ---------- | ------------------ |
| `diff`    | 比较文件   | 比较两个文件的差异 |
| `diff -u` | 统一格式   | 以统一格式显示差异 |
| `diff -r` | 递归比较   | 递归比较目录       |
| `cmp`     | 逐字节比较 | 逐字节比较两个文件 |

**比较技巧**：

- `diff` 输出可以用 `patch` 命令应用到文件
- `diff -u` 格式更易读，常用于代码比较
- `diff` 可以比较目录，显示文件差异列表

## 第三部分：系统管理命令

系统管理命令帮助您监控和管理系统资源、进程和网络连接。

### 进程管理

| 命令      | 功能         | 说明                                 |
| --------- | ------------ | ------------------------------------ |
| `ps`      | 显示进程     | 显示当前运行的进程                   |
| `ps aux`  | 详细进程信息 | 显示所有用户的详细进程信息           |
| `ps -ef`  | 完整格式     | 以完整格式显示进程信息               |
| `top`     | 实时进程监控 | 实时显示系统进程和资源使用情况       |
| `htop`    | 增强版 top   | top 的增强版本，界面更友好           |
| `kill`    | 终止进程     | 向进程发送信号终止进程               |
| `kill -9` | 强制终止     | 强制终止进程（SIGKILL）              |
| `killall` | 按名称终止   | 按进程名称终止所有匹配的进程         |
| `jobs`    | 显示作业     | 显示当前 shell 的作业列表            |
| `fg`      | 前台运行     | 将后台作业切换到前台                 |
| `bg`      | 后台运行     | 将暂停的作业放到后台运行             |
| `nohup`   | 后台运行     | 在后台运行命令，退出终端后仍继续运行 |

**进程管理技巧**：

- `ps aux | grep process_name` 可以查找特定进程
- `kill -9` 是最后手段，应该先尝试 `kill`（发送 SIGTERM）
- `nohup command &` 可以让命令在后台运行，输出重定向到 nohup.out

### 系统信息

| 命令       | 功能         | 说明                           |
| ---------- | ------------ | ------------------------------ |
| `uname`    | 系统信息     | 显示系统信息                   |
| `uname -a` | 全部信息     | 显示所有系统信息               |
| `df`       | 磁盘空间     | 显示文件系统的磁盘空间使用情况 |

| `du`       | 目录大小     | 显示目录的磁盘使用情况         |
| `du -h`    | 人类可读格式 | 以人类可读格式显示             |
| `du -sh`   | 总大小       | 只显示目录总大小               |
| `free`     | 内存信息     | 显示内存使用情况               |
| `free -h`  | 人类可读格式 | 以人类可读格式显示内存信息     |
| `uptime`   | 运行时间     | 显示系统运行时间和负载         |
| `whoami`   | 当前用户     | 显示当前用户名                 |
| `id`       | 用户 ID      | 显示用户和组的 ID              |

**系统信息技巧**：

- `df -h` 和 `du -h` 的输出更易读，推荐使用
- `du -sh *` 可以快速查看当前目录下各文件/目录的大小
- `uptime` 显示的负载平均值反映了系统负载情况

### 网络操作

| 命令                         | 功能       | 说明                           |
| ---------------------------- | ---------- | ------------------------------ |
| `curl`                       | 传输数据   | 从服务器传输数据，支持多种协议 |
| `curl -O`                    | 下载文件   | 下载文件并保存为原文件名       |
| `curl -L`                    | 跟随重定向 | 自动跟随 HTTP 重定向           |
| `wget`                       | 下载文件   | 从网络下载文件                 |
| `wget -r`                    | 递归下载   | 递归下载网站内容               |
| `ssh`                        | 远程登录   | 通过 SSH 协议连接远程主机      |
| `scp`                        | 安全复制   | 通过 SSH 协议复制文件          |
| `scp file user@host:/path`   | 上传文件   | 将文件复制到远程主机           |
| `scp user@host:/path/file .` | 下载文件   | 从远程主机复制文件到本地       |
| `rsync`                      | 同步文件   | 高效的文件同步工具             |

**网络操作技巧**：

- `curl` 和 `wget` 功能相似，`curl` 更灵活，`wget` 更适合批量下载
- `scp` 和 `rsync` 都需要 SSH 访问权限
- `rsync` 比 `scp` 更高效，特别是同步大量文件时

### 压缩归档

| `tar -czf` | 创建压缩归档  | 创建 gzip 压缩的 tar 文件 |

| `gunzip`   | 解压文件      | 解压 gzip 压缩的文件      |
| `zip`      | 创建 zip 文件 | 创建 zip 压缩文件         |

**压缩技巧**：

- `tar -czf archive.tar.gz dir/` 创建压缩归档
- `tar -xzf archive.tar.gz` 解压归档
- `zip` 格式在 Windows 和 Linux 之间兼容性好

## 第四部分：命令行编辑快捷键 - 光标移动

Bash 提供了丰富的快捷键来高效编辑命令行，这些快捷键基于 Readline 库实现。掌握这些快捷键可以大幅提升命令行编辑效率。

### 行内移动

| 快捷键     | 功能             | 说明                                 |
| ---------- | ---------------- | ------------------------------------ |
| `Ctrl + A` | 移动到行首       | 将光标移动到命令行的开头             |
| `Ctrl + E` | 移动到行尾       | 将光标移动到命令行的末尾             |
| `Ctrl + B` | 向左移动一个字符 | 光标向后移动一个字符（等同于左箭头） |
| `Ctrl + F` | 向右移动一个字符 | 光标向前移动一个字符（等同于右箭头） |

**行内移动技巧**：

- `Ctrl + A` 和 `Ctrl + E` 是最常用的快捷键，可以快速跳转到行首或行尾
- 这些快捷键在所有使用 Readline 的程序中都有效（如 Python REPL、MySQL 客户端等）
- 记忆方法：A 是 Alphabet 的第一个字母（行首），E 是 End（行尾）

### 单词移动

| 快捷键     | 功能             | 说明                             |
| ---------- | ---------------- | -------------------------------- |
| `Alt + B`  | 向左移动一个单词 | 光标向后移动一个单词             |
| `Alt + F`  | 向右移动一个单词 | 光标向前移动一个单词             |
| `Ctrl + ←` | 向左移动一个单词 | 在某些终端中可用（等同于 Alt+B） |
| `Ctrl + →` | 向右移动一个单词 | 在某些终端中可用（等同于 Alt+F） |

**单词移动技巧**：

- `Alt + B` 和 `Alt + F` 比逐字符移动快得多，特别适合编辑长命令
- 单词的边界由空格和标点符号定义
- 在某些终端或配置中，`Alt` 键可能被映射为 `Esc`，需要按 `Esc` 然后按 `B` 或 `F`

## 第五部分：命令行编辑快捷键 - 文本编辑

文本编辑快捷键让您能够快速删除、修改和调整命令行内容。

### 删除操作

**删除操作技巧**：

- `Ctrl + U` 和 `Ctrl + K` 是最常用的删除快捷键，可以快速清空部分命令
- `Ctrl + W` 非常适合删除路径中的目录名
- `Ctrl + D` 在空行时用于退出终端或结束输入

### 粘贴操作

| 快捷键     | 功能           | 说明                                             |

**粘贴技巧**：

- `Ctrl + Y` 只能粘贴最近一次删除的内容
- 删除的内容存储在 kill ring 中，可以多次粘贴
- 这个功能在修改长命令时非常有用

### 字符和单词交换

| `Ctrl + T` | 交换字符 | 交换光标所在字符和前一个字符的位置 |
| `Alt + T`  | 交换单词 | 交换光标所在单词和前一个单词的位置 |

**交换技巧**：

- `Ctrl + T` 适合修正打字错误，如将 "teh" 快速改为 "the"
- `Alt + T` 可以快速调整命令中参数的顺序

### 大小写转换

| 快捷键    | 功能       | 说明                             |
| --------- | ---------- | -------------------------------- |
| `Alt + C` | 首字母大写 | 将光标所在单词的首字母转换为大写 |

| `Alt + L` | 全部小写   | 将光标所在单词转换为小写         |

**大小写转换技巧**：

- 这些快捷键在输入文件名或变量名时非常有用
- `Alt + C` 常用于修正命令或文件名的大小写
- 转换后光标会移动到单词末尾

## 第六部分：历史命令操作

Bash 维护着命令历史记录，通过快捷键可以快速访问和重用之前的命令。

### 历史导航

| 快捷键     | 功能       | 说明                               |
| ---------- | ---------- | ---------------------------------- |
| `Ctrl + P` | 上一条命令 | 显示上一条历史命令（等同于上箭头） |
| `Ctrl + N` | 下一条命令 | 显示下一条历史命令（等同于下箭头） |
| `↑`        | 上一条命令 | 向上浏览历史命令                   |
| `↓`        | 下一条命令 | 向下浏览历史命令                   |

**历史导航技巧**：

- 历史命令存储在 `~/.bash_history` 文件中
- 可以使用 `history` 命令查看所有历史命令
- `HISTSIZE` 环境变量控制历史记录的数量

### 历史搜索

| 快捷键     | 功能     | 说明                                     |
| ---------- | -------- | ---------------------------------------- |
| `Ctrl + R` | 反向搜索 | 向后搜索命令历史，输入关键词进行搜索     |
| `Ctrl + G` | 退出搜索 | 退出历史搜索模式                         |
| `Ctrl + S` | 向前搜索 | 向前搜索命令历史（需要先禁用终端流控制） |

**历史搜索技巧**：

- `Ctrl + R` 是最强大的历史命令查找方式，输入关键词即可快速找到命令
- 在搜索模式下，继续按 `Ctrl + R` 可以查找更早的匹配项
- 找到命令后按 `Enter` 执行，或按 `Ctrl + G` 取消

### 历史展开

| 语法       | 功能                     | 说明                         |
| ---------- | ------------------------ | ---------------------------- |
| `!!`       | 上一条命令               | 重复执行上一条命令           |
| `!n`       | 第 n 条命令              | 执行历史记录中的第 n 条命令  |
| `!-n`      | 倒数第 n 条              | 执行倒数第 n 条命令          |
| `!string`  | 最近匹配命令             | 执行最近以 string 开头的命令 |
| `!?string` | 包含字符串               | 执行最近包含 string 的命令   |
| `!$`       | 上一条命令的最后一个参数 | 引用上一条命令的最后一个参数 |
| `!^`       | 上一条命令的第一个参数   | 引用上一条命令的第一个参数   |
| `!:n`      | 上一条命令的第 n 个参数  | 引用上一条命令的第 n 个参数  |

**历史展开示例**：

```bash
# 执行上一条命令
sudo !!

# 执行历史中的第 100 条命令
!100

# 执行最近以 git 开头的命令
!git

# 使用上一条命令的最后一个参数
ls /very/long/path
cd !$

# 使用上一条命令的第一个参数
grep pattern file.txt
vim !^
```

**历史展开技巧**：

- `!!` 常用于在命令前添加 `sudo`
- `!$` 非常适合在操作文件后切换到该文件所在目录
- 历史展开支持 Tab 补全，输入 `!` 后按 Tab 可以查看匹配的命令

## 第七部分：进程控制快捷键

进程控制快捷键用于管理正在运行的命令和进程。

### 进程终止

| 快捷键     | 功能     | 说明                                     |
| ---------- | -------- | ---------------------------------------- |
| `Ctrl + C` | 终止进程 | 向当前前台进程发送 SIGINT 信号，终止进程 |

**终止技巧**：

- `Ctrl + C` 是最常用的中断命令方式
- 某些程序可能会捕获 SIGINT 信号并执行清理操作，而不是立即退出
- 如果 `Ctrl + C` 无效，可以使用 `Ctrl + Z` 暂停进程，然后用 `kill` 命令终止

### 进程暂停

| 快捷键     | 功能     | 说明                                                |
| ---------- | -------- | --------------------------------------------------- |
| `Ctrl + Z` | 暂停进程 | 向当前前台进程发送 SIGTSTP 信号，暂停进程并放入后台 |

**暂停技巧**：

- 暂停的进程可以使用 `fg` 命令恢复到前台，或使用 `bg` 命令在后台继续运行
- `jobs` 命令可以查看所有暂停和后台运行的作业
- 暂停的进程仍然占用内存，只是不执行

### 退出终端

| 快捷键     | 功能     | 说明                                                 |
| ---------- | -------- | ---------------------------------------------------- |
| `Ctrl + D` | 退出终端 | 发送 EOF（文件结束）信号，在空行时退出终端或结束输入 |

**退出技巧**：

- `Ctrl + D` 在空行时才会退出终端，如果当前行有内容，则只删除一个字符
- 某些程序使用 `Ctrl + D` 来结束输入，如 `cat` 命令
- 这是退出终端的安全方式，比直接关闭窗口更优雅

### 清屏

| 快捷键     | 功能 | 说明                                  |
| ---------- | ---- | ------------------------------------- |
| `Ctrl + L` | 清屏 | 清除终端屏幕，等同于执行 `clear` 命令 |

**清屏技巧**：

- `Ctrl + L` 比输入 `clear` 命令更快
- 清屏后历史命令仍然可以通过上下箭头访问
- 这个快捷键在所有使用 Readline 的程序中都有效

### 输出控制

| 快捷键     | 功能     | 说明                   |
| ---------- | -------- | ---------------------- |
| `Ctrl + S` | 暂停输出 | 暂停终端输出（流控制） |
| `Ctrl + Q` | 恢复输出 | 恢复终端输出           |

**输出控制技巧**：

- `Ctrl + S` 和 `Ctrl + Q` 是流控制功能，用于控制数据流
- 如果不小心按了 `Ctrl + S`，终端会"冻结"，按 `Ctrl + Q` 可以恢复
- 在现代终端中，这个功能可能默认禁用

## 第八部分：高级技巧与组合使用

掌握命令组合和高级技巧可以让您更高效地使用 Bash，实现复杂的操作。

### 管道与重定向

管道和重定向是 Bash 最强大的特性之一，允许将命令的输出传递给其他命令或文件。

| 操作符 | 功能          | 说明                             |
| ------ | ------------- | -------------------------------- | -------------------------------------- |
| `      | `             | 管道                             | 将前一个命令的输出作为后一个命令的输入 |
| `>`    | 输出重定向    | 将命令输出重定向到文件（覆盖）   |
| `>>`   | 追加重定向    | 将命令输出追加到文件末尾         |
| `<`    | 输入重定向    | 从文件读取输入                   |
| `<<`   | Here Document | 从标准输入读取直到遇到指定字符串 |

**管道与重定向示例**：

```bash
# 管道：查找包含 error 的行并统计数量
grep "error" log.txt | wc -l

# 输出重定向：将命令输出保存到文件
ls -l > file_list.txt

# 追加重定向：将输出追加到文件
echo "new line" >> file.txt

# 输入重定向：从文件读取输入
sort < unsorted.txt

# Here Document：创建多行文本
cat << EOF
Line 1
Line 2
Line 3
EOF
```

**管道技巧**：

- 管道可以串联多个命令：`command1 | command2 | command3`
- `>` 会覆盖文件，`>>` 会追加内容
- `2>&1` 可以将错误输出重定向到标准输出

### 命令组合

Bash 提供了多种方式来组合命令，实现条件执行和顺序执行。

| 操作符 | 功能     | 说明                                     |
| ------ | -------- | ---------------------------------------- | ------ | -------------------------------- |
| `&&`   | 逻辑与   | 前一个命令成功后才执行后一个命令         |
| `      |          | `                                        | 逻辑或 | 前一个命令失败后才执行后一个命令 |
| `;`    | 顺序执行 | 无论前一个命令是否成功，都执行后一个命令 |
| `&`    | 后台执行 | 在后台执行命令                           |

**命令组合示例**：

```bash
# 逻辑与：只有前一个命令成功才执行下一个
mkdir newdir && cd newdir

# 逻辑或：前一个命令失败才执行下一个
command || echo "Command failed"

# 顺序执行：依次执行命令
cd /tmp; ls -l; pwd

# 后台执行：在后台运行命令
long_running_command &
```

**组合技巧**：

- `&&` 常用于确保前一步成功后再执行下一步
- `||` 常用于错误处理，提供备用方案
- `&` 可以让命令在后台运行，释放终端

### 别名与函数

别名和函数可以简化常用命令，提高工作效率。

**别名示例**：

```bash
# 创建别名
alias ll='ls -lah'
alias grep='grep --color=auto'
alias ..='cd ..'
alias ...='cd ../..'

# 查看所有别名
alias

# 删除别名
unalias ll

# 永久保存别名（添加到 ~/.bashrc）
echo "alias ll='ls -lah'" >> ~/.bashrc
```

**函数示例**：

```bash
# 定义函数
function mkcd() {
    mkdir -p "$1" && cd "$1"
}

# 使用函数
mkcd newproject

# 永久保存函数（添加到 ~/.bashrc）
cat >> ~/.bashrc << 'EOF'
function mkcd() {
    mkdir -p "$1" && cd "$1"
}
EOF
```

**别名与函数技巧**：

- 别名适合简单的命令替换，函数适合复杂的逻辑
- 函数可以接受参数，别名不能
- 永久别名和函数应该添加到 `~/.bashrc` 文件中

### 环境变量

环境变量用于配置 Bash 的行为和存储系统信息。

| 命令     | 功能         | 说明                                   |
| -------- | ------------ | -------------------------------------- |
| `export` | 导出变量     | 将变量导出为环境变量                   |
| `env`    | 显示环境变量 | 显示所有环境变量                       |
| `set`    | 显示所有变量 | 显示所有变量（包括环境变量和局部变量） |
| `unset`  | 删除变量     | 删除变量                               |

**环境变量示例**：

```bash
# 设置环境变量
export PATH="$PATH:/usr/local/bin"
export EDITOR=vim

# 查看环境变量
echo $PATH
env | grep PATH

# 临时设置环境变量（仅对当前命令有效）
EDITOR=nano command

# 删除环境变量
unset EDITOR
```

**常用环境变量**：

- `PATH`：命令搜索路径
- `HOME`：用户主目录
- `USER`：当前用户名
- `SHELL`：当前 shell
- `EDITOR`：默认编辑器
- `HISTSIZE`：历史记录数量

### 实用组合示例

以下是一些实用的命令组合示例，展示了如何将多个命令和快捷键组合使用。

**示例 1：查找并处理文件**

```bash
# 查找所有 .log 文件并删除
find . -name "*.log" -type f -delete

# 查找大文件并显示详细信息
find . -type f -size +100M -exec ls -lh {} \;
```

**示例 2：文本处理管道**

```bash
# 统计日志文件中错误的数量
grep -i error log.txt | wc -l

# 提取 IP 地址并排序去重
grep -oE '[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}' access.log | sort -u
```

**示例 3：进程管理组合**

```bash
# 查找并终止进程
ps aux | grep process_name | grep -v grep | awk '{print $2}' | xargs kill

# 后台运行命令并记录输出
nohup command > output.log 2>&1 &
```

**示例 4：快速目录操作**

```bash
# 创建目录并立即进入
mkdir -p project/src/main/java && cd project/src/main/java

# 返回上一个目录
cd -
```

**示例 5：历史命令重用**

```bash
# 执行上一条命令并添加 sudo
sudo !!

# 使用上一条命令的参数
ls /very/long/path/to/file
vim !$
```
