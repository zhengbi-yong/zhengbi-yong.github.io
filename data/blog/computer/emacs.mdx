## 引言

Emacs 是一款功能强大的文本编辑器，以其高度可定制性和扩展性而闻名于世。自 1976 年由 Richard Stallman 创建以来，Emacs 已经发展成为不仅仅是文本编辑器，更是一个完整的开发环境和操作系统。Emacs 的设计哲学强调可扩展性和用户定制，通过 Emacs Lisp（Elisp）编程语言，用户可以深度定制编辑器的每一个细节，打造完全符合个人工作流的编辑环境。

与传统的文本编辑器不同，Emacs 提供了丰富的内置功能，包括文件管理、邮件客户端、新闻阅读器、调试器、版本控制系统集成等。许多开发者将 Emacs 视为"编辑器的终极形态"，甚至有人表示"Emacs 不仅仅是一个编辑器，而是一种生活方式"。

本文档旨在提供一份全面而实用的 Emacs 使用指南，从基础的文件操作到高级的定制配置，从原生的快捷键系统到 Vim 键位支持，帮助您系统性地学习和掌握 Emacs 的强大功能。无论您是刚开始接触 Emacs 的新手，还是希望深入理解 Emacs 高级特性的资深用户，都能从本文中找到有价值的内容。

**学习建议**

- **循序渐进**：先掌握基础的快捷键和文件操作，再逐步学习高级功能
- **实践为主**：快捷键的记忆需要通过实际使用来巩固，建议在日常编辑中刻意练习
- **理解概念**：理解 Emacs 的缓冲区、窗口、模式等核心概念，可以事半功倍
- **建立习惯**：将 Emacs 快捷键融入日常工作流，形成肌肉记忆

## 第一部分：Emacs 基础概念

理解 Emacs 的核心概念是高效使用 Emacs 的基础。Emacs 的设计理念与传统编辑器有很大不同，掌握这些概念将帮助您更好地理解和使用 Emacs。

### Emacs 的设计哲学

**可扩展性**

Emacs 的核心设计理念是"可扩展性"。几乎所有功能都可以通过 Emacs Lisp 进行定制和扩展。用户不仅可以修改现有功能，还可以创建全新的功能。这种设计使得 Emacs 能够适应各种不同的使用场景，从简单的文本编辑到复杂的软件开发环境。

**键盘优先**

Emacs 强调键盘操作，几乎所有功能都可以通过键盘快捷键访问。虽然 Emacs 也支持鼠标操作，但熟练的用户会发现键盘操作更加高效。Emacs 的快捷键系统使用前缀键（如 `C-x`、`C-c`、`M-x`）来组织命令，使得大量命令可以通过组合键快速访问。

**统一接口**

Emacs 提供了统一的用户接口，无论是编辑文本、管理文件、浏览网页还是阅读邮件，都使用相同的快捷键和操作方式。这种一致性使得学习 Emacs 的投入能够获得长期回报。

### 核心概念

**缓冲区（Buffer）**

缓冲区是 Emacs 中文本的容器。每个打开的文件、每个正在编辑的文本、甚至帮助文档，都显示在缓冲区中。缓冲区是 Emacs 的核心抽象，理解缓冲区是理解 Emacs 的关键。

**窗口（Window）**

窗口是显示缓冲区的区域。一个 Emacs 框架（Frame）可以包含多个窗口，每个窗口显示一个缓冲区。窗口可以水平或垂直分割，允许同时查看多个缓冲区。

**框架（Frame）**

框架是 Emacs 的顶级窗口，类似于其他应用程序的窗口。一个 Emacs 进程可以创建多个框架，每个框架可以包含多个窗口。

**模式（Mode）**

模式是 Emacs 中用于定制编辑器行为的功能集合。Emacs 有两种类型的模式：

- **主模式（Major Mode）**：每个缓冲区只能有一个主模式，用于定义该缓冲区的主要编辑行为（如编程语言模式、文本模式）
- **次模式（Minor Mode）**：每个缓冲区可以有多个次模式，用于提供额外的功能（如行号显示、自动补全）

### 快捷键表示法

Emacs 使用特殊的表示法来描述快捷键：

| 表示法    | 含义                     | 说明                                       |
| --------- | ------------------------ | ------------------------------------------ |
| `C-x`     | Ctrl + x                 | 按住 Ctrl 键，然后按 x 键                  |
| `M-x`     | Meta + x                 | 按住 Meta 键（通常是 Alt 键），然后按 x 键 |
| `C-x C-f` | Ctrl + x，然后 Ctrl + f  | 先按 `C-x`，然后按 `C-f`                   |
| `C-x 4 f` | Ctrl + x，然后 4，然后 f | 先按 `C-x`，然后按数字 4，然后按 f         |

**记忆技巧**：

- `C-` 表示 Ctrl 键
- `M-` 表示 Meta 键（通常是 Alt 键，在某些系统中是 Esc 键）
- `S-` 表示 Shift 键
- 多个键的组合按顺序按下

## 第二部分：基础文件操作

文件操作是使用任何编辑器的基本技能。Emacs 提供了丰富的文件操作命令，掌握这些命令是高效使用 Emacs 的前提。

### 打开和保存文件

| 快捷键    | 功能           | 说明                        |
| --------- | -------------- | --------------------------- |
| `C-x C-f` | 打开文件       | 查找并打开文件（find-file） |
| `C-x C-s` | 保存文件       | 保存当前缓冲区到文件        |
| `C-x C-w` | 另存为         | 将当前缓冲区保存为新文件    |
| `C-x s`   | 保存所有缓冲区 | 保存所有已修改的缓冲区      |

**文件操作技巧**：

- `C-x C-f` 不仅可以打开文件，还可以创建新文件（输入新文件名即可）
- 在 `C-x C-f` 提示符下，可以使用 Tab 键进行文件名补全
- 使用 `C-x C-v` 可以重新加载文件，放弃未保存的修改

### 关闭和退出

| 快捷键    | 功能       | 说明                                 |
| --------- | ---------- | ------------------------------------ |
| `C-x C-c` | 退出 Emacs | 退出 Emacs，如果有未保存的修改会提示 |
| `C-x k`   | 关闭缓冲区 | 关闭当前缓冲区（kill-buffer）        |

**退出技巧**：

- `C-x C-c` 会提示保存未保存的修改
- 使用 `C-x C-c` 退出比直接关闭窗口更安全，可以确保所有工作都已保存

### 文件浏览

| 快捷键    | 功能       | 说明                                 |
| --------- | ---------- | ------------------------------------ |
| `C-x C-d` | 列出目录   | 显示当前目录的内容                   |
| `C-x d`   | Dired 模式 | 进入目录编辑模式，可以管理文件和目录 |

**Dired 模式**是 Emacs 的文件管理器，提供了丰富的文件操作功能：

- `d`：标记删除
- `u`：取消标记
- `x`：执行标记的操作
- `C`：复制文件
- `R`：重命名或移动文件
- `+`：创建目录

## 第三部分：光标移动

高效的光标移动是文本编辑的基础。Emacs 提供了多种光标移动方式，从字符级到文件级，满足各种编辑需求。

### 字符级移动

| 快捷键 | 功能             | 说明                          |
| ------ | ---------------- | ----------------------------- |
| `C-f`  | 向前移动一个字符 | Forward（等同于右箭头）       |
| `C-b`  | 向后移动一个字符 | Backward（等同于左箭头）      |
| `C-n`  | 向下移动一行     | Next line（等同于下箭头）     |
| `C-p`  | 向上移动一行     | Previous line（等同于上箭头） |

**记忆技巧**：

- `f` = Forward（向前）
- `b` = Backward（向后）
- `n` = Next（下一行）
- `p` = Previous（上一行）

### 单词级移动

| 快捷键 | 功能             | 说明          |
| ------ | ---------------- | ------------- |
| `M-f`  | 向前移动一个单词 | Forward word  |
| `M-b`  | 向后移动一个单词 | Backward word |

**单词移动技巧**：

- `M-f` 和 `M-b` 比逐字符移动快得多，特别适合编辑代码
- 单词的边界由空格和标点符号定义

### 行内移动

| 快捷键 | 功能           | 说明                  |
| ------ | -------------- | --------------------- |
| `C-a`  | 移动到行首     | Beginning of line     |
| `C-e`  | 移动到行尾     | End of line           |
| `M-a`  | 移动到句子开头 | Beginning of sentence |
| `M-e`  | 移动到句子结尾 | End of sentence       |

**行内移动技巧**：

- `C-a` 和 `C-e` 是最常用的行内移动快捷键
- `M-a` 和 `M-e` 适合编辑自然语言文本

### 文件级跳转

| 快捷键    | 功能           | 说明                   |
| --------- | -------------- | ---------------------- |
| `M-<`     | 跳转到文件开头 | Beginning of buffer    |
| `M->`     | 跳转到文件结尾 | End of buffer          |
| `C-l`     | 重绘屏幕       | 将当前行移动到屏幕中央 |
| `M-g M-g` | 跳转到指定行   | 跳转到指定的行号       |

**文件跳转技巧**：

- `C-l` 可以多次按，每次按会将当前行移动到不同的位置（中央、顶部、底部）
- `M-g M-g` 后输入行号可以快速跳转到指定行

## 第四部分：文本编辑

文本编辑是编辑器的核心功能。Emacs 提供了丰富的文本编辑命令，掌握这些命令可以大幅提升编辑效率。

### 删除操作

| 快捷键  | 功能           | 说明                            |
| ------- | -------------- | ------------------------------- |
| `C-d`   | 删除后一个字符 | Delete character                |
| `Del`   | 删除前一个字符 | Delete backward（等同于退格键） |
| `M-d`   | 删除到单词结尾 | Kill word                       |
| `M-Del` | 删除前一个单词 | Backward kill word              |
| `C-k`   | 删除到行尾     | Kill line                       |
| `C-w`   | 删除选中区域   | Kill region（需要先标记区域）   |

**删除操作说明**：

- Emacs 中的"kill"操作会将删除的内容保存到 kill ring（删除环）中，可以使用 `C-y` 粘贴
- `C-k` 如果在一行的中间使用，只删除到行尾；如果在行首使用，删除整行

### 复制和粘贴

| 快捷键 | 功能             | 说明                                          |
| ------ | ---------------- | --------------------------------------------- |
| `M-w`  | 复制选中区域     | Copy region                                   |
| `C-w`  | 剪切选中区域     | Kill region（剪切并删除）                     |
| `C-y`  | 粘贴             | Yank（粘贴最近删除或复制的内容）              |
| `M-y`  | 粘贴历史中的内容 | 在粘贴后使用，可以循环浏览 kill ring 中的内容 |

**复制粘贴技巧**：

- Emacs 的 kill ring 可以保存多个删除/复制的内容
- 使用 `M-y` 可以在粘贴后循环浏览 kill ring 中的历史内容
- `C-y` 粘贴最近的内容，`M-y` 可以替换为更早的内容

### 撤销操作

| 快捷键           | 功能 | 说明              |
| ---------------- | ---- | ----------------- |
| `C-x u` 或 `C-_` | 撤销 | Undo 最后一次操作 |
| `C-/`            | 撤销 | Undo 的替代快捷键 |

**撤销技巧**：

- Emacs 支持多次撤销，可以撤销到历史编辑状态
- `C-x u` 和 `C-_` 是同一个命令的不同表示法

### 标记和选择

| 快捷键    | 功能         | 说明                             |
| --------- | ------------ | -------------------------------- |
| `C-SPC`   | 设置标记     | 设置标记点（mark），开始选择区域 |
| `C-x C-x` | 交换点和标记 | 交换光标位置和标记位置           |
| `C-w`     | 删除选中区域 | 删除标记点到光标之间的内容       |
| `M-w`     | 复制选中区域 | 复制标记点到光标之间的内容       |

**标记技巧**：

- `C-SPC` 设置标记后，移动光标就会选择区域
- `C-x C-x` 可以快速跳转到标记位置，再次按可以跳回

## 第五部分：搜索与替换

搜索和替换是文本编辑中的常用操作。Emacs 提供了强大的搜索和替换功能，支持正则表达式，可以高效地查找和修改文本。

### 搜索操作

| 快捷键    | 功能         | 说明                        |
| --------- | ------------ | --------------------------- |
| `C-s`     | 向前搜索     | Incremental search forward  |
| `C-r`     | 向后搜索     | Incremental search backward |
| `C-s C-s` | 重复向前搜索 | 重复上一次搜索              |
| `C-r C-r` | 重复向后搜索 | 重复上一次搜索              |

**搜索技巧**：

- Emacs 的增量搜索（incremental search）会在您输入时实时显示匹配结果
- 在搜索模式下，按 `C-s` 或 `C-r` 可以查找下一个或上一个匹配
- 按 `Enter` 或 `Esc` 退出搜索模式

### 替换操作

| 快捷键               | 功能           | 说明                   |
| -------------------- | -------------- | ---------------------- |
| `M-%`                | 查询替换       | Query replace          |
| `M-x replace-string` | 无条件替换     | 替换所有匹配的文本     |
| `M-x replace-regexp` | 正则表达式替换 | 使用正则表达式进行替换 |

**替换技巧**：

- `M-%` 会逐个询问是否替换，可以按 `y` 替换，`n` 跳过，`!` 替换所有
- `replace-string` 会无条件替换所有匹配，使用前请确认

### 正则表达式搜索

| 快捷键  | 功能               | 说明                   |
| ------- | ------------------ | ---------------------- |
| `C-M-s` | 正则表达式向前搜索 | Regexp search forward  |
| `C-M-r` | 正则表达式向后搜索 | Regexp search backward |

**正则表达式基础**：

- `.`：匹配任意字符
- `*`：匹配前一个字符 0 次或多次
- `+`：匹配前一个字符 1 次或多次
- `?`：匹配前一个字符 0 次或 1 次
- `^`：匹配行首
- `$`：匹配行尾

## 第六部分：缓冲区管理

缓冲区是 Emacs 的核心概念。每个打开的文件、每个正在编辑的文本都显示在缓冲区中。掌握缓冲区管理可以高效地在多个文件间切换和编辑。

### 缓冲区切换

| 快捷键    | 功能           | 说明             |
| --------- | -------------- | ---------------- |
| `C-x b`   | 切换缓冲区     | Switch to buffer |
| `C-x C-b` | 列出所有缓冲区 | List buffers     |
| `C-x k`   | 关闭缓冲区     | Kill buffer      |

**缓冲区切换技巧**：

- `C-x b` 后输入缓冲区名称，可以使用 Tab 补全
- 直接按 Enter 可以切换到上一个缓冲区
- `C-x C-b` 会打开缓冲区列表，可以在这个列表中管理缓冲区

### 缓冲区列表操作

在缓冲区列表（`C-x C-b`）中：

| 快捷键 | 功能       | 说明             |
| ------ | ---------- | ---------------- |
| `d`    | 标记删除   | 标记缓冲区为删除 |
| `s`    | 保存缓冲区 | 保存标记的缓冲区 |
| `u`    | 取消标记   | 取消删除标记     |
| `x`    | 执行操作   | 执行标记的操作   |
| `q`    | 退出列表   | 退出缓冲区列表   |

## 第七部分：窗口管理

窗口是显示缓冲区的区域。Emacs 支持在一个框架中创建多个窗口，可以同时查看和编辑多个缓冲区。

### 窗口分割

| 快捷键  | 功能           | 说明                      |
| ------- | -------------- | ------------------------- |
| `C-x 2` | 水平分割窗口   | Split window horizontally |
| `C-x 3` | 垂直分割窗口   | Split window vertically   |
| `C-x 1` | 只保留当前窗口 | Delete other windows      |
| `C-x 0` | 关闭当前窗口   | Delete current window     |

**窗口分割技巧**：

- `C-x 2` 在当前窗口下方创建新窗口
- `C-x 3` 在当前窗口右侧创建新窗口
- 新窗口显示相同的缓冲区，可以使用 `C-x b` 切换到不同的缓冲区

### 窗口切换

| 快捷键    | 功能             | 说明                     |
| --------- | ---------------- | ------------------------ |
| `C-x o`   | 切换到下一个窗口 | Other window             |
| `C-M-v`   | 向下滚动其他窗口 | Scroll other window down |
| `C-M-S-v` | 向上滚动其他窗口 | Scroll other window up   |

**窗口切换技巧**：

- `C-x o` 可以在多个窗口间循环切换
- `C-M-v` 可以在不切换窗口的情况下滚动其他窗口的内容

### 窗口调整

| 快捷键  | 功能         | 说明                        |
| ------- | ------------ | --------------------------- |
| `C-x ^` | 增加窗口高度 | Enlarge window vertically   |
| `C-x }` | 增加窗口宽度 | Enlarge window horizontally |
| `C-x {` | 减少窗口宽度 | Shrink window horizontally  |
| `C-x -` | 减少窗口高度 | Shrink window vertically    |

## 第八部分：模式系统

模式是 Emacs 中用于定制编辑器行为的功能集合。理解模式系统是高效使用 Emacs 的关键。

### 主模式（Major Mode）

主模式定义了缓冲区的主要编辑行为。每个缓冲区只能有一个主模式。

**常见主模式**：

| 模式               | 说明        | 快捷键                   |
| ------------------ | ----------- | ------------------------ |
| `fundamental-mode` | 基础模式    | 默认模式                 |
| `text-mode`        | 文本模式    | 用于编辑纯文本           |
| `prog-mode`        | 编程模式    | 编程语言的基类模式       |
| `python-mode`      | Python 模式 | 用于编辑 Python 代码     |
| `emacs-lisp-mode`  | Elisp 模式  | 用于编辑 Emacs Lisp 代码 |
| `org-mode`         | Org 模式    | 用于编辑 Org 文档        |

**切换主模式**：

- 使用 `M-x` 然后输入模式名称，如 `M-x python-mode`
- 打开文件时，Emacs 会根据文件扩展名自动选择合适的主模式

### 次模式（Minor Mode）

次模式提供额外的功能，每个缓冲区可以有多个次模式。

**常见次模式**：

| 模式             | 说明     | 快捷键       |
| ---------------- | -------- | ------------ |
| `linum-mode`     | 行号显示 | 显示行号     |
| `auto-fill-mode` | 自动换行 | 自动换行     |
| `abbrev-mode`    | 缩写模式 | 自动展开缩写 |
| `flyspell-mode`  | 拼写检查 | 实时拼写检查 |

**切换次模式**：

- 使用 `M-x` 然后输入模式名称，如 `M-x linum-mode`
- 某些次模式有快捷键，如 `M-x linum-mode` 可以切换行号显示

## 第九部分：Vim 键位支持 - Evil 模式

对于习惯于 Vim 键位的用户，Emacs 提供了 Evil 模式（Emulation Vim Layer），可以在 Emacs 中使用 Vim 的键位和操作方式。这使得用户既能享受 Emacs 的强大功能，又能保持 Vim 的高效操作习惯。

### Evil 模式简介

**Evil 模式是什么**

Evil 是一个 Emacs 扩展插件，旨在模拟 Vim 的键位和操作方式。通过启用 Evil 模式，用户可以在 Emacs 中使用 Vim 的大部分键位和命令，包括普通模式、插入模式、可视模式等。

**Evil 模式的优势**

- **Vim 键位**：支持 Vim 的大部分键位和命令
- **Emacs 功能**：可以同时使用 Emacs 的强大功能和扩展
- **无缝集成**：Evil 模式与 Emacs 的其他功能良好集成
- **高度可定制**：可以通过配置调整 Evil 模式的行为

### 安装和启用 Evil 模式

**使用 package.el 安装**

```elisp
;; 在 Emacs 配置文件中添加
(require 'package)
(add-to-list 'package-archives
             '("melpa" . "https://melpa.org/packages/"))
(package-initialize)

;; 安装 evil
(unless (package-installed-p 'evil)
  (package-refresh-contents)
  (package-install 'evil))

;; 启用 evil
(require 'evil)
(evil-mode 1)
```

**使用 use-package 安装（推荐）**

```elisp
;; 使用 use-package 管理配置
(use-package evil
  :ensure t
  :init
  (evil-mode 1))
```

### Evil 模式的基本使用

**模式切换**

启用 Evil 模式后，您可以使用 Vim 的模式切换方式：

| 快捷键 | 功能           | 说明                   |
| ------ | -------------- | ---------------------- |
| `i`    | 进入插入模式   | 在光标位置前插入文本   |
| `a`    | 进入插入模式   | 在光标位置后追加文本   |
| `v`    | 进入可视模式   | 字符选择模式           |

**光标移动**

在 Evil 模式的普通模式下，可以使用 Vim 的光标移动方式：

| 快捷键 | 功能                 | 说明               |
| ------ | -------------------- | ------------------ |
| `h`    | 左移一个字符         | 向左移动光标       |
| `l`    | 右移一个字符         | 向右移动光标       |

### Evil 模式与 Emacs 模式的切换

**临时使用 Emacs 键位**

在 Evil 模式下，可以使用 `C-z` 临时切换到 Emacs 键位模式：

| 快捷键 | 功能              | 说明                        |
| ------ | ----------------- | --------------------------- |
| `C-z`  | 切换到 Emacs 模式 | 临时使用 Emacs 键位         |
| `C-z`  | 切换回 Evil 模式  | 再次按 `C-z` 返回 Evil 模式 |

**注意**：在终端中，`C-z` 可能会挂起进程。如果需要挂起功能，可以使用 `C-x C-z`。

### Spacemacs - 集成的解决方案

**Spacemacs 简介**

Spacemacs 是一个基于 Emacs 的社区驱动的配置框架，旨在将 Emacs 和 Vim 的优点结合在一起。它默认集成了 Evil 模式，并提供了丰富的插件和配置，方便用户快速上手。

**Spacemacs 的特点**

- **默认集成 Evil**：默认启用 Evil 模式，支持 Vim 键位
- **丰富的插件**：预配置了大量常用插件
- **统一配置**：使用统一的配置系统
- **社区支持**：活跃的社区和丰富的文档

**安装 Spacemacs**

```bash
# 备份现有配置（如果有）
mv ~/.emacs.d ~/.emacs.d.backup

# 克隆 Spacemacs
git clone https://github.com/syl20bnr/spacemacs ~/.emacs.d
```

首次启动 Emacs 时，Spacemacs 会询问您选择编辑风格：

- `vim`：使用 Vim 键位（推荐给 Vim 用户）
- `emacs`：使用 Emacs 键位
- `hybrid`：混合模式

### Evil 模式的高级配置

**自定义键位映射**

```elisp
;; 自定义 Evil 模式的键位映射
(evil-define-key 'normal 'global-map
  (kbd "C-h") 'evil-window-left
  (kbd "C-j") 'evil-window-down
  (kbd "C-k") 'evil-window-up
  (kbd "C-l") 'evil-window-right)
```

**配置 Evil 模式的行为**

```elisp
;; 配置 Evil 模式
(setq evil-want-C-i-jump nil)  ; 禁用 C-i 跳转
(setq evil-want-C-u-scroll t)  ; 启用 C-u 滚动
(setq evil-want-Y-yank-to-eol t)  ; Y 复制到行尾
```

### Evil 模式的限制和注意事项

**不完全兼容**

虽然 Evil 模式模拟了 Vim 的大部分功能，但并不是 100% 兼容。某些 Vim 的高级功能可能无法完全模拟。

**与 Emacs 功能的集成**

某些 Emacs 功能可能需要使用 Emacs 的快捷键（如 `C-x C-f` 打开文件），但 Evil 模式提供了替代方案，如使用 `:e` 命令。

**性能考虑**

在某些情况下，Evil 模式可能会略微影响性能，但对于大多数用户来说，这种影响是可以忽略的。
