## 📋 概述

## 📝 Frontmatter 规范

所有博客文章必须包含完整的 frontmatter，使用 YAML 格式：

```yaml
---
title: 文章标题
date: 'YYYY-MM-DD'
tags: [标签1, 标签2, 标签3]
author: 作者名
description: 文章描述
image: "/images/articles/xxx.jpg"  # 可选
draft: false  # 可选，默认 false
summary: 文章摘要  # 可选
showTOC: true  # 可选，显示目录
category: 分类  # 可选
---
```

### 必填字段说明
- **title**: 简洁明确，能准确概括文章内容
- **date**: 文章发布日期，格式为 'YYYY-MM-DD'
- **tags**: 3-6个相关标签，使用英文，首字母大写
- **author**: 作者英文名
- **description**: 1-2句话概括文章内容，用于 SEO

### 可选字段说明
- **image**: 文章封面图片路径，建议尺寸 1200x630
- **draft**: 是否为草稿，草稿不会发布
- **summary**: 文章摘要，比 description 更详细
- **showTOC**: 是否自动生成目录
- **category**: 文章分类，对应目录名

### 写作风格要求

#### 语言风格
- **正式专业**: 使用专业术语，避免口语化表达
- **简洁明了**: 句子不宜过长，保持逻辑清晰
- **客观准确**: 基于事实，避免主观臆断
- **时态统一**: 技术描述使用现在时，历史回顾使用过去时

#### 内容组织
- **逻辑递进**: 从基础到高级，从理论到实践
- **层次分明**: 使用清晰的小标题结构
- **段落适中**: 每段聚焦一个核心观点
- **过渡自然**: 使用恰当的过渡词句

## 🎨 格式与样式

### 文本格式
- **粗体**: 用于强调重要概念 `**重要概念**`
- **斜体**: 用于变量、书籍名 `*变量名*`
- **行内代码**: 使用反引号 `` `code()` ``
- **下标**: 使用 HTML 标签 `<sub>下标</sub>`
- **上标**: 使用 HTML 标签 `<sup>上标</sup>`

## 🔧 技术内容规范

### 代码块
- 使用三个反引号包裹，指定语言
- 提供清晰的注释
- 重要代码行需要详细解释

```python
def example_function():
    """函数说明"""
    # 初始化
    x = 0
    return x
```

### 数学公式
- 使用 KaTeX 语法
- 行内公式：`$E = mc^2$`
- 独立公式：
  ```latex
  $$f(x) = \int_{-\infty}^{\infty} g(t) dt$$
  ```

### 图表与可视化
- **ASCII 图表**: 用于简单的信号波形图
  ```text
  信号 A: ──┐ ┌─┐ ┌─┐
          │ │ │ │ │
          └─┘ └─┘ └─┘
  ```
- **Mermaid 图表**: 用于流程图、时序图
  ```mermaid
  graph LR
    A[输入] --> B[处理]
    B --> C[输出]
  ```

### 图片规范
- 图片存放在 `/public/images/articles/` 目录
- 文件名使用英文，与文章相关
- 使用 Next.js Image 组件：
  ```jsx
  <Image
    src="/images/articles/example.jpg"
    alt="示例图片"
    width={800}
    height={400}
  />
  ```

## 🚀 发布流程

1. **草稿阶段**
   - 创建 MDX 文件，`draft: true`
   - 完成初稿，补充基本内容

2. **审核阶段**
   - 技术审核：确保内容准确
   - 格式审核：符合规范要求
   - 语言审核：无语法错误

3. **测试阶段**
   - 本地预览，确保渲染正常
   - 检查链接有效性
   - 验证代码可运行性

4. **发布阶段**
   - 设置 `draft: false`
   - 提交到代码仓库
   - 更新相关文档
