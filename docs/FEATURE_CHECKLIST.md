## 1. 用户认证与账户管理

| # | 功能 | 测试方法 | ✓ |
|---|------|---------|---|
| 1.1 | **用户注册** — 新用户可通过邮箱/用户名/密码注册 | 访问 `/login` → 切换到注册模式 → 填写表单 → 提交 | ☐ |
| 1.2 | **用户登录** — 已注册用户可通过邮箱+密码登录 | 访问 `/login` → 输入邮箱和密码 → 登录成功跳转首页 | ☐ |
| 1.3 | **登录后重定向** — 支持 `?redirect=` 参数，登录后跳回目标页 | 未登录时访问 `/settings` → 重定向到 `/login?redirect=/settings` → 登录 → 自动跳回 `/settings` | ☐ |
| 1.4 | **JWT Token 刷新** — Access Token 过期后自动用 Refresh Token 续期 | 等待 access token 过期 → 刷新页面 → 应自动续期不退出登录 | ☐ |
| 1.5 | **Token 轮转防窃** — 同一 Refresh Token 家族中旧 Token 被使用则整族吊销 | （安全测试，暂不列为常规回归） | ☐ |
| 1.6 | **登出** — 用户可主动退出登录 | 登录后 → 点击登出按钮 → 跳转首页，访问 `/settings` 需重新登录 | ☐ |
| 1.7 | **密码重置** — 通过邮箱发送重置链接 | 登录页点击"忘记密码" → 输入邮箱 → 检查邮件 → 使用链接重置 | ☐ |
| 1.8 | **邮箱验证** — 注册后发送验证邮件 | 新注册用户 → 检查收件箱 → 点击验证链接 → 数据库 `email_verified=true` | ☐ |
| 1.9 | **登录限流** — 连续失败后触发速率限制 | 连续输错密码 5 次 → 应返回 429 Too Many Requests | ☐ |
| 1.10 | **角色权限** — `admin` / `moderator` / `user` 三种角色 | 用 user 角色访问 `/admin` → 应拒绝；admin 角色可访问 | ☐ |
| 1.11 | **账户状态** — `suspended` / `banned` 用户无法登录 | 后台将某用户设为 banned → 该用户登录应失败 | ☐ |
