# VLA-Scratch深度解析：模块化高性能VLA训练框架

VLA-Scratch是由EGalahad开发的一个模块化、高性能的Vision-Language-Action（VLA）模型训练框架。该框架旨在为机器人学习提供一套完整的训练基础设施，具有清晰的模块化设计、卓越的性能优化和灵活的实验工作流。

## 1. VLA-Scratch概述

### 1.1 项目背景

VLA（Vision-Language-Action）模型是机器人学习领域的前沿方向，它将视觉感知、语言理解和动作生成统一到一个框架中。VLA-Scratch提供了一个从零开始构建和训练VLA模型的完整解决方案。

### 2.2 Transform Pipeline

VLA-Scratch通过四阶段的Transform Pipeline处理数据：

```python
class TransformPipeline:
    """VLA-Scratch的Transform Pipeline"""

    def __init__(self):
        self.pipeline = [
            'dataset_transforms',    # 数据集特定转换
            'normalization',         # 标准化
            'to_data_sample',        # 转换为TensorClass
            'policy_transforms'      # 策略特定转换
        ]

    def build_pipeline(self, dataset, policy):
        """构建完整的转换管道"""
        pipeline = []

        # 阶段1: 数据集转换
        pipeline.extend(dataset.get_transforms())

        # 阶段2: 标准化
        pipeline.append(Normalize())

        # 阶段3: 转换为DataSample
        pipeline.append(ToDataSample())

        # 阶段4: 策略转换
        pipeline.extend(policy.get_transforms())

        return pipeline

# Pipeline数据流
# 原始数据 -> Dataset Keys -> Processed Keys -> Normalized -> DataSample -> Policy Input
```

数据演进的各个阶段：

| 阶段 | 输出表示 | 描述 |
|------|---------|------|
| 原始数据集 | 数据集特定键 (如`CAM_FRONT_KEY`) | 每个数据集的原始模态 |
| 数据集转换 | 规范化处理键 (如`PROCESSED_STATE_KEY`) | 统一的中间表示 |
| Normalize | 规范化处理键 (已标准化) | 标准/归一化的数据 |
| ToDataSample | `DataSample` TensorClass | 结构化数据对象 |
| 策略转换 | `DataSample` + `policy_input` | 策略特定的输入 |

## 3. 性能优化深度解析

VLA-Scratch实现了三个关键的性能优化，使得训练吞吐量提升2倍，内存节省2倍。

### 3.1 优化一：CPU元数据存储

**问题**：在图像特征编码过程中，位置嵌入需要根据图像的空间维度进行插值。原始实现中，形状元数据（高度和宽度）从CUDA张量中读取，导致频繁的主机-设备同步。

```python
# 原始实现 - 导致同步
def pos_embed_interpolate_original(self, grid_thw):
    grid_ts, grid_hs, grid_ws = grid_thw.unbind(1)
    for t, h, w in zip(grid_ts, grid_hs, grid_ws):
        # 从CUDA张量读取形状 - 触发同步
        h_idxs = torch.arange(0, h, self.step)  # h在GPU上
        w_idxs = torch.arange(0, w, self.step)

# 优化实现 - 避免同步
def pos_embed_interpolate_optimized(self, grid_thw, img_h, img_w):
    grid_ts, grid_hs, grid_ws = grid_thw.unbind(1)
    for t, h, w in zip(grid_ts, grid_hs, grid_ws):
        # 直接使用CPU整数 - 无同步
        h_idxs = torch.arange(0, img_h, self.step, device='cpu')
        w_idxs = torch.arange(0, img_w, self.step, device='cpu')
```

**优化策略**：将动态形状的元数据（高度和宽度）作为CPU整数存储在dataloader输出中，避免不必要的同步。

### 3.2 优化二：预处理3D-RoPE索引

**问题**：Qwen3-VL的3D旋转位置嵌入（3D-RoPE）索引计算需要对每个token进行条件逻辑判断，在CUDA张量上执行时引入细粒度的同步。

```python
# 原始实现 - 前向传播中的条件逻辑
def compute_rope_original(self, input_ids):
    position_ids = []
    for token in input_ids:  # token在GPU上
        if token == vision_start_token_id:      # 条件判断触发同步
            ...
        if token == image_token_id:             # 条件判断触发同步
            ...
    return torch.tensor(position_ids)

# 优化实现 - 数据加载器中预计算
def compute_rope_in_dataloader(self, input_ids):
    """在数据加载器worker中预计算"""
    position_ids = []
    for token in input_ids.cpu():  # 在CPU上执行
        if token == vision_start_token_id:
            ...
        if token == image_token_id:
            ...
    return torch.tensor(position_ids)
```

**优化策略**：将不依赖模型激活的索引计算移到数据加载器的多进程worker中，与模型计算并行执行。

### 3.3 优化三：形状稳定的Forward Pass

**问题**：DeepStack操作将图像嵌入融合到LLM层的隐藏状态中时，创建依赖于CUDA张量值的动态形状张量。

```python
# 原始实现 - 动态形状
local_delta = hidden_states[visual_pos_masks, :].clone() + visual_embeds
hidden_states[visual_pos_masks, :] = local_delta  # 动态索引触发同步

# 优化实现 - 形状稳定
global_delta = torch.zeros_like(hidden_states)  # 固定形状
global_delta.masked_scatter_(visual_pos_masks.unsqueeze(-1), visual_embeds)
hidden_states = hidden_states + global_delta  # 无动态索引
```

**优化策略**：使用`masked_scatter`算子保持静态张量形状，避免动态索引。

### 3.4 内存优化：FSDP + 梯度检查点

```python
class MemoryOptimizedTraining:
    """内存优化的训练配置"""

    def __init__(self):
        self.fsdp_config = {
            'sharding_strategy': 'FULL_SHARD',      # 完全分片
            'cpu_ram_efficient_loading': True,       # CPU内存高效加载
            'auto_wrap_policy': 'transformer_auto_wrap_policy',
        }

        self.gradient_checkpointing = True  # 梯度检查点

    def apply_layerwise_sharding(self, model):
        """应用分层FSDP分片"""
        # 为不同层应用不同的分片策略
        for name, module in model.named_modules():
            if 'vision_encoder' in name:
                # 视觉编码器使用更大分片
                module.sharding_size = 4
            elif 'action_head' in name:
                # 动作头使用小分片
                module.sharding_size = 2
```

```python
# vla_scratch/transforms/normalization.py
class Normalize:
    """标准化转换"""

    def __init__(self, stats_path):
        self.stats = torch.load(stats_path)

    def __call__(self, data):
        data[PROCESSED_STATE_KEY] = (
            data[PROCESSED_STATE_KEY] - self.stats['state_mean']
        ) / self.stats['state_std']
        return data
```

### 4.4 Policies模块

```python
# vla_scratch/policies/base.py
class BasePolicy:
    """策略基类"""

    def encode_prefix(self, observation):
        """编码前缀（观测）"""
        raise NotImplementedError

    def predict_suffix(self, vlm_outputs, actions=None):
        """预测后缀（动作）"""
        raise NotImplementedError

    def compute_loss(self, batch):
        """计算损失"""
        # 编码观测
        vlm_outputs = self.encode_prefix(batch.observation)

        # 计算动作损失
        loss = self.predict_suffix(vlm_outputs, batch.actions)
        return loss

    def sample_actions(self, observation):
        """采样动作"""
        vlm_outputs = self.encode_prefix(observation)
        actions = self.predict_suffix(vlm_outputs)
        return actions

# vla_scratch/policies/pi/config.py
@dataclass
class PiPolicyConfig:
    """Pi策略配置"""
    _target_: str = "vla_scratch.policies.pi.PiPolicy"

    # VLM配置
    vlm_backbone: str = "qwen2-vl"
    vlm_path: str = "Qwen/Qwen2-VL-7B"

    # 动作配置
    action_horizon: int = 10
    state_history: int = 0
    use_state: bool = False

    # 转换配置
    transforms: list = None

# vla_scratch/policies/modules/action_expert/dit.py
class ActionExpert(nn.Module):
    """DiT风格动作专家网络"""

    def __init__(self, config):
        super().__init__()
        self.diffusion_model = DiT1D(
            input_dim=config.action_dim,
            hidden_dim=config.hidden_dim,
            num_layers=config.num_layers
        )

    def forward(self, vlm_features, noisy_actions, timesteps):
        """预测流匹配速度场"""
        # 拼接VLM特征和噪声动作
        combined = torch.cat([vlm_features, noisy_actions], dim=-1)

        # 预测速度场
        velocity = self.diffusion_model(combined, timesteps)
        return velocity

# vla_scratch/policies/modules/vlm_bridge/qwen/bridge.py
class QwenBridge:
    """Qwen VLM桥接器"""

    def __init__(self, config):
        self.model = QwenVLModel(config)
        self.processor = QwenProcessor(config)

    def forward(self, observation):
        """优化的前向传播"""
        # 预处理的policy_input
        policy_input = observation.policy_input

        # 使用预计算的RoPE索引
        outputs = self.model(
            input_ids=policy_input['input_ids'],
            pixel_values=policy_input['pixel_values'],
            image_grid_thw=policy_input['image_grid_thw'],
            rope_indices=policy_input['rope_indices']  # 预计算！
        )

        return VLMOutputs(
            hidden_states=outputs.last_hidden_state,
            attention_mask=outputs.attention_mask
        )
```

### 4.5 Scripts模块

```python
# scripts/train_policy.py
import hydra
from omegaconf import DictConfig

@hydra.main(config_path="conf", config_name="config")
def train_policy(cfg: DictConfig):
    """训练脚本"""
    # 创建数据集
    train_dataset = create_dataset(cfg.data)

    # 创建策略
    policy = hydra.utils.instantiate(cfg.policy)

    # 应用FSDP和梯度检查点
    policy = apply_fsdp(policy, cfg.fsdp)
    policy = apply_gradient_checkpointing(policy)

    # 训练循环
    for epoch in range(cfg.num_epochs):
        for batch in train_dataset:
            loss = policy.compute_loss(batch)
            loss.backward()
            optimizer.step()
            optimizer.zero_grad()

# 使用示例
# train_policy.py policy=pi-qwen data=libero-spatial batch_size=32
```
