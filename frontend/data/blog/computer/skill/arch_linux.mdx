## 引言

Arch Linux 以其简洁、灵活和高度可定制的特性，成为追求极致性能和完全控制权的 Linux 用户的首选。配合 Hyprland 这一现代化的 Wayland 合成器，可以打造出一个既美观又高性能的桌面环境。

本文将从零开始，详细介绍如何搭建一个**高性能、高鲁棒性**的 Arch Linux 系统，并配置 Hyprland 桌面环境。我们将重点关注：

- **性能优化**：系统级和桌面级的性能调优
- **稳定性配置**：确保系统长期稳定运行
- **最佳实践**：遵循 Arch Linux 社区的最佳实践
- **故障恢复**：配置备份和恢复机制

## 第一部分：Arch Linux 基础安装

### 准备工作

**1. 下载 Arch Linux ISO**

从官方或镜像站下载最新的 Arch Linux ISO：
- 官方站点：https://archlinux.org/download/
- 国内镜像：清华大学、中科大等镜像站

**2. 制作启动盘**

```bash
# Linux/macOS 使用 dd 命令
sudo dd if=archlinux-xxxx.xx.xx-x86_64.iso of=/dev/sdX bs=4M status=progress oflag=sync

# Windows 使用 Rufus 或 balenaEtcher
```

**3. 启动 Live 环境**

- 设置 BIOS/UEFI 从 USB 启动
- 进入 Arch Linux Live 环境
- 确保网络连接正常

### 网络配置

**有线网络（通常自动连接）**

```bash
# 检查网络接口
ip link

# 如果未自动连接，手动启动
systemctl start dhcpcd@enp0s3.service  # 替换为实际接口名
```

**无线网络**

```bash
# 使用 iwd 连接 Wi-Fi
iwctl

# 在 iwd 提示符下
[iwd]# device list
[iwd]# station wlan0 scan
[iwd]# station wlan0 get-networks
[iwd]# station wlan0 connect <SSID>
[iwd]# exit

# 验证连接
ping -c 3 archlinux.org
```

### 磁盘分区

**推荐分区方案（UEFI + GPT）**

```bash
# 查看磁盘
lsblk
fdisk -l

# 使用 cfdisk 进行分区（推荐）
cfdisk /dev/nvme0n1  # 或 /dev/sda

# 分区方案：
# /dev/nvme0n1p1: EFI 系统分区 (512MB-1GB, FAT32)
# /dev/nvme0n1p2: 交换分区 (可选，建议内存大小或 2x 内存)
# /dev/nvme0n1p3: 根分区 (剩余空间, ext4 或 BTRFS)
```

**高级分区方案（BTRFS + 子卷）**

对于追求性能和鲁棒性的用户，推荐使用 BTRFS 文件系统：

```bash
# 创建分区
cfdisk /dev/nvme0n1

# 格式化 EFI 分区
mkfs.fat -F32 /dev/nvme0n1p1

# 格式化 BTRFS 根分区
mkfs.btrfs -L ArchRoot /dev/nvme0n1p3

# 挂载根分区
mount /dev/nvme0n1p3 /mnt

# 创建 BTRFS 子卷（推荐结构）
cd /mnt
btrfs subvolume create @
btrfs subvolume create @home
btrfs subvolume create @var
btrfs subvolume create @snapshots

# 卸载并重新挂载
cd /
umount /mnt

# 挂载根子卷
mount -o subvol=@,compress=zstd:3,noatime /dev/nvme0n1p3 /mnt

# 创建目录并挂载其他子卷
mkdir -p /mnt/{boot/efi,home,var,.snapshots}
mount -o subvol=@home,compress=zstd:3,noatime /dev/nvme0n1p3 /mnt/home
mount -o subvol=@var,compress=zstd:3,noatime /dev/nvme0n1p3 /mnt/var
mount -o subvol=@snapshots,compress=zstd:3,noatime /dev/nvme0n1p3 /mnt/.snapshots

# 挂载 EFI 分区
mount /dev/nvme0n1p1 /mnt/boot/efi

# 如果使用交换分区
mkswap /dev/nvme0n1p2
swapon /dev/nvme0n1p2
```

**传统 ext4 方案（更简单）**

```bash
# 格式化分区
mkfs.fat -F32 /dev/nvme0n1p1
mkfs.ext4 /dev/nvme0n1p3
mkswap /dev/nvme0n1p2
swapon /dev/nvme0n1p2

# 挂载
mount /dev/nvme0n1p3 /mnt
mkdir -p /mnt/boot/efi
mount /dev/nvme0n1p1 /mnt/boot/efi
```

### 配置镜像源（可选但推荐）

```bash
# 编辑镜像列表
vim /etc/pacman.d/mirrorlist

# 或使用 reflector 自动选择最快的镜像
pacman -Sy reflector
reflector --country China --latest 10 --sort rate --save /etc/pacman.d/mirrorlist
```

### 安装基本系统

```bash
# 安装基础系统包
pacstrap /mnt base base-devel linux linux-firmware linux-headers

# 如果使用 BTRFS，添加 btrfs-progs
pacstrap /mnt base base-devel linux linux-firmware linux-headers btrfs-progs

# 生成 fstab
genfstab -U /mnt >> /mnt/etc/fstab

# 检查 fstab（BTRFS 用户需要确认挂载选项）
cat /mnt/etc/fstab
```

### 进入新系统

```bash
# 切换到新系统
arch-chroot /mnt

# 设置时区
ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
hwclock --systohc

# 本地化设置
vim /etc/locale.gen
# 取消注释：en_US.UTF-8 UTF-8 和 zh_CN.UTF-8 UTF-8
locale-gen
echo "LANG=en_US.UTF-8" > /etc/locale.conf

# 设置主机名
echo "myarch" > /etc/hostname

# 配置 hosts
cat >> /etc/hosts << EOF
127.0.0.1	localhost
::1		localhost
127.0.1.1	myarch.localdomain	myarch
EOF
```

### 安装引导加载程序

**GRUB（推荐）**

```bash
# 安装 GRUB
pacman -S grub efibootmgr

# 安装到 EFI 分区
grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB

# 生成配置文件
grub-mkconfig -o /boot/grub/grub.cfg
```

**systemd-boot（更轻量）**

```bash
# 安装 systemd-boot
bootctl install

# 创建引导条目
cat > /boot/loader/entries/arch.conf << EOF
title   Arch Linux
linux   /vmlinuz-linux
initrd  /initramfs-linux.img
options root=UUID=你的根分区UUID rw
EOF

# 配置引导加载器
cat > /boot/loader/loader.conf << EOF
default arch.conf
timeout 3
editor  no
EOF
```

### 创建用户

```bash
# 设置 root 密码
passwd

# 创建普通用户（添加到 wheel 组以使用 sudo）
useradd -m -G wheel -s /bin/bash username

# 设置用户密码
passwd username

# 配置 sudo
pacman -S sudo
visudo
# 取消注释：%wheel ALL=(ALL) ALL
```

### 完成安装

```bash
# 退出 chroot
exit

# 卸载分区
umount -R /mnt

# 重启
reboot
```

### 性能优化配置

**1. 内核参数优化**

编辑 `/etc/default/grub`：

```bash
sudo vim /etc/default/grub
```

添加以下参数：

```bash
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash nowatchdog mitigations=off iommu=soft pcie_aspm.policy=performance"
```

对于 NVIDIA 用户：

```bash
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash nvidia-drm.modeset=1 nowatchdog mitigations=off iommu=soft pcie_aspm.policy=performance"
```

应用更改：

```bash
sudo grub-mkconfig -o /boot/grub/grub.cfg
```

**2. 系统服务优化**

禁用不必要的服务：

```bash
# 查看所有服务
systemctl list-unit-files --type=service

# 禁用不必要的服务（根据需求调整）
sudo systemctl disable bluetooth.service  # 如果不用蓝牙
sudo systemctl disable avahi-daemon.service
sudo systemctl disable cups.service  # 如果不用打印机
```

**3. 文件系统优化**

**BTRFS 优化（如果使用）**

编辑 `/etc/fstab`，确保挂载选项包含：

```bash
subvol=@,compress=zstd:3,noatime,space_cache=v2,commit=120
```

创建 BTRFS 快照脚本：

```bash
sudo vim /usr/local/bin/btrfs-snapshot.sh
```

```bash
#!/bin/bash
# BTRFS 快照脚本

SNAPSHOT_DIR="/.snapshots"
DATE=$(date +%Y%m%d-%H%M%S)

# 创建根分区快照
btrfs subvolume snapshot -r / /.snapshots/root-$DATE

# 创建家目录快照
btrfs subvolume snapshot -r /home /.snapshots/home-$DATE

# 保留最近 10 个快照
ls -dt /.snapshots/root-* | tail -n +11 | xargs -r btrfs subvolume delete
ls -dt /.snapshots/home-* | tail -n +11 | xargs -r btrfs subvolume delete

echo "快照创建完成: $DATE"
```

```bash
sudo chmod +x /usr/local/bin/btrfs-snapshot.sh

# 添加到 crontab（每天凌晨 2 点）
sudo crontab -e
# 添加：0 2 * * * /usr/local/bin/btrfs-snapshot.sh
```

**ext4 优化**

编辑 `/etc/fstab`，添加挂载选项：

```bash
noatime,commit=60
```

**4. CPU 调频优化**

```bash
# 安装 CPU 调频工具
sudo pacman -S cpupower

# 设置性能模式
sudo cpupower frequency-set -g performance

# 永久设置（创建 systemd 服务）
sudo vim /etc/systemd/system/cpupower.service
```

```ini
[Unit]
Description=CPU performance governor

[Service]
Type=oneshot
ExecStart=/usr/bin/cpupower frequency-set -g performance

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable cpupower.service
```

**5. I/O 调度器优化**

```bash
# 查看当前调度器
cat /sys/block/nvme0n1/queue/scheduler

# 设置调度器（SSD 推荐 none 或 mq-deadline，HDD 推荐 mq-deadline）
echo mq-deadline | sudo tee /sys/block/nvme0n1/queue/scheduler

# 永久设置
sudo vim /etc/udev/rules.d/60-io-scheduler.rules
```

```
ACTION=="add|change", KERNEL=="sd[a-z]|nvme[0-9]*", ATTR{queue/scheduler}="mq-deadline"
```

**6. 内存优化**

```bash
# 编辑 sysctl 配置
sudo vim /etc/sysctl.d/99-sysctl.conf
```

添加以下内容：

```bash
# 减少 swap 使用倾向
vm.swappiness=10

# 提高文件系统缓存
vm.vfs_cache_pressure=50

# 网络优化
net.core.rmem_max = 16777216
net.core.wmem_max = 16777216
net.ipv4.tcp_rmem = 4096 87380 16777216
net.ipv4.tcp_wmem = 4096 65536 16777216
net.ipv4.tcp_congestion_control = bbr

# 文件描述符限制
fs.file-max = 2097152
```

应用配置：

```bash
sudo sysctl --system
```

**7. 启用 BBR（TCP 拥塞控制）**

```bash
# 检查是否已启用
sysctl net.ipv4.tcp_congestion_control

# 如果未启用，编辑 sysctl 配置（已在上面添加）
sudo sysctl -p /etc/sysctl.d/99-sysctl.conf
```

## 第四部分：系统鲁棒性配置

### 1. 自动备份配置

创建备份脚本 `~/.local/bin/backup-config.sh`：

```bash
#!/bin/bash
# 配置文件备份脚本

BACKUP_DIR="$HOME/.config-backups"
DATE=$(date +%Y%m%d-%H%M%S)
BACKUP_PATH="$BACKUP_DIR/$DATE"

mkdir -p "$BACKUP_PATH"

# 备份重要配置
cp -r ~/.config/hypr "$BACKUP_PATH/"
cp -r ~/.config/waybar "$BACKUP_PATH/"
cp -r ~/.config/rofi "$BACKUP_PATH/"
cp -r ~/.config/alacritty "$BACKUP_PATH/"

# 备份系统配置
sudo cp /etc/fstab "$BACKUP_PATH/"
sudo cp /etc/default/grub "$BACKUP_PATH/"

# 压缩备份
cd "$BACKUP_DIR"
tar -czf "$DATE.tar.gz" "$DATE"
rm -rf "$DATE"

# 保留最近 7 天的备份
find "$BACKUP_DIR" -name "*.tar.gz" -mtime +7 -delete

echo "备份完成: $BACKUP_PATH.tar.gz"
```

```bash
chmod +x ~/.local/bin/backup-config.sh

# 添加到 crontab（每天凌晨 3 点）
crontab -e
# 添加：0 3 * * * $HOME/.local/bin/backup-config.sh
```

### 4. 错误恢复

**创建恢复脚本**

```bash

```

```bash
#!/bin/bash
# 系统恢复脚本

echo "=== Arch Linux 系统恢复工具 ==="
echo "1. 检查文件系统"
echo "2. 重建 initramfs"
echo "3. 更新 GRUB"
echo "4. 清理 pacman 缓存"
echo "5. 检查损坏的包"

read -p "选择操作 (1-5): " choice

case $choice in
    1)
        echo "检查文件系统..."
        sudo fsck -A -y
        ;;
    2)
        echo "重建 initramfs..."
        sudo mkinitcpio -P
        ;;
    3)
        echo "更新 GRUB..."
        sudo grub-mkconfig -o /boot/grub/grub.cfg
        ;;
    4)
        echo "清理 pacman 缓存..."
        sudo pacman -Sc
        ;;
    5)
        echo "检查损坏的包..."
        sudo pacman -Qkk
        ;;
    *)
        echo "无效选择"
        ;;
esac
```

```bash
sudo chmod +x /usr/local/bin/system-recovery.sh
```

### 5. 定期维护

创建维护脚本：

```bash
vim ~/.local/bin/system-maintenance.sh
```

```bash
#!/bin/bash
# 系统维护脚本

echo "=== Arch Linux 系统维护 ==="

# 更新系统
echo "更新系统..."
sudo pacman -Syu

# 清理 pacman 缓存
echo "清理 pacman 缓存..."
sudo pacman -Sc

# 清理 AUR 构建缓存
echo "清理 AUR 构建缓存..."
yay -Sc

# 清理 orphan 包
echo "清理孤立包..."
sudo pacman -Rns $(pacman -Qtdq) 2>/dev/null || true

# 清理系统日志
echo "清理系统日志..."
sudo journalctl --vacuum-time=30d

# 如果使用 BTRFS，清理旧快照
if mount | grep -q btrfs; then
    echo "清理旧 BTRFS 快照..."
    sudo btrfs subvolume list /.snapshots | tail -n +11 | awk '{print $9}' | xargs -r -I {} sudo btrfs subvolume delete /.snapshots/{}
fi

echo "维护完成！"
```

```bash
chmod +x ~/.local/bin/system-maintenance.sh

# 每周执行一次
crontab -e
# 添加：0 2 * * 0 $HOME/.local/bin/system-maintenance.sh
```
