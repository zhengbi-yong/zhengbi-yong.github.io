# PyTorch Lightning 完全指南：让深度学习训练更简洁高效

## 前言：为什么选择 PyTorch Lightning？

在深度学习领域，**PyTorch** 以其灵活性和强大的功能成为研究者和工程师的首选框架。然而，随着项目规模的增长，vanilla PyTorch 代码中充斥着大量重复的训练循环、样板代码和设备管理逻辑。这不仅降低了代码可读性，还增加了维护成本。

**PyTorch Lightning** 应运而生——它是 PyTorch 的轻量级封装，旨在将研究代码与工程代码分离，让你专注于模型架构本身，而将训练、验证、日志记录、分布式训练等工程细节交给框架自动处理。

### Lightning 的核心优势

#### 1. **消除样板代码**

Vanilla PyTorch 训练循环通常需要 50+ 行代码，而 Lightning 将其减少到零——你只需定义模型逻辑。

```python
# Vanilla PyTorch - 需要手动管理训练循环
for epoch in range(epochs):
    for batch in train_loader:
        optimizer.zero_grad()
        loss = model(batch)
        loss.backward()
        optimizer.step()
    # 验证循环、日志记录、检查点保存...

# PyTorch Lightning - 只需定义模型逻辑
class LitModel(pl.LightningModule):
    def training_step(self, batch, batch_idx):
        loss = self(batch)
        return loss
# Trainer 自动处理所有训练逻辑
```

#### 2. **无缝扩展到多 GPU/多节点**

一行代码即可从 CPU 训练扩展到多 GPU、多节点，甚至 TPU：

```python
# CPU
trainer = Trainer(max_epochs=10)

# 单 GPU
trainer = Trainer(max_epochs=10, accelerator="gpu", devices=1)

# 多 GPU (DataParallel / DistributedDataParallel)
trainer = Trainer(max_epochs=10, accelerator="gpu", devices=4, strategy="ddp")

# 多节点集群
trainer = Trainer(max_epochs=10, accelerator="gpu", devices=8, num_nodes=4)
```

#### 3. **内置最佳实践**

- **自动混合精度训练 (AMP)**：提升训练速度，减少显存占用
- **梯度累积**：模拟大 batch size
- **学习率监控**：自动检测学习率异常
- **模型检查点**：自动保存最佳模型
- **早期停止**：防止过拟合
- **40+ Callbacks**：扩展训练流程

#### 4. **标准化的实验管理**

- 自动记录所有超参数
- 集成 TensorBoard、W&B、MLflow、Comet 等日志工具
- 实验可复现性保证
- 模型版本管理

#### 5. **最新特性（2025）**

PyTorch Lightning 2.5+ 引入了多项新特性：

- **分布式张量支持**：更高效的大模型训练
- **TorchAO 集成**：更快的训练和推理优化
- **改进的编译支持**：与 `torch.compile` 更好地集成
- **性能提升**：减少开销，提升训练速度

### 1.2 第一个 Lightning 模型

让我们从一个简单的 MNIST 分类器开始，理解 Lightning 的核心模式。

#### 完整示例代码

```python
import pytorch_lightning as pl
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch.utils.data import DataLoader, random_split
from torchvision import datasets, transforms

# 1. 定义 LightningModule
class MNISTModel(pl.LightningModule):
    def __init__(self, hidden_dim=128, learning_rate=1e-3):
        super().__init__()
        self.save_hyperparameters()  # 保存超参数

        # 模型层
        self.layer_1 = nn.Linear(28 * 28, hidden_dim)
        self.layer_2 = nn.Linear(hidden_dim, 10)
        self.val_accuracy = []
        self.test_accuracy = []

    def forward(self, x):
        # 前向传播
        x = x.view(x.size(0), -1)  # 展平: [batch, 1, 28, 28] -> [batch, 784]
        x = self.layer_1(x)
        x = F.relu(x)
        x = self.layer_2(x)
        return x

    def training_step(self, batch, batch_idx):
        # 训练步骤
        x, y = batch
        y_hat = self(x)
        loss = F.cross_entropy(y_hat, y)
        self.log("train_loss", loss, prog_bar=True)
        return loss

    def validation_step(self, batch, batch_idx):
        # 验证步骤
        x, y = batch
        y_hat = self(x)
        loss = F.cross_entropy(y_hat, y)
        preds = torch.argmax(y_hat, dim=1)
        acc = (preds == y).float().mean()
        self.log("val_loss", loss, prog_bar=True)
        self.log("val_acc", acc, prog_bar=True)
        return {"val_loss": loss, "val_acc": acc}

    def test_step(self, batch, batch_idx):
        # 测试步骤
        x, y = batch
        y_hat = self(x)
        preds = torch.argmax(y_hat, dim=1)
        acc = (preds == y).float().mean()
        self.log("test_acc", acc)
        return acc

    def configure_optimizers(self):
        # 配置优化器
        return torch.optim.Adam(self.parameters(), lr=self.hparams.learning_rate)

    def prepare_data(self):
        # 下载/准备数据（只在主进程执行一次）
        datasets.MNIST(".", train=True, download=True)
        datasets.MNIST(".", train=False, download=True)

    def setup(self, stage=None):
        # 设置数据集（在每个进程上执行）
        transform = transforms.Compose([transforms.ToTensor()])
        if stage == "fit" or stage is None:
            mnist_full = datasets.MNIST(".", train=True, transform=transform)
            self.mnist_train, self.mnist_val = random_split(
                mnist_full, [55000, 5000]
            )
        if stage == "test" or stage is None:
            self.mnist_test = datasets.MNIST(".", train=False, transform=transform)

    def train_dataloader(self):
        return DataLoader(self.mnist_train, batch_size=32, num_workers=4)

    def val_dataloader(self):
        return DataLoader(self.mnist_val, batch_size=32, num_workers=4)

    def test_dataloader(self):
        return DataLoader(self.mnist_test, batch_size=32, num_workers=4)


# 2. 训练模型
if __name__ == "__main__":
    model = MNISTModel()

    # 创建 Trainer
    trainer = pl.Trainer(
        max_epochs=5,
        accelerator="auto",  # 自动检测可用硬件
        devices="auto",      # 自动使用所有可用设备
        precision=16,        # 混合精度训练
    )

    # 训练
    trainer.fit(model)

    # 测试
    trainer.test(model)

    # 预测
    # predictions = trainer.predict(model, test_dataloader)
```

**代码解析：**

1. **LightningModule**：继承自 `pl.LightningModule`，封装模型逻辑
2. **training_step**：定义单个训练步骤
3. **configure_optimizers**：配置优化器
4. **Trainer**：管理整个训练流程
5. **数据准备**：`prepare_data` + `setup` + `*_dataloader` 模式

## 第二部分：核心概念深度解析

### 2.1 LightningModule 详解

`LightningModule` 是 Lightning 的核心，它将模型逻辑、训练逻辑和数据处理逻辑组织在一起。

#### 核心方法

```python
class LitModel(pl.LightningModule):
    def __init__(self):
        super().__init__()
        # 1. 初始化模型层
        self.model = nn.Sequential(
            nn.Linear(784, 128),
            nn.ReLU(),
            nn.Linear(128, 10)
        )

    def forward(self, x):
        # 2. 前向传播（用于推理）
        return self.model(x)

    def training_step(self, batch, batch_idx):
        # 3. 训练步骤
        x, y = batch
        y_hat = self(x)
        loss = F.cross_entropy(y_hat, y)

        # 记录指标
        self.log("train_loss", loss, on_step=True, on_epoch=True, prog_bar=True)

        # 可选：返回任意值（会被 callbacks 使用）
        return {"loss": loss, "predictions": y_hat}

    def validation_step(self, batch, batch_idx):
        # 4. 验证步骤
        x, y = batch
        y_hat = self(x)
        loss = F.cross_entropy(y_hat, y)
        self.log("val_loss", loss)

    def test_step(self, batch, batch_idx):
        # 5. 测试步骤
        x, y = batch
        y_hat = self(x)
        loss = F.cross_entropy(y_hat, y)
        self.log("test_loss", loss)

    def predict_step(self, batch, batch_idx):
        # 6. 预测步骤
        x, _ = batch
        return self(x)

    def configure_optimizers(self):
        # 7. 配置优化器和学习率调度器
        optimizer = torch.optim.Adam(self.parameters(), lr=1e-3)
        scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
            optimizer, T_max=10
        )
        return {
            "optimizer": optimizer,
            "lr_scheduler": {
                "scheduler": scheduler,
                "monitor": "val_loss",  # 基于验证损失调整学习率
            }
        }
```

#### 日志记录详解

`self.log()` 方法提供了丰富的日志记录选项：

```python
# 基础用法
self.log("loss", loss)

# 高级选项
self.log(
    "train_loss",
    loss,
    on_step=True,        # 每个步骤记录
    on_epoch=True,       # 每个 epoch 记录（自动平均）
    prog_bar=True,       # 在进度条显示
    logger=True,         # 记录到日志器
    sync_dist=True,      # 分布式训练时同步
    enable_graph=False,  # 不计算梯度（节省内存）
)

# 记录多个指标
self.log_dict({
    "train_loss": loss,
    "train_acc": accuracy,
    "train_f1": f1_score
}, prog_bar=True)
```

#### 训练生命周期钩子

Lightning 提供了丰富的钩子函数，可以在训练的不同阶段插入自定义逻辑：

```python
class LitModel(pl.LightningModule):
    def on_train_start(self):
        """训练开始前"""
        print("Starting training...")

    def on_train_epoch_start(self):
        """每个 epoch 开始前"""
        print(f"Epoch {self.current_epoch} starting...")

    def on_train_batch_start(self, batch, batch_idx):
        """每个 batch 开始前"""
        if batch_idx == 0:
            print(f"First batch of epoch {self.current_epoch}")

    def on_train_batch_end(self, outputs, batch, batch_idx):
        """每个 batch 结束后"""
        pass

    def on_train_epoch_end(self):
        """每个 epoch 结束后"""
        # 示例：记录当前学习率
        opt = self.optimizers()
        current_lr = opt.param_groups[0]["lr"]
        self.log("lr", current_lr)

    def on_train_end(self):
        """训练结束后"""
        print("Training completed!")
```

### 2.2 Trainer 完全指南

`Trainer` 是 Lightning 的引擎，负责管理整个训练流程。

#### 基础配置

```python
from pytorch_lightning import Trainer

trainer = Trainer(
    # ===== 训练控制 =====
    max_epochs=100,           # 最大 epoch 数
    max_steps=10000,          # 最大步数（与 max_epochs 二选一）
    max_time="00:12:00:00",   # 最大训练时间（DD:HH:MM:SS）

    # ===== 硬件加速 =====
    accelerator="auto",       # "cpu", "gpu", "tpu", "mps", "auto"
    devices="auto",           # 设备数量：1, 4, "auto"
    precision=16,             # 32 (全精度), 16 (混合精度), "bf16"
    amp_backend="native",     # "native" (apex), "native"

    # ===== 批处理 =====
    accumulate_grad_batches=4,  # 梯度累积步数

    # ===== 检查点 =====
    enable_checkpointing=True,
    default_root_dir="checkpoints/",  # 检查点保存目录

    # ===== 日志 =====
    logger=True,              # 自动使用 TensorBoard
    log_every_n_steps=50,     # 日志记录频率

    # ===== 验证 =====
    check_val_every_n_epoch=1,     # 每个 epoch 验证一次
    val_check_interval=0.5,        # 每个 epoch 一半时验证
    limit_val_batches=1.0,         # 验证时使用的批次比例

    # ===== 早期停止 =====
    enable_progress_bar=True,
    enable_model_summary=True,

    # ===== 确定性 =====
    deterministic=True,        # 保证可复现性
    benchmark=False,           # 使用 cuDNN 自动调优（更快但不可复现）
)
```

### 2.3 DataModule：数据管理最佳实践

`LightningDataModule` 将数据集、数据变换和数据加载器封装在一起，实现数据逻辑的重用。

#### 基础 DataModule

```python
from pytorch_lightning import LightningDataModule
from torch.utils.data import DataLoader, random_split

class MNISTDataModule(LightningDataModule):
    def __init__(
        self,
        data_dir: str = "./",
        batch_size: int = 32,
        num_workers: int = 4,
    ):
        super().__init__()
        self.data_dir = data_dir
        self.batch_size = batch_size
        self.num_workers = num_workers
        self.transform = transforms.Compose([
            transforms.ToTensor(),
            transforms.Normalize((0.1307,), (0.3081,))
        ])

    def prepare_data(self):
        """下载数据（只在主进程执行一次）"""
        datasets.MNIST(self.data_dir, train=True, download=True)
        datasets.MNIST(self.data_dir, train=False, download=True)

    def setup(self, stage: str):
        """设置数据集（每个进程执行）"""
        if stage == "fit" or stage is None:
            mnist_full = datasets.MNIST(
                self.data_dir, train=True, transform=self.transform
            )
            self.mnist_train, self.mnist_val = random_split(
                mnist_full, [55000, 5000]
            )

        if stage == "test" or stage is None:
            self.mnist_test = datasets.MNIST(
                self.data_dir, train=False, transform=self.transform
            )

        if stage == "predict" or stage is None:
            self.mnist_predict = datasets.MNIST(
                self.data_dir, train=False, transform=self.transform
            )

    def train_dataloader(self):
        return DataLoader(
            self.mnist_train,
            batch_size=self.batch_size,
            num_workers=self.num_workers,
            shuffle=True,
            pin_memory=True,  # 加速 GPU 训练
        )

    def val_dataloader(self):
        return DataLoader(
            self.mnist_val,
            batch_size=self.batch_size,
            num_workers=self.num_workers,
            pin_memory=True,
        )

    def test_dataloader(self):
        return DataLoader(
            self.mnist_test,
            batch_size=self.batch_size,
            num_workers=self.num_workers,
            pin_memory=True,
        )

    def predict_dataloader(self):
        return DataLoader(
            self.mnist_predict,
            batch_size=self.batch_size,
            num_workers=self.num_workers,
        )


# 使用 DataModule
if __name__ == "__main__":
    datamodule = MNISTDataModule(batch_size=64)
    model = MNISTModel()

    trainer = Trainer(max_epochs=5)
    trainer.fit(model, datamodule=datamodule)
```

### 2.4 Callbacks：自定义训练流程

Callbacks 允许你在训练的关键点插入自定义逻辑，而无需修改 `LightningModule`。

#### 内置 Callbacks

**1. ModelCheckpoint - 模型检查点**

```python
from pytorch_lightning.callbacks import ModelCheckpoint

checkpoint_callback = ModelCheckpoint(
    dirpath="checkpoints/",
    filename="model-{epoch:02d}-{val_loss:.2f}",
    monitor="val_loss",        # 监控的指标
    save_top_k=3,              # 保存最好的 3 个模型
    mode="min",                # "min" (越小越好) or "max" (越大越好)
    save_last=True,            # 保存最后一个 epoch 的模型
    save_weights_only=False,   # False 保存整个模型，True 只保存权重
    every_n_train_steps=50,    # 每 N 步保存一次
)

trainer = Trainer(callbacks=[checkpoint_callback])
```

**2. EarlyStopping - 早期停止**

```python
from pytorch_lightning.callbacks import EarlyStopping

early_stop_callback = EarlyStopping(
    monitor="val_loss",
    patience=5,               # 容忍 5 个 epoch 不改进
    verbose=True,
    mode="min",
    min_delta=0.001,          # 改进小于此值不算改进
    check_on_train_epoch_end=True,
)

trainer = Trainer(callbacks=[early_stop_callback])
```

**3. LearningRateMonitor - 学习率监控**

```python
from pytorch_lightning.callbacks import LearningRateMonitor

lr_monitor = LearningRateMonitor(
    logging_interval="step",  # "step" or "epoch"
)

trainer = Trainer(callbacks=[lr_monitor])
```

**4. ModelSummary - 模型摘要**

```python
from pytorch_lightning.callbacks import ModelSummary

model_summary = ModelSummary(
    max_depth=3,  # 显示的层数深度
)

trainer = Trainer(callbacks=[model_summary])
```

```python
from pytorch_lightning.callbacks import Callback

class MyCustomCallback(Callback):

    def on_train_end(self, trainer, pl_module):
        print("Training is ending!")

    def on_validation_end(self, trainer, pl_module):
        # 每个 epoch 验证结束后打印一些指标
        print(f"Epoch {trainer.current_epoch} validation ended")
        print(f"Current val_loss: {trainer.callback_metrics['val_loss']}")

    def on_batch_end(self, trainer, pl_module, outputs, batch, batch_idx, dataloader_idx):
        # 每个 batch 结束后执行
        if batch_idx % 100 == 0:
            print(f"Batch {batch_idx} ended")

    def on_exception(self, trainer, pl_module, exception):
        # 训练异常时执行
        print(f"Exception occurred: {exception}")


# 组合多个 Callbacks
trainer = Trainer(
    callbacks=[
        ModelCheckpoint(monitor="val_loss", save_top_k=1),
        EarlyStopping(monitor="val_loss", patience=5),
        LearningRateMonitor(logging_interval="epoch"),
        MyCustomCallback(),
    ]
)
```

## 第三部分：高级特性

### 3.1 多 GPU 训练策略

Lightning 支持多种并行策略，从单机多卡到多节点分布式训练。

#### 策略对比

| 策略 | 全称 | 适用场景 | 通信开销 | 代码改动 |
|------|------|----------|----------|----------|
| **dp** | DataParallel | 单机多卡（小模型） | 高 | 无 |
| **ddp** | DistributedDataParallel | 单机多卡/多节点（推荐） | 中 | 无 |
| **ddp_spawn** | DDP with spawn | Jupyter/Interactive | 中 | 无 |
| **fsdp** | FullyShardedDataParallel | 大模型（分片） | 低 | 无 |
| **deepspeed** | DeepSpeed | 超大模型 | 低 | 需配置 |

#### DataParallel (DP)

```python
# 最简单，但性能较差（不推荐用于生产）
trainer = Trainer(
    accelerator="gpu",
    devices=4,
    strategy="dp",  # DataParallel
)
```

**特点：**
- ✅ 最简单，无需修改代码
- ❌ 单进程多线程，GPU 利用率不均衡
- ❌ 显存占用高（每个 GPU 存储完整模型）
- ❌ 通信开销大

#### DistributedDataParallel (DDP) - **推荐**

```python
# 最佳性能（推荐）
trainer = Trainer(
    accelerator="gpu",
    devices=4,
    strategy="ddp",  # DistributedDataParallel
    num_nodes=1,     # 单节点
)

# 多节点分布式训练
trainer = Trainer(
    accelerator="gpu",
    devices=8,       # 每个节点 8 张卡
    num_nodes=4,     # 4 个节点
    strategy="ddp",
)
```

**启动多节点训练：**

```bash
# 节点 0 (主节点)
python train.py --accelerator gpu --devices 8 --num_nodes 4 --node_rank 0

# 节点 1
python train.py --accelerator gpu --devices 8 --num_nodes 4 --node_rank 1

# 节点 2
python train.py --accelerator gpu --devices 8 --num_nodes 4 --node_rank 2

# 节点 3
python train.py --accelerator gpu --devices 8 --num_nodes 4 --node_rank 3
```

**特点：**
- ✅ 多进程并行，GPU 利用率高
- ✅ 显存占用低（每个 GPU 只存储模型切片）
- ✅ 通信效率高
- ✅ 可扩展到多节点

#### FullyShardedDataParallel (FSDP) - 大模型优化

```python
# 用于超大模型（无法放入单个 GPU）
trainer = Trainer(
    accelerator="gpu",
    devices=8,
    strategy="fsdp",  # FullyShardedDataParallel
    precision="bf16", # 推荐 BF16
)
```

**FSDP 高级配置：**

```python
from pytorch_lightning.strategies import FSDPStrategy

# 自定义 FSDP 配置
strategy = FSDPStrategy(
    sharding_strategy="FULL_SHARD",  # "FULL_SHARD", "SHARD_GRAD_OP", "NO_SHARD"
    cpu_offload=False,               # CPU 卸载
    auto_wrap_policy=...             # 自动包装策略
)

trainer = Trainer(
    accelerator="gpu",
    devices=8,
    strategy=strategy,

```

#### DeepSpeed 集成

```python
from pytorch_lightning.strategies import DeepSpeedStrategy

# DeepSpeed 配置
strategy = DeepSpeedStrategy(
    stage=2,  # ZeRO Stage 0/1/2/3
    offload_optimizer=True,  # CPU 卸载优化器状态
    offload_parameters=False,

trainer = Trainer(
    accelerator="gpu",

```

**ZeRO Stage 说明：**
- **Stage 0**：无分片（等同于 DDP）
- **Stage 1**：优化器状态分片
- **Stage 2**：优化器状态 + 梯度分片
- **Stage 3**：优化器状态 + 梯度 + 参数分片

### 3.2 混合精度训练 (AMP)

混合精度训练使用 16 位浮点数（FP16）或 BFloat16（BF16）进行计算，显著提升训练速度并减少显存占用。

#### FP16 混合精度

```python

    accelerator="gpu",
    devices=1,

    precision=16,
    amp_backend="native",  # "native" (PyTorch AMP) or "apex"

```

#### BF16（推荐，2025+）

```python

    precision="bf16",
    accelerator="gpu",

```

**FP16 vs BF16 对比：**

| 特性 | FP16 | BF16 |
|------|------|------|
| **数值范围** | 有限（需要 Loss Scaling） | 与 FP32 相同 |
| **稳定性** | 需要调参 | 更稳定 |
| **硬件要求** | Volta+ | Ampere+ (A100, RTX 30xx) |
| **推荐场景** | 老款 GPU | 新款 GPU (A100, 3090, 4090) |

### 3.3 梯度操作

#### 梯度累积

梯度累积允许在多次小 batch 计算后再更新参数，模拟大 batch size 训练：

```python
# 基础用法
trainer = Trainer(
    accumulate_grad_batches=4,  # 每 4 个 batch 更新一次参数
)

# 动态累积步数（不同 epoch 使用不同步数）
trainer = Trainer(
    accumulate_grad_batches={
        0: 4,   # 前 5 个 epoch: 累积 4 步
        5: 8,   # 第 5-9 个 epoch: 累积 8 步
        10: 16, # 第 10+ 个 epoch: 累积 16 步
    }
)
```

**有效 Batch Size 计算：**

```python
actual_batch_size = batch_size * accumulate_grad_batches * num_gpus * num_nodes
# 例如: 32 * 4 * 4 * 1 = 512
```

#### 梯度裁剪

```python
# 按范数裁剪
trainer = Trainer(
    gradient_clip_val=1.0,           # 裁剪阈值
    gradient_clip_algorithm="norm",  # "norm" or "value"
)

# 按值裁剪
trainer = Trainer(
    gradient_clip_val=0.5,
    gradient_clip_algorithm="value",  # 每个梯度元素裁剪到 [-0.5, 0.5]
)
```

**在 LightningModule 中自定义：**

```python
class LitModel(pl.LightningModule):
    def configure_gradient_clipping(self, optimizer, optimizer_idx, gradient_clip_val, gradient_clip_algorithm):

            gradient_clip_val=gradient_clip_val,

        )
```

```python
from torchvision import models

class ResNetClassifier(pl.LightningModule):
    def __init__(
        self,
        num_classes: int = 10,
        learning_rate: float = 1e-3,
        pretrained: bool = True,
    ):
        super().__init__()
        self.save_hyperparameters()

        # 加载预训练 ResNet
        self.model = models.resnet50(pretrained=pretrained)

        # 替换最后一层（适配分类数）
        num_features = self.model.fc.in_features
        self.model.fc = nn.Linear(num_features, num_classes)

        # 损失函数
        self.criterion = nn.CrossEntropyLoss()

        # 评估指标
        self.train_accuracy = []
        self.val_accuracy = []
        self.test_accuracy = []

    def forward(self, x):
        return self.model(x)

    def training_step(self, batch, batch_idx):
        x, y = batch
        y_hat = self(x)
        loss = self.criterion(y_hat, y)

        # 计算准确率
        preds = torch.argmax(y_hat, dim=1)
        acc = (preds == y).float().mean()

        # 记录指标
        self.log("train_loss", loss, prog_bar=True, on_step=True, on_epoch=True)
        self.log("train_acc", acc, prog_bar=True, on_step=True, on_epoch=True)

        return loss

    def validation_step(self, batch, batch_idx):
        x, y = batch
        y_hat = self(x)
        loss = self.criterion(y_hat, y)

        preds = torch.argmax(y_hat, dim=1)
        acc = (preds == y).float().mean()

        self.log("val_loss", loss, prog_bar=True)
        self.log("val_acc", acc, prog_bar=True)

        return {"val_loss": loss, "val_acc": acc}

    def test_step(self, batch, batch_idx):
        x, y = batch
        y_hat = self(x)
        loss = self.criterion(y_hat, y)

        preds = torch.argmax(y_hat, dim=1)
        acc = (preds == y).float().mean()

        self.log("test_loss", loss)
        self.log("test_acc", acc)

        return {"test_loss": loss, "test_acc": acc}

    def configure_optimizers(self):
        optimizer = torch.optim.AdamW(
            self.parameters(),
            lr=self.hparams.learning_rate,
            weight_decay=0.01,
        )

        scheduler = torch.optim.lr_scheduler.CosineAnnealingWarmRestarts(
            optimizer,
            T_0=10,
            T_mult=2,
        )

        return {
            "optimizer": optimizer,
            "lr_scheduler": {
                "scheduler": scheduler,
                "interval": "epoch",
                "frequency": 1,
            },
        }

    def on_train_epoch_end(self):
        # 每个 epoch 结束后记录学习率
        opt = self.optimizers()
        current_lr = opt.param_groups[0]["lr"]
        self.log("lr", current_lr)
```

### 4.2 自然语言处理：文本分类（BERT）

```python
import pytorch_lightning as pl
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, Dataset
from transformers import BertModel, BertTokenizer, AdamW

# 自定义数据集
class TextClassificationDataset(Dataset):
    def __init__(self, texts, labels, tokenizer, max_length=128):
        self.texts = texts
        self.labels = labels
        self.tokenizer = tokenizer
        self.max_length = max_length

    def __len__(self):
        return len(self.texts)

    def __getitem__(self, idx):
        text = self.texts[idx]
        label = self.labels[idx]

        encoding = self.tokenizer.encode_plus(
            text,
            add_special_tokens=True,
            max_length=self.max_length,
            return_token_type_ids=False,
            padding="max_length",
            truncation=True,
            return_attention_mask=True,
            return_tensors="pt",
        )

        return {
            "input_ids": encoding["input_ids"].flatten(),
            "attention_mask": encoding["attention_mask"].flatten(),
            "labels": torch.tensor(label, dtype=torch.long),
        }


# BERT 分类器
class BertClassifier(pl.LightningModule):
    def __init__(
        self,
        num_classes: int = 2,
        learning_rate: float = 2e-5,
        model_name: str = "bert-base-uncased",
    ):
        super().__init__()
        self.save_hyperparameters()

        # BERT 模型
        self.bert = BertModel.from_pretrained(model_name)

        # 分类头
        self.classifier = nn.Sequential(
            nn.Dropout(0.1),
            nn.Linear(self.bert.config.hidden_size, num_classes),
        )

        # 损失函数
        self.criterion = nn.CrossEntropyLoss()

    def forward(self, input_ids, attention_mask):
        outputs = self.bert(input_ids=input_ids, attention_mask=attention_mask)
        pooled_output = outputs.pooler_output
        logits = self.classifier(pooled_output)
        return logits

    def training_step(self, batch, batch_idx):
        input_ids = batch["input_ids"]
        attention_mask = batch["attention_mask"]
        labels = batch["labels"]

        logits = self(input_ids, attention_mask)
        loss = self.criterion(logits, labels)

        preds = torch.argmax(logits, dim=1)
        acc = (preds == labels).float().mean()

        self.log("train_loss", loss, prog_bar=True)
        self.log("train_acc", acc, prog_bar=True)

        return loss

    def validation_step(self, batch, batch_idx):
        input_ids = batch["input_ids"]
        attention_mask = batch["attention_mask"]
        labels = batch["labels"]

        logits = self(input_ids, attention_mask)
        loss = self.criterion(logits, labels)

        preds = torch.argmax(logits, dim=1)
        acc = (preds == labels).float().mean()

        self.log("val_loss", loss, prog_bar=True)
        self.log("val_acc", acc, prog_bar=True)

    def test_step(self, batch, batch_idx):
        input_ids = batch["input_ids"]
        attention_mask = batch["attention_mask"]
        labels = batch["labels"]

        logits = self(input_ids, attention_mask)
        loss = self.criterion(logits, labels)

        preds = torch.argmax(logits, dim=1)
        acc = (preds == labels).float().mean()

        self.log("test_loss", loss)
        self.log("test_acc", acc)

    def configure_optimizers(self):
        return AdamW(self.parameters(), lr=self.hparams.learning_rate)


# DataModule
class TextDataModule(pl.LightningDataModule):
    def __init__(
        self,

        train_labels,
        val_texts,
        val_labels,

        self.val_texts = val_texts

        self.tokenizer = BertTokenizer.from_pretrained(model_name)

    def setup(self, stage=None):

    def train_dataloader(self):
```

### 4.3 推荐系统：神经协同过滤

```python
import pytorch_lightning as pl
import torch
import torch.nn as nn
import torch.nn.functional as F

class NeuralCollaborativeFiltering(pl.LightningModule):
    def __init__(
        self,
        num_users: int,
        num_items: int,
        embedding_dim: int = 64,
        hidden_dims: list = [128, 64],
        learning_rate: float = 1e-3,
        dropout: float = 0.2,
    ):
        super().__init__()
        self.save_hyperparameters()

        # 嵌入层
        self.user_embedding = nn.Embedding(num_users, embedding_dim)
        self.item_embedding = nn.Embedding(num_items, embedding_dim)

        # MLP 层
        input_dim = embedding_dim * 2
        layers = []
        prev_dim = input_dim
        for dim in hidden_dims:
            layers.append(nn.Linear(prev_dim, dim))
            layers.append(nn.ReLU())
            layers.append(nn.Dropout(dropout))
            prev_dim = dim
        layers.append(nn.Linear(prev_dim, 1))
        self.mlp = nn.Sequential(*layers)

        # 初始化权重
        self._init_weights()

    def _init_weights(self):
        nn.init.normal_(self.user_embedding.weight, std=0.01)
        nn.init.normal_(self.item_embedding.weight, std=0.01)

    def forward(self, user_ids, item_ids):
        user_embed = self.user_embedding(user_ids)  # [batch, embedding_dim]
        item_embed = self.item_embedding(item_ids)  # [batch, embedding_dim]

        # 拼接嵌入向量
        concat = torch.cat([user_embed, item_embed], dim=1)  # [batch, 2*embedding_dim]

        # MLP 预测
        output = self.mlp(concat)  # [batch, 1]
        return output.squeeze()

    def training_step(self, batch, batch_idx):
        user_ids, item_ids, ratings = batch
        predictions = self(user_ids, item_ids)

        loss = F.mse_loss(predictions, ratings.float())

        self.log("train_loss", loss, prog_bar=True)

        return loss

    def validation_step(self, batch, batch_idx):
        user_ids, item_ids, ratings = batch
        predictions = self(user_ids, item_ids)

        loss = F.mse_loss(predictions, ratings.float())

        # RMSE
        rmse = torch.sqrt(loss)

        self.log("val_loss", loss, prog_bar=True)
        self.log("val_rmse", rmse, prog_bar=True)

    def test_step(self, batch, batch_idx):
        user_ids, item_ids, ratings = batch
        predictions = self(user_ids, item_ids)

        loss = F.mse_loss(predictions, ratings.float())
        rmse = torch.sqrt(loss)

        self.log("test_loss", loss)
        self.log("test_rmse", rmse)

    def configure_optimizers(self):
        optimizer = torch.optim.Adam(
            self.parameters(),
            lr=self.hparams.learning_rate,
        )

        scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(
            optimizer,
            mode="min",
            factor=0.5,
            patience=5,
        )

        return {
            "optimizer": optimizer,
            "lr_scheduler": {
                "scheduler": scheduler,
                "monitor": "val_loss",
            },
        }


# 数据集示例
class MovieLensDataset(torch.utils.data.Dataset):

        self.user_ids = torch.LongTensor(user_ids)
```

### 5.3 常见陷阱和解决方案

#### 陷阱 1：忘记 `self.log()` 中的 `on_epoch=True`

```python
# ❌ 错误
def training_step(self, batch, batch_idx):
    loss = self(batch)
    self.log("train_loss", loss)  # 只记录每步的值
    return loss

# ✅ 正确
def training_step(self, batch, batch_idx):
    loss = self(batch)
    self.log("train_loss", loss, on_step=True, on_epoch=True)  # 同时记录步和 epoch
    return loss
```

```python
# ❌ 错误：在 setup 中下载数据（会被每个进程调用）

# ✅ 正确：在 prepare_data 中下载（只调用一次）
def prepare_data(self):
    datasets.MNIST(".", download=True)  # 只在主进程下载

def setup(self, stage=None):
    self.data = datasets.MNIST(".", download=False)
```

```python

import pytorch_lightning as pl
import torch.nn as nn

class LitModel(pl.LightningModule):
    def __init__(self):
        super().__init__()
        self.layer = nn.Linear(10, 1)

    def forward(self, x):
        return self.layer(x)

    def training_step(self, batch, batch_idx):
        x, y = batch
        y_hat = self(x)
        loss = nn.functional.mse_loss(y_hat, y)
        return loss

    def configure_optimizers(self):
        return torch.optim.Adam(self.parameters(), lr=0.001)

# 训练
model = LitModel()
trainer = pl.Trainer(max_epochs=10)
trainer.fit(model, train_dataloader, val_dataloader)
```

**核心优势回顾：**

1. ✅ **代码简洁**：减少 50%+ 的样板代码
2. ✅ **易于扩展**：一行代码从 CPU 扩展到多 GPU/多节点
3. ✅ **内置最佳实践**：混合精度、梯度累积、自动检查点等
4. ✅ **生态系统完善**：与 W&B、TensorBoard、MLflow 无缝集成
5. ✅ **活跃的社区**：持续更新和改进（2025 已发布 2.5+ 版本）
