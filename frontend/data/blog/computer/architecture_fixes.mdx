# Rust 博客架构方案关键修正清单

## P0 - 会直接导致上线失败的问题

### 1. 修正分区表主键约束

**问题**：分区表的唯一约束必须包含分区键，否则无法保证跨分区唯一性。

**修正方案**：

```sql
-- 用户活动表（修正版）
CREATE TABLE user_activities (
    id UUID NOT NULL DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id),
    action VARCHAR(50) NOT NULL,
    resource_type VARCHAR(50),
    resource_id UUID,
    metadata JSONB DEFAULT '{}',
    ip_address INET,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
) PARTITION BY RANGE (created_at);

-- 父表不强制要求 PK，只在分区内建索引
CREATE INDEX idx_user_activities_created_at ON user_activities(created_at);
CREATE INDEX idx_user_activities_user_created ON user_activities(user_id, created_at DESC);

-- 创建分区并添加默认分区防止写入失败
CREATE TABLE user_activities_2024_01 PARTITION OF user_activities
    FOR VALUES FROM ('2024-01-01') TO ('2024-02-01');

CREATE TABLE user_activities_default PARTITION OF user_activities
    DEFAULT PARTITION OF user_activities;
```

### 2. 修正 Ingress 限流注解

**问题**：`rate-limit` 不是 ingress-nginx 标准注解。

**修正方案**：


### 4. 修正 Axum 路由模式获取

**问题**：使用 `axum::routing::Route` 不是稳定的方式。

**修正方案**：

```rust
// crates/shared/src/middleware/metrics.rs
use axum::extract::MatchedPath;

pub async fn metrics_middleware<B>(
    request: Request<B>,
    next: Next<B>,
) -> Response {
    let start = Instant::now();
    let method = request.method().to_string();

    // 正确获取路由模式
    let route = request
        .extensions()
        .get::<MatchedPath>()
        .map(|m| m.as_str())
        .unwrap_or("unknown");

    let response = next.run(request).await;
    let status = response.status().as_u16();
    let duration = start.elapsed().as_secs_f64();

    // 记录指标
    HTTP_REQUESTS_TOTAL
        .with_label_values(&[&method, route, &status.to_string()])
        .inc();

    HTTP_REQUEST_DURATION
        .with_label_values(&[&method, route])
        .observe(duration);

    response
}
```

### 5. 修正时间戳获取

**问题**：`Instant` 不能和 `UNIX_EPOCH` 做计算。

**修正方案**：

```rust
use std::time::SystemTime;

// 修正前
let now = Instant::now()
    .duration_since(std::time::UNIX_EPOCH)?;

// 修正后
let now = SystemTime::now()
    .duration_since(SystemTime::UNIX_EPOCH)?
    .as_secs();
```

### 10. 避免索引重复

**迁移策略**：

- 开发环境：只创建基础索引，跳过 CONCURRENTLY 迁移
- 生产环境：将并发索引创建放到独立的运维脚本中

```sql
-- migrations/002_create_indexes.sql (开发环境)
CREATE INDEX idx_users_profile_gin ON users USING GIN (profile);
CREATE INDEX idx_post_stats_updated ON post_stats(updated_at DESC);

-- scripts/prod_indexes.sql (生产环境，单独执行）
-- no-transaction
CREATE INDEX CONCURRENTLY idx_users_profile_gin ON users USING GIN (profile);
CREATE INDEX CONCURRENTLY idx_post_stats_updated ON post_stats(updated_at DESC);
```

### 12. 优化真实 IP 获取

**修正方案**：

```rust
fn extract_real_ip(request: &Request<Body>) -> String {
    // Cloudflare 优先
    if let Some(ip) = request.headers().get("cf-connecting-ip") {
        return ip.to_str().unwrap_or("unknown").to_string();
    }

    // 其次 x-real-ip
    if let Some(ip) = request.headers().get("x-real-ip") {
        return ip.to_str().unwrap_or("unknown").to_string();
    }

    // 最后 x-forwarded-for (取第一个)
    if let Some(xff) = request.headers().get("x-forwarded-for") {
        return xff.to_str()
            .ok()
            .and_then(|s| s.split(',').next())
            .map(|s| s.trim().to_string())
            .unwrap_or("unknown");
    }

    "unknown".to_string()
}
```
