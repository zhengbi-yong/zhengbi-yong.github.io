## 引言

Clash Party（原名 Mihomo Party）是一款基于 Mihomo（Clash Meta）内核的跨平台网络代理客户端，支持 Windows、macOS 和 Linux 系统。作为 Clash 生态中的重要一员，Clash Party 不仅继承了 Clash 的强大功能，还在此基础上引入了智能规则覆写、双内核切换、WebDAV 备份等创新特性，为用户提供了更加智能、便捷的代理管理体验。

与传统的代理客户端不同，Clash Party 采用了现代化的设计理念，提供了直观的图形用户界面，同时保持了高度的可定制性。无论是初次接触代理工具的新手，还是需要精细控制的高级用户，都能在 Clash Party 中找到适合自己的使用方式。其内置的 Smart Core 规则覆写功能，基于 AI 模型自动选择最优节点，大大简化了节点选择的复杂性；双内核系统（Smart 内核与 Mihomo 原生内核）的灵活切换，让用户可以根据不同场景选择最适合的内核；WebDAV 备份功能的引入，使得配置在不同设备间的同步变得轻而易举。

Clash Party 支持多种主流代理协议，包括 Shadowsocks（SS）、ShadowsocksR（SSR）、Trojan、V2Ray（VMess/VLESS）、Hysteria、Reality、TUIC、WireGuard 等，几乎涵盖了当前所有主流的代理协议。这种广泛的协议支持，使得 Clash Party 能够兼容各种代理服务提供商，为用户提供了最大的选择灵活性。

本文档旨在提供一份全面而实用的 Clash Party 使用指南，从基础的安装配置到高级的功能定制，从快速入门到故障排除，帮助您系统性地学习和掌握 Clash Party 的强大功能。无论您是刚开始接触代理工具的新手，还是希望深入理解 Clash Party 高级特性的资深用户，都能从本文中找到有价值的内容。

- **循序渐进**：先掌握基础的安装和配置，再逐步学习高级功能
- **实践为主**：配置和优化的技巧需要通过实际使用来巩固，建议在日常使用中不断尝试和调整

## 第一部分：基础概念与安装

理解 Clash Party 的核心概念和架构设计是高效使用的基础。在开始使用之前，我们需要了解 Clash Party 的技术背景、系统要求以及安装方法。

### Clash Party 概述

**什么是 Clash Party**

Clash Party 是一款基于 Mihomo（Clash Meta）内核开发的跨平台网络代理客户端。Mihomo 是 Clash 的一个分支项目，在保持 Clash 核心功能的基础上，增加了对新协议的支持和性能优化。Clash Party 作为 Mihomo 的图形化客户端，将命令行工具的强大功能封装在友好的用户界面中，使得普通用户也能轻松使用。

**技术架构**

Clash Party 采用客户端-内核分离的架构设计：

- **图形界面层**：负责用户交互、配置管理、状态显示等
- **Mihomo 内核**：负责实际的网络代理处理、规则匹配、流量转发等核心功能
- **Smart 内核**：Clash Party 特有的智能内核，提供 AI 模型驱动的节点选择功能

这种架构设计使得 Clash Party 既保持了核心功能的稳定性，又能够灵活地扩展用户界面功能。

**与其他 Clash 客户端的区别**

Clash 生态中有多个客户端，每个都有其特色：

| 客户端            | 特点                          | 适用场景                     |
| ----------------- | ----------------------------- | ---------------------------- |
| Clash for Windows | 最早的图形化客户端，功能成熟  | Windows 用户，需要稳定可靠   |
| Clash Verge       | 现代化界面，功能丰富          | 跨平台用户，追求美观界面     |
| Clash Party       | 智能规则、双内核、WebDAV 备份 | 需要智能优化和配置同步的用户 |
| ClashX            | macOS 专用，系统集成度高      | macOS 用户，需要系统级集成   |

Clash Party 的主要优势在于其智能化的节点选择、灵活的配置管理和便捷的备份同步功能。

### 系统要求与平台支持

**Windows 系统**

- **最低要求**：Windows 7 或更高版本
- **架构支持**：32 位（x86）和 64 位（x64）
- **安装方式**：安装版（.exe）或便携版（绿色免安装）
- **推荐配置**：Windows 10/11，64 位系统

**macOS 系统**

- **最低要求**：macOS 11.0（Big Sur）或更高版本
- **架构支持**：
  - Intel 芯片（x64）
  - Apple Silicon（M1/M2/M3 等，ARM64）
- **安装方式**：.dmg 安装包
- **推荐配置**：macOS 12 或更高版本

**Linux 系统**

- **发行版支持**：
  - Debian/Ubuntu（.deb 包）
  - RedHat/CentOS/Fedora（.rpm 包）
- **架构支持**：
  - AMD64（64 位）
  - ARM64
- **依赖要求**：现代 Linux 发行版，通常无需额外依赖

### 下载与安装

**官方下载渠道**

Clash Party 的官方发布地址是 GitHub Releases 页面。为确保安全，请务必从官方渠道下载：

```
https://github.com/mihomo-party-org/clash-party/releases
```

**Windows 安装步骤**

1. **下载安装包**
   - 访问 GitHub Releases 页面
   - 下载适合您系统的版本：
     - `Clash.Party.Setup.x.x.x.exe`（安装版）
     - `Clash.Party.x.x.x-win-x64.zip`（便携版）

2. **安装版安装**
   - 双击 `.exe` 文件启动安装程序
   - 按照向导提示完成安装
   - 安装完成后，Clash Party 会出现在开始菜单中

3. **便携版使用**
   - 解压下载的 `.zip` 文件到任意目录
   - 双击 `Clash.Party.exe` 即可运行
   - 便携版的所有配置都保存在程序目录中

**macOS 安装步骤**

1. **下载安装包**
   - 访问 GitHub Releases 页面
   - 根据您的 Mac 芯片类型选择：
     - Intel 芯片：`Clash.Party-x.x.x-macos-x64.dmg`
     - Apple Silicon：`Clash.Party-x.x.x-macos-arm64.dmg`

2. **安装应用**
   - 双击下载的 `.dmg` 文件
   - 将 Clash Party 拖拽到 Applications 文件夹
   - 首次运行时，如果遇到"无法打开"提示：
     - 打开"系统设置" > "隐私与安全性"
     - 点击"仍要打开"允许运行

**Linux 安装步骤**

1. **Debian/Ubuntu 系统**

   ```bash
   # 下载 .deb 包
   wget https://github.com/mihomo-party-org/clash-party/releases/download/vx.x.x/clash-party_x.x.x_amd64.deb

   # 安装
   sudo dpkg -i clash-party_x.x.x_amd64.deb

   # 如果遇到依赖问题，运行
   sudo apt-get install -f
   ```

2. **RedHat/CentOS/Fedora 系统**

   ```bash
   # 下载 .rpm 包
   wget https://github.com/mihomo-party-org/clash-party/releases/download/vx.x.x/clash-party-x.x.x-x86_64.rpm

   # 安装
   sudo rpm -i clash-party-x.x.x-x86_64.rpm
   ```

3. **启动应用**
   - 在应用程序菜单中找到 Clash Party
   - 或通过命令行启动：`clash-party`

**安装后验证**

安装完成后，首次启动 Clash Party 应该能看到主界面。如果遇到问题：

- **Windows**：检查防火墙设置，确保 Clash Party 有网络访问权限
- **macOS**：检查系统隐私设置，允许 Clash Party 访问网络
- **Linux**：检查是否有必要的系统权限，某些功能可能需要 root 权限

## 第二部分：快速入门

完成安装后，让我们快速上手 Clash Party，掌握基本的配置和使用方法。

### 首次启动与界面介绍

**主界面布局**

Clash Party 的主界面通常包含以下区域：

- **顶部菜单栏**：文件、编辑、视图、帮助等菜单
- **左侧边栏**：订阅列表、节点列表、规则配置等导航
- **主内容区**：显示当前选中的功能模块
- **底部状态栏**：显示连接状态、延迟、流量统计等信息

**主要功能区域**

| 功能区域 | 说明                             |
| -------- | -------------------------------- |
| 订阅管理 | 添加、更新、删除订阅链接         |
| 节点列表 | 查看和管理所有代理节点           |
| 代理模式 | 选择规则模式、全局模式或直连模式 |
| 系统代理 | 开启/关闭系统代理设置            |
| 配置管理 | 编辑配置文件、导入导出配置       |
| 日志查看 | 查看运行日志和连接信息           |

### 订阅管理

**导入订阅链接**

1. **获取订阅链接**
   - 从代理服务提供商（机场）获取订阅地址
   - 订阅链接通常是一个 URL，格式如：`https://example.com/subscribe?token=xxx`

2. **添加订阅**
   - 点击"订阅"或"Subscriptions"菜单
   - 点击"添加订阅"或"+"按钮
   - 输入订阅名称（自定义，如"主订阅"）
   - 粘贴订阅链接
   - 点击"确定"或"添加"

3. **更新订阅**
   - 在订阅列表中，点击订阅项右侧的"更新"按钮
   - 或右键点击订阅，选择"更新订阅"
   - Clash Party 会自动从订阅链接获取最新的节点信息

**订阅设置**

- **自动更新**：可以设置订阅自动更新的间隔（如每 6 小时、每天等）
- **多订阅管理**：支持添加多个订阅，节点会自动合并显示
- **订阅重命名**：可以自定义订阅名称，方便识别

**订阅更新失败处理**

如果订阅更新失败，可能的原因：

- 订阅链接已过期或无效
- 网络连接问题
- 订阅服务器暂时不可用

解决方法：

- 检查订阅链接是否正确
- 检查网络连接
- 稍后重试
- 联系服务提供商确认订阅状态

### 节点选择与测试

**节点列表查看**

导入订阅后，所有节点会显示在节点列表中。节点信息通常包括：

- **节点名称**：服务商提供的节点标识
- **延迟**：节点响应时间（ping 值）
- **协议类型**：SS、Trojan、VMess 等
- **地区**：节点所在地区或国家

**延迟测试**

1. **自动测试**
   - Clash Party 会自动测试所有节点的延迟
   - 延迟值会实时显示在节点列表中
   - 延迟越低，通常连接速度越快

2. **手动测试**
   - 右键点击节点，选择"测试延迟"
   - 或点击节点列表上方的"测试所有节点"按钮

3. **测试结果解读**
   - `< 100ms`：优秀，适合日常使用
   - `100-200ms`：良好，可以正常使用
   - `200-500ms`：一般，可能影响体验
   - `> 500ms`：较差，不推荐使用

**节点选择策略**

- **日常使用**：选择延迟最低的节点
- **特定地区**：如果需要访问特定地区的内容，选择对应地区的节点
- **负载均衡**：某些服务商提供负载均衡节点，可以自动分配流量
- **备用节点**：建议选择 2-3 个备用节点，以防主节点故障

### 代理模式设置

Clash Party 提供三种主要的代理模式，每种模式适用于不同的使用场景。

**规则模式（Rule）**

规则模式是 Clash Party 的默认模式，也是最常用的模式。

- **工作原理**：根据配置的规则，自动判断哪些流量走代理，哪些直连
- **典型规则**：
  - 国内网站直连
  - 国外网站走代理
  - 特定域名或 IP 走指定节点
- **适用场景**：日常使用，兼顾速度和功能
- **优点**：智能分流，节省流量，速度快
- **缺点**：需要维护规则列表

**全局模式（Global）**

全局模式下，所有网络流量都通过代理转发。

- **工作原理**：所有流量都走选中的代理节点
- **适用场景**：
  - 需要访问所有网站都通过代理
  - 测试代理节点是否正常工作
  - 某些特殊网络环境
- **优点**：配置简单，所有流量都加密
- **缺点**：国内网站也走代理，可能影响速度

**直连模式（Direct）**

直连模式下，所有流量都不经过代理。

- **工作原理**：所有流量直接连接，不使用代理
- **适用场景**：
  - 临时关闭代理
  - 测试网络连接
  - 某些不需要代理的场景
- **优点**：速度最快，无代理延迟
- **缺点**：无法访问被限制的内容

**TUN 模式**

TUN 模式是 Clash Party 的高级功能，提供系统级的代理。

- **工作原理**：在系统层面创建虚拟网络接口，所有流量自动通过代理
- **特点**：
  - 无需配置系统代理
  - 支持 UDP 流量代理
  - 更彻底的流量控制
- **适用场景**：
  - 需要代理所有应用（包括不支持系统代理的应用）
  - 游戏、视频通话等需要 UDP 代理的场景
- **注意事项**：
  - 某些系统可能需要管理员权限
  - 可能影响某些网络功能

**模式切换**

在 Clash Party 界面中，通常有一个模式选择器，可以快速切换：

- 点击模式选择器
- 选择"规则"、"全局"或"直连"
- TUN 模式通常有独立的开关

### 系统代理设置

**开启系统代理**

启用系统代理后，系统级的网络请求会通过 Clash Party 转发。

1. **Windows/macOS**
   - 在 Clash Party 界面中找到"系统代理"开关
   - 点击开启系统代理
   - 系统会自动配置代理设置

2. **Linux**
   - 某些 Linux 发行版可能需要手动配置
   - Clash Party 会显示代理地址和端口
   - 在系统网络设置中配置 HTTP/HTTPS/SOCKS5 代理

**代理地址和端口**

Clash Party 默认的代理地址和端口：

- **HTTP 代理**：`127.0.0.1:7890`
- **HTTPS 代理**：`127.0.0.1:7890`
- **SOCKS5 代理**：`127.0.0.1:7891`

这些地址和端口可以在设置中修改。

**验证代理是否生效**

1. **检查 Clash Party 状态**
   - 查看界面中的连接状态指示
   - 检查是否有流量统计

2. **访问测试网站**
   - 访问 `https://www.google.com` 等被限制的网站
   - 如果能够访问，说明代理正常工作

3. **查看 IP 地址**
   - 访问 `https://ip.sb` 或 `https://ipinfo.io`
   - 查看显示的 IP 地址是否与代理节点地区一致

## 第三部分：核心功能详解

掌握了基础操作后，让我们深入了解 Clash Party 的核心功能，这些功能是 Clash Party 区别于其他客户端的重要特性。

### 双内核系统

Clash Party 内置了两个内核：Smart 内核和 Mihomo（Clash Meta）原生内核。用户可以根据需求自由切换。

**Smart 内核**

Smart 内核是 Clash Party 的特色功能，提供了智能化的节点选择和管理。

- **AI 模型驱动**：基于 AI 模型自动选择最优节点
- **智能规则覆写**：一键应用 Smart Core 规则，自动优化分流规则
- **自适应优化**：根据网络状况自动调整节点选择策略
- **适用场景**：
  - 希望自动化节点选择
  - 需要智能化的规则管理
  - 追求最佳的网络体验

**Mihomo 原生内核**

Mihomo 内核是 Clash Meta 的官方内核，提供了稳定可靠的代理功能。

- **协议支持全面**：支持所有主流代理协议
- **性能稳定**：经过长期验证，稳定性高
- **配置灵活**：支持完整的 Clash 配置语法
- **适用场景**：
  - 需要精确控制配置
  - 使用特殊协议或功能
  - 追求稳定性和兼容性

**内核切换方法**

1. **在设置中切换**
   - 打开 Clash Party 设置
   - 找到"内核"或"Core"选项
   - 选择"Smart 内核"或"Mihomo 内核"
   - 重启 Clash Party 使设置生效

2. **切换注意事项**
   - 切换内核后，某些配置可能需要调整
   - Smart 内核的智能功能在 Mihomo 内核中不可用
   - 建议根据实际需求选择合适的内核

### 智能规则覆写

Smart Core 规则覆写是 Clash Party 的亮点功能之一，可以一键应用优化的分流规则。

**Smart Core 规则简介**

Smart Core 规则是一套经过优化的分流规则集，包含：

- **国内直连规则**：识别国内网站，自动直连
- **国外代理规则**：识别国外网站，自动走代理
- **广告拦截规则**：拦截常见广告域名
- **隐私保护规则**：阻止追踪和恶意域名

**启用 Smart Core 规则**

1. **在设置中启用**
   - 打开 Clash Party 设置
   - 找到"规则"或"Rules"选项
   - 启用"Smart Core 规则覆写"
   - 选择规则更新源（通常使用默认源）

2. **规则自动更新**
   - Smart Core 规则会自动定期更新
   - 确保规则列表始终是最新的
   - 可以在设置中配置更新频率

**自定义规则**

虽然 Smart Core 规则已经覆盖了大部分场景，但您仍然可以添加自定义规则：

```yaml
rules:
  # 自定义规则示例
  - DOMAIN-SUFFIX,example.com,DIRECT
  - DOMAIN-KEYWORD,test,PROXY
  - IP-CIDR,192.168.1.0/24,DIRECT
```

**规则优先级**

规则按照配置顺序匹配，先匹配的规则优先生效：

1. 自定义规则（最优先）
2. Smart Core 规则
3. 默认规则（最后匹配）

### 配置管理

Clash Party 的配置管理功能强大而灵活，支持多种配置方式。

**配置文件结构**

Clash Party 使用 YAML 格式的配置文件，主要包含以下部分：

```yaml
# 端口配置
port: 7890
socks-port: 7891

# 允许局域网连接
allow-lan: false

# 规则模式
mode: rule

# 日志级别
log-level: info

# 外部控制器
external-controller: 127.0.0.1:9090

# DNS 配置
dns:
  enable: true
  listen: 0.0.0.0:53
  default-nameserver:
    - 223.5.5.5
    - 8.8.8.8

# 代理节点
proxies:
  - name: '节点名称'
    type: ss
    server: example.com
    port: 443
    cipher: aes-256-gcm
    password: '密码'

# 代理组
proxy-groups:
  - name: '自动选择'
    type: select
    proxies:
      - '节点名称'

# 规则
rules:
  - DOMAIN-SUFFIX,google.com,PROXY
  - GEOIP,CN,DIRECT
  - MATCH,PROXY
```

**配置导入导出**

## 第四部分：高级配置

对于需要精细控制的用户，Clash Party 提供了丰富的高级配置选项。

### 代理协议详解

Clash Party 支持多种代理协议，每种协议都有其特点和适用场景。

**Shadowsocks (SS)**

Shadowsocks 是最早的代理协议之一，简单高效。

- **特点**：轻量级，性能好，兼容性强
- **加密方式**：AES-256-GCM、ChaCha20-Poly1305 等
- **适用场景**：日常使用，对速度要求高
- **配置示例**：
  ```yaml
  - name: 'SS节点'
    type: ss
    server: example.com
    port: 443
    cipher: aes-256-gcm
    password: '密码'
  ```

**ShadowsocksR (SSR)**

SSR 是 Shadowsocks 的增强版，增加了混淆功能。

- **特点**：支持协议和混淆插件，更难被检测
- **适用场景**：网络环境严格，需要混淆
- **配置示例**：
  ```yaml
  - name: 'SSR节点'
    type: ssr
    server: example.com
    port: 443
    cipher: aes-256-cfb
    password: '密码'
    protocol: origin
    obfs: plain
  ```

**Trojan**

Trojan 使用 TLS 加密，伪装成正常的 HTTPS 流量。

- **特点**：安全性高，难以被识别，性能好
- **适用场景**：对安全性要求高，需要伪装流量
- **配置示例**：
  ```yaml
  - name: 'Trojan节点'
    type: trojan
    server: example.com
    port: 443
    password: '密码'
    sni: example.com
  ```

**V2Ray (VMess/VLESS)**

V2Ray 是功能强大的代理协议，支持多种传输方式。

- **VMess**：V2Ray 的经典协议
- **VLESS**：VMess 的简化版，性能更好
- **特点**：功能丰富，支持多种传输方式（TCP、WebSocket、HTTP/2 等）
- **适用场景**：需要复杂配置，对功能要求高
- **配置示例**：
  ```yaml
  - name: 'VMess节点'
    type: vmess
    server: example.com
    port: 443
    uuid: 'uuid字符串'
    alterId: 0
    cipher: auto
    network: ws
    ws-path: /path
    ws-headers:
      Host: example.com
  ```

**Hysteria/Hysteria2**

Hysteria 是基于 QUIC 协议的高性能代理。

- **特点**：速度极快，抗丢包能力强
- **适用场景**：对速度要求极高，网络质量一般
- **配置示例**：
  ```yaml
  - name: 'Hysteria节点'
    type: hysteria
    server: example.com
    port: 443
    auth: '认证字符串'
    protocol: udp
  ```

**Reality**

Reality 是 V2Ray 的新协议，提供更强的伪装能力。

- **特点**：使用真实网站进行伪装，难以被检测
- **适用场景**：网络环境非常严格
- **配置示例**：
  ```yaml
  - name: 'Reality节点'
    type: vless
    server: example.com
    port: 443
    uuid: 'uuid字符串'
    network: tcp
    tls: true
    reality-opts:
      public-key: '公钥'
      short-id: '短ID'
  ```

**TUIC**

TUIC 是基于 QUIC 的新协议，性能优秀。

- **特点**：速度快，延迟低
- **适用场景**：对性能要求高
- **配置示例**：
  ```yaml
  - name: 'TUIC节点'
    type: tuic
    server: example.com
    port: 443
    token: 'token字符串'
    uuid: 'uuid字符串'
  ```

**WireGuard**

WireGuard 是现代的 VPN 协议。

- **特点**：安全性高，性能好，配置简单
- **适用场景**：需要 VPN 级别的安全保护
- **配置示例**：
  ```yaml
  - name: 'WireGuard节点'
    type: wireguard
    server: example.com
    port: 51820
    ip: '10.0.0.2/24'
    private-key: '私钥'
    public-key: '公钥'
    pre-shared-key: '预共享密钥'
  ```

### 规则配置

规则配置是 Clash Party 的核心功能之一，决定了流量的分流策略。

**规则类型**

Clash 支持多种规则类型：

| 规则类型       | 说明           | 示例                             |
| -------------- | -------------- | -------------------------------- |
| DOMAIN         | 完整域名匹配   | `DOMAIN,example.com,PROXY`       |
| DOMAIN-SUFFIX  | 域名后缀匹配   | `DOMAIN-SUFFIX,google.com,PROXY` |
| DOMAIN-KEYWORD | 域名关键词匹配 | `DOMAIN-KEYWORD,google,PROXY`    |
| IP-CIDR        | IP 段匹配      | `IP-CIDR,192.168.1.0/24,DIRECT`  |
| GEOIP          | 地理位置匹配   | `GEOIP,CN,DIRECT`                |
| MATCH          | 默认规则       | `MATCH,PROXY`                    |

**常用规则示例**

```yaml
rules:
  # 直连国内域名
  - DOMAIN-SUFFIX,cn,DIRECT
  - DOMAIN-SUFFIX,com.cn,DIRECT

  # 代理 Google 服务
  - DOMAIN-SUFFIX,google.com,PROXY
  - DOMAIN-SUFFIX,googleapis.com,PROXY
  - DOMAIN-SUFFIX,gstatic.com,PROXY

  # 代理社交媒体
  - DOMAIN-SUFFIX,twitter.com,PROXY
  - DOMAIN-SUFFIX,facebook.com,PROXY
  - DOMAIN-SUFFIX,instagram.com,PROXY

  # 直连国内 IP
  - GEOIP,CN,DIRECT

  # 其他流量走代理
  - MATCH,PROXY
```

**规则优先级**

规则按照配置顺序从上到下匹配，先匹配的规则优先生效：

1. 具体规则（DOMAIN、IP-CIDR 等）
2. 通用规则（DOMAIN-SUFFIX、DOMAIN-KEYWORD 等）
3. 地理位置规则（GEOIP）
4. 默认规则（MATCH）

**代理组配置**

代理组用于组织和管理节点：

```yaml
proxy-groups:
  # 手动选择组
  - name: '手动选择'
    type: select
    proxies:
      - '节点1'
      - '节点2'
      - '自动选择'
      - DIRECT

  # 自动选择组（选择延迟最低的节点）
  - name: '自动选择'
    type: url-test
    proxies:
      - '节点1'
      - '节点2'
    url: 'http://www.gstatic.com/generate_204'
    interval: 300

  # 负载均衡组（轮询使用节点）
  - name: '负载均衡'
    type: load-balance
    proxies:
      - '节点1'
      - '节点2'
    url: 'http://www.gstatic.com/generate_204'
    interval: 300
```

### 性能优化

合理的配置可以显著提升 Clash Party 的性能和稳定性。

**节点选择优化**

- **延迟测试**：定期测试节点延迟，选择最优节点
- **自动选择**：使用自动选择组，自动切换到最快节点
- **节点分组**：按地区或用途分组，便于管理

**连接数设置**

```yaml
# 限制并发连接数
tcp-concurrent: true

# 每个代理的最大连接数
max-connections: 100
```

**DNS 优化**

```yaml
dns:
  enable: true
  listen: 0.0.0.0:53
  enhanced-mode: fake-ip
  fake-ip-range: 198.18.0.1/16
  nameserver:
    - 223.5.5.5
    - 8.8.8.8
  fallback:
    - 1.1.1.1
    - 8.8.4.4
  fallback-filter:
    geoip: true
    geoip-code: CN
    ipcidr:
      - 240.0.0.0/4
```

**延迟优化技巧**

- **使用 Hysteria/TUIC**：这些协议在延迟优化方面表现优秀
- **选择地理位置近的节点**：物理距离越近，延迟越低
- **避免高峰期使用**：网络高峰期可能影响延迟
- **使用 TUN 模式**：TUN 模式可以减少系统代理的开销

在使用 Clash Party 的过程中，可能会遇到各种问题。本节将介绍常见问题的解决方法。

### 常见问题

**无法连接代理**

可能的原因和解决方法：

1. **节点不可用**
   - 检查节点是否在线
   - 测试节点延迟
   - 尝试切换其他节点

2. **订阅过期**
   - 更新订阅链接
   - 检查订阅是否有效
   - 联系服务提供商

3. **配置错误**
   - 检查配置文件语法
   - 验证节点配置是否正确
   - 查看日志文件

4. **网络问题**
   - 检查本地网络连接
   - 检查防火墙设置
   - 尝试重启 Clash Party

**连接速度慢**

优化建议：

1. **选择更快的节点**
   - 测试所有节点延迟
   - 选择延迟最低的节点
   - 使用自动选择功能

2. **优化协议**
   - 尝试使用 Hysteria 或 TUIC 协议
   - 避免使用过时的协议

3. **调整配置**
   - 减少并发连接数
   - 优化 DNS 设置
   - 使用 TUN 模式

4. **网络环境**
   - 检查本地网络质量
   - 避免在网络高峰期使用
   - 尝试更换网络环境

**订阅更新失败**

解决方法：

1. **检查订阅链接**
   - 验证链接是否正确
   - 检查链接是否过期
   - 尝试在浏览器中访问链接

2. **网络问题**
   - 检查网络连接
   - 尝试使用代理访问订阅链接
   - 检查 DNS 设置

3. **服务商问题**
   - 联系服务提供商
   - 检查服务商公告
   - 尝试稍后重试

**系统代理不生效**

排查步骤：

1. **检查 Clash Party 状态**
   - 确认 Clash Party 正在运行
   - 检查代理端口是否正确
   - 查看日志是否有错误

2. **系统设置**
   - Windows：检查系统代理设置
   - macOS：检查网络设置中的代理配置
   - Linux：检查环境变量和系统设置

3. **应用程序设置**
   - 某些应用可能需要单独配置代理
   - 检查应用的网络设置
   - 尝试使用 TUN 模式
