## 引言

开源电机控制器在过去十年中经历了飞速发展，从简单的电动滑板控制器演变为能够驱动复杂机器人系统的高性能伺服驱动平台。VESC 和 ODrive 作为这一领域的两个标杆项目，为开源电机控制技术树立了标准，推动了整个行业的发展。

然而，2024-2025 年见证了这一领域的重大变化。ODrive ——曾经以完全开源和双电机控制著称的项目——在其新一代产品中转向了闭源模式，仅提供预编译的固件二进制文件。这一转变不仅引发了社区的热烈讨论，更让人们开始寻找和关注其他完全开源的替代方案。

与此同时，一批新的开源电机控制器项目正在崛起。moteus 专为四足机器人设计的高性能伺服控制器、SimpleFOC 以易用性著称的 Arduino FOC 库、MESC 作为 VESC 的低成本替代方案，以及基于 FPGA 的高性能 FOC 实现等，都在各自的细分领域展现出独特的优势。

本文将全面深入地对比分析这些开源电机驱动器，从技术规格、应用场景、开源生态系统到成本可获得性，帮助研究者和 DIY 爱好者做出明智的选择。无论您是在构建四足机器人、电动车辆，还是进行学术研究，都能从本文中找到最适合您需求的电机控制解决方案。

## 第一部分：开源电机控制器格局变化

### VESC 与 ODrive 的历史地位

**VESC 的诞生与影响**

VESC（Vedder Electronic Speed Controller）由瑞典工程师 Benjamin Vedder 于 2015 年创建，最初的目标是为电动滑板提供高性能、低成本且完全开源的电机控制器。VESC 的出现彻底改变了电动车辆 DIY 领域：

- **技术创新**：率先在开源控制器中实现 FOC（磁场定向控制）算法
- **开源理念**：硬件和固件完全开源，可自由定制和生产
- **社区驱动**：建立了庞大的全球开发者社区
- **生态完整**：提供了 VESC Tool 图形化配置工具和丰富的通信接口

VESC 的成功证明了开源项目可以达到商业产品的性能水平，甚至更胜一筹。

**ODrive 的突破**

ODrive 项目由 ODrive Robotics 公司开发，专注于高精度机器人应用。其核心创新包括：

- **双电机控制**：单板控制两个电机，降低系统复杂度
- **高精度位置控制**：专为机器人关节设计
- **即插即用**：相比 VESC 更少的调参需求
- **闭环性能**：优秀的速度和位置闭环控制

ODrive 在机器人研究社区，特别是四足机器人和机械臂领域获得了广泛应用。

### 开源生态的新变化

**ODrive 开源状态转变**

2024-2025 年，ODrive 项目发生了重要的战略转变：

**ODrive v3.x（旧版本）**
- ✅ 开源代码可在 GitHub 获取
- ⚠️ NRND（Not Recommended for New Design，不推荐用于新设计）
- ⚠️ 不再积极维护
- ⚠️ 技术逐渐落后于新一代产品

**ODrive Pro / S1 / Micro（新一代）**
- ❌ 固件源代码不公开（闭源）
- ✅ 积极维护和开发
- 🔧 仅提供预编译的固件二进制文件
- ⚡ 性能和功能显著增强

这一转变在社区中引发了广泛讨论，Reddit 等平台上有用户表达了对"ODrive 转向闭源"的失望。这一变化标志着开源电机控制器领域进入了一个新的阶段——混合模式的出现，即开源项目在商业化过程中可能转向部分闭源。

**社区反应与影响**

ODrive 的闭源转变对开源社区产生了深远影响：

1. **信任问题**：用户担心开源项目可能随时转向闭源
2. **替代方案寻求**：社区开始积极寻找完全开源的替代方案
3. **项目多样性**：催生了更多开源电机控制器项目
4. **生态分化**：完全开源与部分开源项目形成不同的生态系统

### 新兴替代方案的崛起

正是在这样的背景下，多个新的开源电机控制器项目获得了关注：

1. **moteus**：专为四足机器人设计的专业级解决方案，完全开源
2. **SimpleFOC**：以易用性著称的 Arduino FOC 库，降低入门门槛
3. **MESC**：VESC 的低成本、易学习替代方案
4. **FPGA-FOC**：基于 FPGA 的超高性能实现
5. **MP2-ESC**：模块化多平台解决方案

这些项目各有特色，在不同的应用场景中展现出独特的优势。接下来的章节将深入分析这些项目的技术特点、适用场景和选择建议。

## 第二部分：ODrive 深度分析

### 项目历史与成就

**发展历程**

ODrive 项目由 ODrive Robotics 公司开发，旨在为机器人应用提供高性能的电机控制解决方案。项目的发展可以分为几个阶段：

**早期阶段（2016-2019）**
- 初始版本专注于双电机 BLDC/PMSM 控制
- 面向研究机构和高端机器人应用
- 完全开源的开发模式

**成长阶段（2020-2022）**
- ODrive v3.6 成为流行版本
- 在四足机器人、机械臂等领域获得应用
- 建立 odrivetool 配置工具
- 形成一定的社区规模

**商业化阶段（2023-至今）**
- 发布 ODrive Pro（高功率版本）
- 发布 ODrive S1（伺服版本）
- 发布 ODrive Micro（紧凑型版本，2024年）
- 新版本采用闭源模式

**技术成就**

ODrive 在电机控制技术方面取得了多项重要成就：

1. **双电机控制**：
   - 单板控制两个电机
   - 降低多轴系统的硬件复杂度
   - 简化布线和集成

2. **高精度控制**：
   - 亚微弧度的位置精度
   - 优秀的速度和转矩控制性能
   - 适合需要精确定位的应用

3. **高级控制算法**：
   - FOC（磁场定向控制）
   - 多种观测器算法
   - 自适应控制

4. **易用性**：
   - 相比 VESC 更少的调参需求
   - 即插即用的体验
   - 完善的文档和工具

### 开源状态变化详解

**ODrive v3.x：开源但 NRND**

ODrive v3.x 系列是 ODrive 的经典版本，也是最后一个完全开源的版本：

**技术规格**
- **电压范围**：最高 48V
- **连续电流**：2kW（v3.6）
- **控制模式**：位置、速度、转矩
- **接口**：USB、UART、CAN、Step/Dir、PWM
- **编码器支持**：增量、霍尔、SPI 绝对

**开源状态**
- ✅ 固件源代码在 GitHub 完全开放
- ✅ 硬件设计文件可获取
- ⚠️ 官方标注为 NRND（不推荐用于新设计）
- ⚠️ 不再接受新功能的 Pull Request
- ⚠️ 仅维护关键 bug 修复

**仍适用的场景**
- 学习和研究电机控制算法
- 不要求最新功能的应用
- 预算受限的项目（二手市场）
- 不希望使用闭源固件的原则性开源用户

**ODrive Pro / S1 / Micro：闭源但高性能**

**ODrive Pro**
- **定位**：高功率应用
- **电压范围**：最高 56V
- **连续电流**：更高功率输出
- **应用**：工业机器人、自动化设备
- **开源状态**：❌ 闭源固件
- **优势**：专业性能、官方支持

**ODrive S1**
- **定位**：伺服应用
- **集成设计**：优化的伺服性能
- **应用**：机械臂、精密定位
- **开源状态**：❌ 闭源固件
- **优势**：开箱即用、调参简单

**ODrive Micro（2024年发布）**
- **定位**：紧凑型空间受限应用
- **发布时间**：2024年7月
- **特点**：超小型设计、CAN 和 USB 接口
- **应用**：空间受限的机器人系统
- **开源状态**：❌ 闭源固件
- **优势**：体积小、性能强

**闭源策略分析**

ODrive Robotics 选择闭源新一代产品固件，可能出于以下考虑：

1. **商业保护**：保护核心技术和商业利益
2. **竞争优势**：维持技术领先优势
3. **支持成本**：提供官方支持需要商业模式支撑
4. **产品质量**：控制固件质量，避免不当使用

**最新固件版本（2024-2025）**
- v0.6.11（2025年4月14日）
- v0.6.11-1（2025年5月23日，补丁版本）
- 新功能：
  - 完整的 odrivetool-over-CAN 支持
  - GUI-over-CAN 使用支持
  - Bug 修复和稳定性改进

### 社区反应与讨论

**Reddit 社区讨论**

Reddit r/LouisRossmann 和其他技术论坛上有用户讨论 ODrive "转向闭源"的问题：

- **失望情绪**：许多用户对 ODrive 不再完全开源感到失望
- **理解声音**：也有用户理解商业化的必要性
- **替代方案寻求**：讨论转向完全开源的替代方案

**开源社区的分歧**

ODrive 的闭源转变在开源社区引发了关于"开源项目商业化"的广泛讨论：

1. **开源纯粹主义者**：认为项目应该始终保持完全开源
2. **务实派**：理解商业化需求，但仍希望部分开源
3. **中间路线**：接受"开源核心，闭源高级功能"的模式

**对生态系统的影响**

ODrive 的闭源转变产生了多方面影响：

**正面影响**
- 推动了 ODrive 商业化成功
- 提供了官方支持和产品质量保证
- 促进了商业应用

**负面影响**
- 削弱了开源社区贡献
- 限制了学习者和研究者的深入定制
- 可能导致社区分裂

### ODrive 仍适用的场景

尽管新一代 ODrive 产品采用闭源模式，ODrive 仍在某些场景下具有优势：

**商业应用**
- 需要官方技术支持和 SLA（服务级别协议）
- 对可靠性和稳定性有极高要求
- 预算允许使用商业产品

**快速原型开发**
- 需要"即插即用"的体验
- 不想深入了解底层技术
- 时间紧迫的项目

**高性能要求**
- 需要绝对最新的性能优化
- 需要官方提供的高级功能
- 对成本不太敏感

**但是**，对于以下场景，应该考虑完全开源的替代方案：

- 研究和教学（需要深入理解算法）
- 长期维护（避免厂商锁定）
- 成本敏感项目
- 原则性开源需求
- 需要定制功能

## 第三部分：moteus - 机器人专业解决方案

### 项目背景与定位

**开发者与团队**

moteus 由 mjbots（Machine Jelly Robots）开发，这是一个专注于高性能机器人系统的团队和组织。mjbots 在机器人社区，特别是四足机器人领域享有盛誉。

**项目定位**

moteus 的定位非常明确：**专为四足机器人和高性能机器人关节设计的专业级伺服控制器**。这与 VESC 和 ODrive 的广泛适用性形成对比——moteus 更加专注和专业。

**核心设计哲学**

1. **专业第一**：不是通用电机控制器，而是专为机器人设计
2. **性能至上**：在机器人的实际工作负载下优化性能
3. **完整生态**：提供从控制器到软件的完整解决方案
4. **学术与工业结合**：既满足研究需求，也满足实用要求

### 技术架构与规格

**硬件架构**

moteus r4.11（最新版本）的硬件架构：

- **主控芯片**：基于 STM32 系列高性能微控制器
- **功率级**：定制化功率级设计
- **电流检测**：高精度电流采样
- **电压范围**：支持宽电压范围输入
- **尺寸**：紧凑设计，适合直接安装在机器人关节中

**控制算法**

moteus 实现了完整的 FOC 控制算法：

- **FOC（磁场定向控制）**
- **位置控制**
- **速度控制**
- **转矩控制**
- **轨迹跟踪**

**通信接口**

- **CAN 总线**：主要通信方式，支持多电机协调
- **UART**：用于调试和配置
- **辅助接口**：编码器、传感器等

**传感器支持**

moteus 支持多种高精度编码器：

- **MA732**：磁性编码器
- **AS5047**：绝对编码器
- **增量编码器**：通用 ABI 编码器
- **其他**：可通过自定义配置添加

**技术规格亮点**

与 ODrive 和 VESC 相比，moteus 在以下方面具有优势：

1. **专为机器人优化**：
   - 在实际机器人工作负载下测试和优化
   - 考虑了机器人的振动、冲击等实际工况
   - 优化了动态响应

2. **四足机器人专业**：
   - 在 MIT Mini Cheetah 等四足机器人项目中验证
   - 针对四足机器人的特殊需求优化
   - 支持多轴协调控制

3. **紧凑设计**：
   - 可直接安装在机器人关节中
   - 减少布线和系统复杂度
   - 提高系统可靠性

### 硬件与软件生态

**硬件产品线**

mjbots 提供了完整的硬件产品线：

4. **hoverbot（2024年新项目）**：

   - 展示了 moteus 在新应用中的潜力

**文档与资源**

moteus 拥有完善的文档体系：

2. **博客**：
   - https://blog.mjbots.com/
   - 项目更新、技术文章、案例分析

3. **学术论文**：
   - TurtleRabbit 2024 SSL（Soccer-Playing League）团队论文
   - 引用 moteus 的设计和实现
   - 学术认可

### 在四足机器人中的应用

**MIT Mini Cheetah**

MIT Mini Cheetah 是著名的四足机器人项目，其设计目标之一是让四足机器人像动物一样运动。moteus 控制器在该项目中发挥了关键作用：

1. **关节控制**：
   - 每个关节使用一个或多个 moteus 控制器
   - 高精度位置控制
   - 快速动态响应

2. **多轴协调**：
   - 通过 CAN 总线协调多个关节
   - 实现复杂的运动模式
   - 实时性能优化

3. **可靠性**：
   - 在实际机器人中验证
   - 承受振动、冲击等工况
   - 长期稳定运行

**其他四足机器人项目**

moteus 在多个四足机器人项目中得到应用：

- **学术研究**：大学和研究机构的四足机器人项目
- **竞赛**：RoboCup 等机器人竞赛
- **商业原型**：商业化四足机器人原型

**成功因素**

moteus 在四足机器人领域成功的关键因素：

1. **专业专注**：不是通用控制器，而是专为机器人优化
2. **完整生态**：从硬件到软件的完整解决方案
3. **开源透明**：完全开源，允许定制和学习
4. **实战验证**：在实际项目中证明性能
5. **持续开发**：2024年仍在活跃更新

## 第四部分：SimpleFOC - 易用性与灵活性

### 项目理念与设计哲学

**核心使命**

SimpleFOC 的核心使命是**让 FOC（磁场定向控制）技术变得简单易懂**，降低学习门槛，使更多人能够使用先进的电机控制技术。

**设计哲学**

1. **简化复杂性**：
   - 将复杂的 FOC 算法简化为易于理解的代码
   - 提供清晰的 API 和接口
   - 减少配置和调参的复杂度

2. **模块化设计**：
   - 核心库与硬件驱动分离
   - 灵活的组件组合
   - 易于扩展和定制

3. **跨平台兼容**：
   - 支持多种硬件平台
   - 不限于特定的微控制器
   - 用户可以选择熟悉的硬件

4. **开放包容**：
   - MIT 许可证，极其宽松
   - 欢迎社区贡献
   - 鼓励 experimentation

**与传统 FOC 实现的区别**

传统的 FOC 实现（如 VESC、ODrive）通常：
- 代码量大、复杂度高
- 依赖特定的硬件平台
- 配置复杂，学习曲线陡峭
- 难以理解和修改

SimpleFOC 的优势：
- 代码简洁清晰
- 可移植到不同平台
- API 直观易用
- 适合学习和 experimentation

### 核心功能与技术特性

**电机类型支持**

SimpleFOC 支持多种电机类型，这是其一大特色：

4. **PMSM 电机**（永磁同步电机）：

**控制模式**

SimpleFOC 提供了多种控制模式，满足不同应用需求：

   - 控制电机位置
   - P-控制器或 PID 控制

**传感器支持**

SimpleFOC 的强大之处在于其广泛的传感器支持：

**编码器**

- **支持类型**：

## 第五部分：MESC - 成本优化替代方案

### 项目定位与目标

**项目起源**

MESC（Molony ESC）由 David Molony 开发，是对 VESC 项目的直接回应和改进。David Molony 认为可以创建一个**更易于学习、更低成本、更灵活**的 FOC 电机控制器，同时保持或超越 VESC 的性能。

**项目定位**

MESC 的核心定位是：

1. **VESC 的低成本替代方案**：
   - 使用更便宜的元器件
   - 使用行业标准封装
   - DIY 友好的设计

2. **学习友好的 FOC 实现**：
   - 代码更易理解
   - 不基于 VESC 代码
   - 便于学习 FOC 原理

3. **高度可移植**：
   - 轻松移植到不同平台
   - 模块化设计
   - 清晰的架构

**设计目标**

MESC 的核心设计目标包括：

- **成本优化**：比 VESC 更低的硬件成本
- **可理解性**：代码清晰，便于学习
- **可定制性**：易于修改和扩展
- **性能**：不牺牲性能
- **社区支持**：活跃的社区

### 技术特色与创新点

**原创 FOC 实现**

MESC 最大的特色是其**完全原创的 FOC 实现**：

**不基于 VESC**
- VESC 基于早期的代码库
- MESC 从零开始编写
- 代码结构清晰

**原创的数学实现**
- FOC 数学公式独立实现
- 不依赖 VESC 的代码
- 便于理解和修改

**简化的架构**
- 模块化设计
- 清晰的代码组织
- 易于导航

**无传感器控制策略**

MESC 实现了创新的无传感器控制策略：

**相位控制**
- 计算虚功率（imaginary power）
- 维持虚功率为零
- 无需电机参数先验知识
- 包括电阻、电感、反电动势

**优势**
- 简化配置流程
- 自动参数识别
- 更好的鲁棒性

**硬件设计**

MESC_FOC_ESC 的硬件设计特点：

**成本优化**
- 使用行业标准封装
- D2PAK 或 H2PAK 封装的 MOSFET
- 标准栅极驱动器
- 避免 VESC 使用的昂贵或特殊器件

**可制造性**
- 标准 PCB 制造工艺
- 易于手工焊接
- 可用标准元件组装

**兼容性**

- 支持多种硬件平台

VESC：
- 通常 4 层板
- 高速信号完整性要求
- 复杂的布局

MESC：
- 可用 2 层板实现
- 更宽松的设计规则
- 更简单的布局

**成本节省**：
- PCB 制造成本降低
- 手工焊接更容易
- DIY 友好

**3. 简化的辅助电路**

VESC：
- 集成的多种保护功能
- 复杂的电源管理
- 丰富的接口

MESC：
- 基础保护功能
- 简化的电源管理
- 必要的接口

**成本影响**：
- 元件数量减少
- BOM 成本降低
- 但功能略有缩减

- **MESC 硬件成本**：$20-50（简化版本）

### FPGA-FOC：基于 FPGA 的超高性能方案

**项目介绍**

FPGA-FOC 是一个基于 FPGA（现场可编程门阵列）的 FOC 控制器实现项目，探索使用硬件加速电机控制算法的可能性。

**GitHub**：https://github.com/WangXuan95/FPGA-FOC

**技术原理**

传统 FOC 实现使用微控制器（MCU）的软件实现：

- 采样频率：通常 8-16 kHz
- 算法延迟：软件执行时间
- 性能瓶颈：MCU 性能限制

FPGA 实现的优势：

- **超高速采样**：可达 100 kHz 甚至更高
- **超低延迟**：硬件并行处理
- **确定性**：固定的执行时间
- **可重构**：可编程硬件

**技术特点**

1. **硬件加速**：
   - 坐标变换在硬件中实现
   - PWM 生成在硬件中实现
   - 电流环控制在硬件中实现

2. **高带宽**：
   - 支持极高的控制频率
   - 适合高性能应用

3. **灵活性**：
   - FPGA 可重新编程
   - 可升级和优化算法

**应用场景**

FPGA-FOC 适合以下应用：

1. **超高速应用**：
   - 高速电机（>100,000 RPM）
   - 超精密控制
   - 需要极高带宽

2. **研究和实验**：
   - 新算法验证
   - FPGA 架构研究
   - 学术论文

3. **专业应用**：
   - 航空航天
   - 高端工业设备
   - 科学仪器

**优势与挑战**

**优势**：
- 性能远超软件实现
- 确定性实时性
- 可定制硬件

**挑战**：
- 开发门槛高
- 需要 FPGA 专业知识
- 成本较高
- 工具链复杂

**学习资源**

- GitHub 仓库文档
- FPGA 开发工具文档
- 相关学术论文

**替代方案**：

- ✅ 紧凑设计，适合关节安装

- 调优 PID 参数

- 需要双电机：考虑 ODrive（v3.x 或新一代）

## 总结

开源电机控制器领域在 2024-2025 年经历了重大变化。ODrive 新一代产品转向闭源标志着这个领域进入了一个新阶段，同时也催生了更多完全开源的替代方案。

**关键发现**

1. **moteus**：专为四足机器人设计的专业级解决方案，完全开源且在 2024 年活跃更新，在机器人领域表现出色。

2. **SimpleFOC**：以易用性著称的 Arduino FOC 库，持续活跃开发（2025年7月最新版本），降低了 FOC 技术的入门门槛。

3. **MESC**：VESC 的低成本、易学习替代方案，原创 FOC 实现，适合预算敏感和学习需求。

4. **VESC**：继续在电动车辆和 DIY 领域保持领先，2025 年仍在活跃更新（v6.06.6），强大的社区和生态系统。

5. **ODrive**：经典版本（v3.x）开源但不再维护，新一代产品性能卓越但闭源，适用于商业应用。

**选择建议摘要**

- **四足机器人/专业机器人**：moteus
- **快速原型/教学/DIY**：SimpleFOC
- **电动车辆/高性能应用**：VESC
- **成本敏感/学习 FOC**：MESC
- **商业应用/即插即用**：ODrive 新一代

每个项目都有其独特的优势和适用场景。重要的是根据具体的应用需求、技术要求、预算和学习曲线选择最合适的方案。开源电机控制器社区正在蓬勃发展，为不同的应用提供了多样化的选择。无论您的需求是什么，都能找到一个合适的开源解决方案。
