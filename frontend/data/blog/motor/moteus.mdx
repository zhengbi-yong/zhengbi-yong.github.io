## 引言

在现代机器人技术领域，关节驱动器是实现动态运动和精确控制的核心组件。传统的电机控制器虽然能够满足基本的运动控制需求，但在面对四足机器人、人形机器人、机械臂等复杂应用时，往往暴露出多轴协调困难、控制频率不足、通信带宽受限等问题。moteus 正是在这样的背景下诞生的——一款由 mjbots（Machine Jelly Robots）开发的开源高性能电机控制器，专为机器人关节应用而优化。

moteus 的起源可以追溯到著名的 MIT Mini Cheetah 四足机器人项目。作为 MIT Mini Cheetah 项目中成功验证的关节控制器方案，moteus 继承了该项目在高动态运动控制方面的技术积累，并通过开源方式将其推广到更广泛的机器人应用中。与 ODrive 和 VESC 等通用电机控制器不同，moteus 从设计之初就专注于解决机器人关节控制的特定挑战：准直驱（QDD）优化、多轴高速协调、低延迟通信等。这种专业化定位使得 moteus 在机器人应用领域，特别是四足机器人和高动态机械臂领域，展现出独特的优势。

**核心技术优势**

moteus 在多个技术维度上实现了突破：

1. **超高频控制**：30kHz 的电流环控制频率，远超 ODrive 的 8kHz 和 VESC 的 8-16kHz，确保更高的控制带宽和更快的动态响应。

2. **CAN-FD 高速通信**：支持高达 5Mbps 的 CAN-FD 通信协议（传统 CAN 2.0 仅为 1Mbps），64 字节数据帧，使得多轴同步控制成为可能。配合 pi3hat 硬件，单个树莓派可以轻松协调 16 个 moteus 控制器。

3. **QDD 优化设计**：针对准直驱（Quasi-Direct Drive）架构进行专门优化，在低减速比（6:1到10:1）条件下仍能实现高扭矩密度和优异的力控制精度。

4. **完整的开源生态**：固件完全开源（Apache 2.0），提供丰富的软件工具链，包括 Python API、moteus-arduino 库、tview 实时监控工具等。

5. **实战验证**：在 MIT Mini Cheetah、Quad A1 四足机器人、DexWrist 机械腕等多个实际项目中得到验证，证明了其可靠性和实用性。

**应用领域**

moteus 的应用场景主要集中在需要高动态性能和多轴协调的机器人系统：

- **四足机器人**：MIT Mini Cheetah、Quad A1 等项目的核心控制器，每个腿的髋关节和膝关节各使用一个 moteus，通过 CAN-FD 总线实现 12 轴的协调控制。

- **机械臂与灵巧手**：从 6 自由度机械臂到多指灵巧手，moteus 的高精度位置控制和扭矩控制能力使其成为关节驱动的理想选择。

- **云台与稳定平台**：利用其快速响应特性，实现高性能的姿态稳定和目标跟踪。

- **研究与教育**：开源特性使得研究者可以深入理解控制算法的实现细节，并根据需求进行定制和实验。

**与其他控制器的定位差异**

在开源电机控制器生态中，moteus、ODrive 和 VESC 各有其独特的定位：

- **moteus**：机器人关节专家，专注于多轴协调、QDD 优化和四足机器人应用。

- **ODrive**：通用高性能电机控制器，支持双电机控制，适用于广泛的精密运动控制应用，但新一代产品已转向闭源。

- **VESC**：电动车辆领域的标杆，以强大的电流能力和成熟的生态系统著称，但在机器人多轴协调方面相对薄弱。

## 第一部分：moteus 项目全景

### 1.1 项目背景与发展历程

**起源：MIT Mini Cheetah 的技术传承**

moteus 的故事始于著名的 MIT Mini Cheetah 四足机器人项目。MIT Mini Cheetah 是 MIT 生物仿生实验室开发的突破性四足机器人，以其高动态运动能力和开源设计而闻名。该项目需要一个能够支持高频控制、多轴协调且高度可靠的关节控制器，而当时的商业产品难以满足这些需求。

正是在这样的背景下，mjbots 的创始人开始开发专为机器人关节优化的电机控制器。moteus 的早期设计直接继承了 MIT Mini Cheetah 项目的技术理念：高控制频率、低延迟通信和准直驱优化。经过多次迭代和优化，moteus 从一个研究项目逐渐演变为成熟的产品，并通过开源方式向更广泛的机器人社区推广。

**发展时间线**

moteus 的发展可以分为几个关键阶段：

**2019-2020年：原型探索期**
- r4.3 和 r4.5 版本：早期原型设计，验证核心概念
- 确立 FOC 控制架构和 CAN-FD 通信方案
- 在 MIT Mini Cheetah 项目中初步验证

**2021-2022年：产品化时期**
- r4.8 版本：改进硬件设计，提升可靠性
- 建立完整的软件工具链（moteus_tool、Python API）
- 开始在四足机器人社区获得关注

**2023-2024年：生态扩展期**
- r4.11 版本：当前标准版本，性能和稳定性达到新高度
- 推出多样化产品线（n1 迷你版、c1 紧凑型、x1 旗舰版）
- pi3hat 多轴协调板的成熟应用
- Quad A1 四足机器人产品发布

**2024-2025年：技术深化期**
- CAN-FD 协议优化，支持更多轴的同步控制
- 固件性能持续优化（2024年10月最新更新）
- 社区生态持续壮大，Discord 活跃用户超过数千人
- 在机械臂、灵巧手等新领域获得应用

**mjbots 公司定位**

mjbots（Machine Jelly Robots）是一家专注于开源机器人硬件的公司，其核心理念是通过开源方式推动机器人技术的普及。与传统的商业公司不同，mjbots 采用"开源硬件 + 商业支持"的混合模式：

- **开源承诺**：固件完全开源（Apache 2.0），硬件设计部分公开
- **商业产品**：提供组装好的控制器和相关配件，降低用户使用门槛
- **社区驱动**：通过 Discord、GitHub、博客等渠道与用户保持密切互动
- **持续创新**：投入研发资源，不断优化产品和扩展功能

这种模式使得 mjbots 能够在保持开源精神的同时，实现可持续的商业运营，为用户提供高质量的产品和技术支持。

**社区与商业生态**

moteus 已经形成了一个繁荣的生态体系：

- **学术研究**：多所大学和研究中心将 moteus 用于四足机器人、人形机器人等研究项目
- **商业应用**：从教育机器人到工业自动化，moteus 在多个商业项目中得到应用

**moteus r4.11：标准款，MIT Cheetah 同款**

r4.11 是 moteus 的经典版本，也是 MIT Mini Cheetah 使用的控制器。它在性能、尺寸和成本之间取得了良好的平衡，是大多数四足机器人项目的首选。

**moteus n1：迷你版，小型机器人理想选择**

n1 是 moteus 的迷你版本，专为空间受限的应用设计。尽管尺寸小巧，但仍保留了 moteus 的核心功能。

**核心特性**：
- **超小尺寸**：38×38×9mm，重量仅 8.9g
- **功率能力**：200W 峰值功率，适合小型关节
- **电压范围**：10-54V，更宽的电压适应范围
- **紧凑设计**：优化的布局，便于集成到紧凑空间

**适用场景**：
- 多指灵巧手关节
- 小型四足机器人（如 Mini Cheetah 缩小版）
- 空间受限的云台系统

**moteus c1：紧凑型，平衡性能与尺寸**

c1 在 r4.11 和 n1 之间提供了折中方案，适中的尺寸和功率能力。

**核心特性**：
- **尺寸适中**：46×46×8mm
- **功率能力**：250W @ 28V
- **宽电压范围**：10-51V

**适用场景**：
- 中型机械臂
- 需要比 r4.11 更紧凑但功率接近的应用

**moteus x1：旗舰版，工业级应用**

x1 是 moteus 的旗舰版本，为高功率、高可靠性的工业应用而设计。

**核心特性**：
- **高功率**：2000W @ 36V，62A 连续相电流，120A 峰值
- **工业级设计**：增强的散热和保护电路
- **宽电压范围**：10-54V

**适用场景**：
- 大型四足机器人
- 工业机械臂
- 高扭矩密度的应用

**硬件架构核心**

所有 moteus 型号都基于相似的硬件架构，主要体现在以下几个方面：

**微控制器平台**：
- **主控芯片**：STM32G474RET6（ARM Cortex-M4，170MHz）
- **浮点单元**：硬件 FPU，加速 FOC 算法计算
- **存储器**：1MB Flash，128KB RAM
- **外设**：12 位 ADC，高级定时器，CAN-FD 控制器

**栅极驱动器**：
- **芯片型号**：DRV8323 智能功率级栅极驱动器
- **特性**：集成三个半桥驱动器、 buck 转换器、运放
- **保护功能**：过流保护、过温保护、欠压锁定

**电流检测**：
- **方案**：分流器 + 运放
- **精度**：12 位 ADC，高精度采样
- **带宽**：支持 30kHz 控制频率

**编码器接口**：
- **板载编码器**：AS5047P（14 位磁编码器，SPI 接口）
- **外部编码器支持**：
  - SPI：AS5047x, MA732, CUI AMT22x
  - RS422：CUI AMT21x
  - ABI：增量编码器
  - 模拟：正余弦编码器

**通信接口**：
- **CAN-FD**：5Mbps 数据速率，64 字节数据帧
- **辅助接口**：UART、SPI、I2C（用于编码器和传感器）

```python
import asyncio
import moteus

async def main():
    # 创建控制器实例
    c = moteus.Controller(id=1)

    # 位置控制
    await c.set_position(
        position=1.0,
        velocity=0.0,
        maximum_torque=10.0,
        kp=50,
        kd=1.0
    )

    await asyncio.sleep(1.0)

asyncio.run(main())
```

**5. pi3hat：树莓派多轴协调板**

pi3hat 是专为树莓派设计的扩展板，是多轴协调的关键硬件：

- **4 路 CAN-FD 接口**：支持最多 16 个 moteus 控制器
- **时间同步**：硬件级同步，确保多轴控制的时间一致性
- **集成传感器**：9 轴 IMU（MPU-6500）和 ABSOLUTE 参考系统
- **高速 SPI**：与树莓派的高速通信

```python
from mjbots.pi3hat import Pi3Hat
import moteus

# 创建 pi3hat 实例
transport = Pi3Hat(
    cpu = 1,
    spi_speed = 3000000,
)

# 创建多个控制器
controllers = [
    moteus.Controller(id=i, transport=transport)
    for i in range(1, 5)
]

# 同步控制
commands = [
    moteus.Command(position=i*0.5, kp=50, kd=1.0)
    for i in range(4)
]

results = await transport.cycle(commands)
```

## 第二部分：核心技术深度解析

### 2.1 FOC 控制算法实现

FOC（Field Oriented Control，磁场定向控制）是现代电机控制的核心技术。moteus 通过高频率的 FOC 实现了卓越的动态性能和控制精度。本节将深入解析 moteus 的 FOC 算法实现，从数学原理到代码架构。

**FOC 基础理论**

FOC 的核心思想是通过坐标变换，将三相交流电机的控制问题转化为类似直流电机的控制问题。主要步骤包括：

1. **Clarke 变换**：三相静止坐标系（a,b,c）→ 两相静止坐标系（α,β）
2. **Park 变换**：两相静止坐标系（α,β）→ 两相旋转坐标系（d,q）
3. **PI 控制**：在 d-q 坐标系中独立控制磁场（d 轴）和转矩（q 轴）
4. **反 Park 变换**：d,q → α,β
5. **SVPWM 调制**：生成三相 PWM 信号

**数学变换详解**

**Clarke 变换（3→2）**：

将三相电流（Ia, Ib, Ic）变换为两相静止坐标系（Iα, Iβ）：

```
Iα = (2/3) * (Ia - 0.5 * Ib - 0.5 * Ic)
Iβ = (2/3) * (0.866 * Ib - 0.866 * Ic)
```

在平衡三相系统中（Ia + Ib + Ic = 0），可以简化为：

```
Iα = Ia
Iβ = (1/√3) * (Ia + 2 * Ib)
```

**Park 变换（静止→旋转）**：

将静止坐标系（Iα, Iβ）变换为旋转坐标系（Id, Iq）：

```
Id = Iα * cos(θ) + Iβ * sin(θ)
Iq = -Iα * sin(θ) + Iβ * cos(θ)
```

其中 θ 是转子电角度。

**SVPWM 调制策略**

空间矢量脉宽调制（SVPWM）是 FOC 的最后环节，其作用是将 d-q 坐标系中的电压指令转换为三相 PWM 信号。moteus 采用连续调制策略，通过计算电压矢量的作用时间，生成最优的 PWM 波形。

**moteus 特有优化**

**超高频控制频率**

moteus 的电流环控制频率可达 30kHz，远超 ODrive 的 8kHz 和 VESC 的 8-16kHz。这意味着：

- **更快的动态响应**：从命令到输出的延迟降低到几十微秒
- **更高的控制带宽**：能够响应更快的扰动和指令变化
- **更小的转矩脉动**：高频率 PWM 减少低频噪声

**硬件加速**

moteus 充分利用 STM32G4 的硬件特性：

- **Cordic 硬件加速**：快速计算三角函数（sin/cos）
- **硬件 FPU**：浮点运算加速
- **DMA 传输**：减少 CPU 干预，降低延迟
- **高级定时器**：纳秒级 PWM 精度

这是 moteus 固件的核心文件，包含了完整的 FOC 控制逻辑。主要功能模块包括：

```cpp
// 电流控制中断服务程序（简化示例）
void BldcServo::ISR_DoCurrent(
    const SinCos& sin_cos,
    float i_d_A_in, float i_q_A_in,
    float feedforward_velocity_rotor,
    bool ignore_position_bounds) {

  // 1. D轴电流控制（磁场控制）
  const float denorm_d_V = pid_d_.Apply(
      status_.d_A, i_d_A_in, rate_hz);

  // 2. Q轴电流控制（转矩控制）
  const float denorm_q_V = pid_q_.Apply(
      status_.q_A, i_q_A_in, rate_hz);

  // 3. 电压限制和解耦
  const auto result = SVM(denorm_d_V, denorm_q_V, sin_cos);

  // 4. 输出 PWM
  ISR_DoVoltageDQ(sin_cos, d_V, q_V);
}
```

**级联控制结构**

moteus 采用经典的三环级联控制：

1. **位置环（最外层）**：目标位置 → 速度指令
2. **速度环（中间层）**：目标速度 → 转矩指令
3. **电流环（最内层）**：目标转矩 → 电压输出

```
位置误差 → PID → 速度指令
    ↓
速度误差 → PID → 转矩指令
    ↓
Iq误差 → PID → 电压输出
```

- **最高优先级**：电流环中断（30kHz）
- **中优先级**：编码器读取、ADC 采样
- **低优先级**：CAN 通信、配置更新

**时序优化**：

- **ADC 触发**：硬件触发，精确对齐 PWM 中心
- **采样窗口**：1μs 精确采样窗口
- **计算时间**：&lt;30μs 完整控制周期

### 2.2 CAN-FD 通信协议

CAN-FD（Controller Area Network Flexible Data Rate）是 moteus 多轴协调的关键技术。相比传统 CAN 2.0，CAN-FD 提供了更高的数据速率和更大的数据帧，使得多电机同步控制成为可能。

**CAN 2.0 vs CAN-FD 对比**

| 特性 | CAN 2.0 | CAN-FD | 优势 |
|------|---------|--------|------|
| **数据段速率** | 最高 1Mbps | 最高 5Mbps | 5倍提升 |
| **数据长度** | 最多 8 字节 | 最多 64 字节 | 8倍提升 |
| **带宽利用率** | ~50% | ~80% | 更高效 |
| **时间确定性** | 好 | 优秀 | 更精确 |

**数据帧结构**

**CAN 2.0 帧**：
```
[SOF][ID][DLC][Data 0-8B][CRC][ACK][EOF]
```

**CAN-FD 帧**：
```
[SOF][ID][DLC][EDL][BRS][Data 0-64B][CRC][ACK][EOF]
```

其中：
- **EDL（Extended Data Length）**：标识为 CAN-FD 帧
- **BRS（Bit Rate Switch）**：数据段切换到高速率

**moteus CAN-FD 协议详解**

moteus 使用自定义的 CAN-FD 协议，针对电机控制优化。主要命令类型包括：

**命令类型表**：

| 命令类型 | CAN ID | 数据长度 | 功能描述 |
|---------|--------|---------|----------|
| **正扭矩命令** | 0x00 | 8 字节 | 设置正扭矩值 |
| **负扭矩命令** | 0x01 | 8 字节 | 设置负扭矩值（再生制动）|
| **位置命令** | 0x02 | 8 字节 | 目标位置+速度+前馈扭矩 |
| **查询状态** | 0x03 | 0 字节 | 请求状态反馈 |
| **状态反馈** | 0x04 | 64 字节 | 完整状态（位置、速度、扭矩、温度）|
| **故障诊断** | 0x05 | 变长 | 错误码和诊断信息 |
| **停止命令** | 0x06 | 0 字节 | 紧急停止 |
| **重置命令** | 0x07 | 0 字节 | 控制器软复位 |

**位置命令详解**

位置命令是 moteus 最常用的命令类型，包含以下字段：

```python
# 位置命令数据结构（简化）
class PositionCommand:
    position: float        # 目标位置（转）
    velocity: float        # 速度前馈（转/秒）
    maximum_torque: float  # 最大扭矩限制（Nm）
    kp_scale: float        # 位置增益缩放
    kd_scale: float        # 阻尼增益缩放
    feedforward_torque: float  # 前馈扭矩（Nm）
    query: bool            # 是否请求状态反馈
```

**Python API 使用示例**：

```python
import asyncio
import moteus

async def main():
    # 创建单个控制器
    c = moteus.Controller(id=1)

    # 位置控制
    await c.set_position(
        position=1.0,          # 目标：1转
        velocity=2.0,          # 速度前馈：2转/秒
        maximum_torque=10.0,   # 最大扭矩：10Nm
        kp=50,                 # 位置增益
        kd=1.0,                # 阻尼增益

    print(f"速度: {result.values[moteus.Register.VELOCITY]}")
    print(f"扭矩: {result.values[moteus.Register.TORQUE]}")

asyncio.run(main())
```

**多轴协调技术**

**时间同步挑战**

在多电机系统中，确保所有电机同时接收命令是实现协调控制的关键。传统 CAN 2.0 由于带宽限制，难以实现高频同步。

**pi3hat 的解决方案**

pi3hat 通过以下机制实现精确同步：

1. **硬件级时间戳**：每个 CAN 帧都带有硬件时间戳
2. **广播命令**：同时向所有电机发送命令
3. **调度算法**：精确计算每个帧的发送时间
4. **优先级管理**：确保关键帧优先发送

**代码示例**：

```python
from mjbots.pi3hat import Pi3Hat
import moteus
import asyncio

async def multi_axis_control():
    # 创建 pi3hat 传输层
    transport = Pi3Hat(
        cpu = 1,               # 绑定到 CPU 核心 1
        spi_speed = 3000000,   # SPI 速率 3Mbps
    )

    # 创建多个控制器
    controllers = [
        moteus.Controller(id=i, transport=transport)
        for i in range(1, 5)
    ]

    # 准备同步命令
    commands = [
        moteus.Command(
            position=i * 0.5,  # 每个电机不同位置
            kp=50,
            kd=1.0,
            maximum_torque=10.0,
            query=True
        )
        for i in range(4)
    ]

    # 发送同步命令
    results = await transport.cycle(commands)

    # 处理结果
    for i, result in enumerate(results):
        print(f"电机 {i+1}: 位置={result.values[moteus.Register.POSITION]:.3f}")

asyncio.run(multi_axis_control())
```

## 第三部分：QDD 技术与机器人关节应用

### 3.1 QDD（准直驱）技术详解

QDD（Quasi-Direct Drive，准直驱）是 moteus 的核心技术优势之一，也是其在机器人关节应用中脱颖而出的关键。与传统的高减速比方案不同，QDD 采用低减速比设计，在保持足够扭矩的同时，实现了优异的力控制性能和反向驱动性。

**传统减速电机 vs QDD**

**传统减速电机**：
```
电机 → 高减速比（100:1） → 输出
特点：高扭矩、低速度、高惯性、低反向驱动性
应用：CNC 机床、精密定位
```

**QDD 方案**：
```
电机 → 低减速比（6:1 到 10:1） → 输出
特点：适中扭矩、高速度、低惯性、高反向驱动性
应用：四足机器人、机械臂、人机交互
```

**数学模型**

**有效惯性计算**：

对于带减速器的系统，反射到电机侧的有效惯性为：

```
J_eff = J_motor + J_load / N²
```

其中：
- `J_eff`：系统总有效惯性
- `J_motor`：电机转子惯性
- `J_load`：负载惯性
- `N`：减速比

**QDD 的优势**：

在 QDD 设计中，减速比 N 较小（6:1 到 10:1），因此：

1. **低有效惯性**：`J_load / N²` 项较小，系统响应更快
2. **高透明度**：负载可以更容易地反向驱动电机
3. **高力控制精度**：减少了摩擦和齿隙的影响

**扭矩密度分析**

传统观念认为高减速比可以提供更高的扭矩密度，但实际情况更为复杂：

**输出扭矩**：
```
T_out = T_motor × N × efficiency
```

其中 `efficiency` 是齿轮箱效率（通常为 0.7-0.9）。

**QDD 的优化策略**：

moteus 通过以下方式优化 QDD 系统的扭矩输出：

1. **高转速电机**：选择高 Kv 值的电机（如 100-200 Kv）
2. **高电流能力**：30kHz 控制频率支持峰值电流达 100A
3. **低损耗减速器**：使用高效行星齿轮或谐波减速器
4. **热管理**：金属芯 PCB 提供优异的散热

**反向驱动性**

**反向驱动性**（Back-drivability）是 QDD 最显著的特性，指的是负载可以轻松地驱动电机转动。

**优势**：

1. **人机交互安全性**：当人与机器人碰撞时，关节会"退让"，减少冲击力
2. **力控制精度**：可以直接通过扭矩控制实现力控，无需额外的力传感器
3. **能量效率**：在重力补偿等场景下，电机可以反向发电，回收能量

**vs 传统方案对比表**：

| 特性 | QDD + moteus | 传统减速电机 + ODrive |
|------|-------------|---------------------|
| **反向驱动性** | 优秀（可手动驱动） | 差（几乎不可驱动） |
| **力控制精度** | 高（直接力反馈） | 中（受摩擦影响） |
| **冲击安全性** | 优秀（柔性吸收） | 差（刚性传递） |
| **透明度** | 高 | 低 |
| **峰值扭矩** | 中等 | 高 |
| **速度范围** | 宽 | 窄 |
| **控制带宽** | 高（>100Hz） | 中（~10-50Hz） |

**1. 虚拟模型控制（VMC）**

VMC 是四足机器人的核心控制算法，将复杂的动力学问题简化为虚拟模型：

```python
# 简化的 VMC 示例
class VirtualModelControl:
    def __init__(self):
        self.mass = 2.5  # 机器人质量
        self.gravity = 9.81
        self.kp_body = 1000  # 身体位置增益
        self.kd_body = 50    # 身体阻尼

    def compute_foot_forces(self, body_state, foot_positions):
        """计算每只脚的期望力"""
        # 计算重力补偿
        f_gravity = self.mass * self.gravity / 4

        # 计算身体控制力
        f_body = self.kp_body * (target_height - actual_height)

        # 分配到每只脚
        foot_forces = f_gravity + f_body

        return foot_forces
```

**2. 摆腿轨迹规划**

摆腿轨迹决定了足端运动的平滑性和稳定性：

```python
def swing_trajectory(t, T, start, end, height=0.05):
    """
    摆腿轨迹（三次样条）

    参数:
        t: 当前时间（0 到 T）
        T: 摆腿周期
        start: 起始位置
        end: 目标位置
        height: 抬腿高度
    """
    # 归一化时间
    tau = t / T

    # 水平运动（三次样条）
    x = start[0] + (end[0] - start[0]) * (3*tau**2 - 2*tau**3)
    y = start[1] + (end[1] - start[1]) * (3*tau**2 - 2*tau**3)

    # 垂直运动（正弦波）
    z = height * math.sin(math.pi * tau)

    return [x, y, z]
```

**3. 地面接触检测**

准确的接触检测是实现稳定步态的关键：

```python
def detect_contact(leg_id, motor_data):
    """通过电流和位置估计接触状态"""
    # 方法1：基于电流
    if motor_data.torque > threshold:
        contact = True

    # 方法2：基于位置误差

        contact = True

    # 方法3：基于加速度（需要 IMU）
```

```python
class AdmittanceControl:
    def __init__(self):
        self.K_spring = 100  # 虚拟刚度
        self.B_damper = 10   # 虚拟阻尼

    def compute(self, target_pos, actual_pos, external_force):
        """计算期望位置"""
        # 虚拟弹簧-阻尼系统
        delta = target_pos - actual_pos
        virtual_force = self.K_spring * delta

        # 考虑外力
        total_force = virtual_force - external_force

        # 转换为位置调整

        return target_pos + pos_adjustment
```

**PID 调优指南**

PID 调优是获得理想控制性能的关键步骤：

**调优步骤**：

1. **初始设置**：

   ```bash
   > conf set servo.kd 0.5
   ```

   - 从小值开始（0.1）
   - 缓慢增加
   - 注意：积分项可能引入振荡

**常见问题诊断**：

| 现象 | 可能原因 | 解决方案 |

### 5.2 轨迹规划与插值

轨迹规划是机器人实现复杂运动的关键技术。moteus 支持多种轨迹规划方法。

**梯形速度轨迹**

梯形速度轨迹是最简单、最常用的轨迹规划方法。

**参数定义**：

```python
def trapezoidal_trajectory(start, end, max_vel, accel):
    """
    计算梯形速度轨迹

    参数:
        start: 起始位置
        end: 目标位置
        max_vel: 最大速度
        accel: 加速度

    返回:
        [(time, position), ...] 离散轨迹点
    """
    distance = abs(end - start)

    # 计算加速时间
    t_accel = max_vel / accel
    dist_accel = 0.5 * accel * t_accel**2

    # 判断是否达到最大速度
    if distance < 2 * dist_accel:
        # 三角轨迹（未达到最大速度）
        t_total = 2 * math.sqrt(distance / accel)
        t_accel = t_total / 2
    else:
        # 梯形轨迹（有匀速段）
        dist_cruise = distance - 2 * dist_accel
        t_cruise = dist_cruise / max_vel
        t_total = 2 * t_accel + t_cruise

    # 生成轨迹点
    trajectory = []
    dt = 0.01  # 10ms 采样
    for i in range(int(t_total / dt)):
        t = i * dt

        if t < t_accel:
            # 加速段
            pos = 0.5 * accel * t**2
        elif t < t_accel + t_cruise:
            # 匀速段
            pos = dist_accel + max_vel * (t - t_accel)
        else:
            # 减速段
            t_dec = t - t_accel - t_cruise
            pos = distance - 0.5 * accel * (t_total - t)**2

        trajectory.append((t, pos))

    return trajectory
```

**使用示例**：

```python
async def follow_trajectory():
    c = moteus.Controller(id=1)

    # 生成轨迹
    trajectory = trapezoidal_trajectory(
        start=0.0,

        max_vel=5.0,
        accel=10.0
    )

    # 执行轨迹
    for t, pos in trajectory:
        await c.set_position(
            position=pos,
            kp=50,
            kd=1.0,
            maximum_torque=10.0,

        )

        await asyncio.sleep(0.01)

```

```python
from scipy import interpolate

def spline_trajectory(waypoints, duration):
    """
    三次样条轨迹

    参数:
        waypoints: 关键点列表 [(t0, pos0), (t1, pos1), ...]
        duration: 总时间

        样条插值函数
```

### 5.3 高级特性

**1. 阻抗控制（虚拟弹簧-阻尼）**

阻抗控制是实现柔性交互的关键技术。

**原理**：

将关节建模为虚拟的弹簧-阻尼系统：

```
F = K * (target_pos - actual_pos) + B * (target_vel - actual_vel)
```

其中：
- `K`：虚拟刚度
- `B`：虚拟阻尼

**代码实现**：

```python
class ImpedanceControl:
    def __init__(self, controller, K=100, B=10):
        self.controller = controller
        self.K = K  # 刚度
        self.B = B  # 阻尼

    async def control(self, target_pos, actual_pos, target_vel, actual_vel):
        """
        阻抗控制

        参数:
            target_pos: 目标位置
            actual_pos: 实际位置（从反馈获取）
            target_vel: 目标速度
            actual_vel: 实际速度（从反馈获取）
        """
        # 计算虚拟力
        pos_error = target_pos - actual_pos
        vel_error = target_vel - actual_vel

        virtual_force = self.K * pos_error + self.B * vel_error

        # 转换为扭矩命令
        await self.controller.set_position(
            position=float('nan'),
            velocity=0.0,
            kp=0.0,
            kd=0.0,
            feedforward_torque=virtual_force,
            query=True
        )

# 使用示例
impedance = ImpedanceControl(c, K=100, B=10)

while True:
    result = await c.set_position(query=True)
    actual_pos = result.values[moteus.Register.POSITION]
    actual_vel = result.values[moteus.Register.VELOCITY]

    await impedance.control(
        target_pos=1.0,
        actual_pos=actual_pos,
        target_vel=0.0,
        actual_vel=actual_vel
    )

    await asyncio.sleep(0.01)
```

**应用场景**：
- 人机交互安全
- 柔性抓取
- 力控装配

```python
def estimate_torque(motor_data):
    """
    通过双编码器估计关节扭矩

    扭矩 = K * (theta_load - theta_motor)
    """
    K_joint = 1000  # 关节刚度（Nm/rad）

    theta_motor = motor_data.motor_position
    theta_load = motor_data.aux_position

    torque = K_joint * (theta_load - theta_motor)
```

**3. 故障检测与恢复**

moteus 实现了完善的故障检测机制。

**故障类型**：

| 故障码 | 描述 | 触发条件 | 恢复方法 |
|--------|------|---------|---------|
| `FAULT_OVER_CURRENT` | 过流 | 电流 > `current_max` | 自动 |
| `FAULT_OVER_VOLTAGE` | 过压 | 电压 > `max_voltage` | 自动 |
| `FAULT_UNDER_VOLTAGE` | 欠压 | 电压 < 最小值 | 自动 |
| `FAULT_ENCODER` | 编码器故障 | 编码器信号丢失 | 检查接线 |
| `FAULT_OVER_TEMP` | 过温 | FET 温度过高 | 冷却后自动 |
| `FAULT_POSITION` | 位置超限 | 超出软限位 | 手动移回 |

**故障处理策略**：

```python
async def safe_fault_handler(controller):
    """
    安全故障处理
    """
    while True:
        result = await controller.set_position(query=True)

        # 检查故障
        fault = result.values[moteus.Register.FAULT]

        if fault != 0:
            print(f"故障检测: {fault:#x}")

            # 根据故障类型处理
            if fault & 0x01:  # 过流
                await controller.set_stop()
                await asyncio.sleep(1.0)

            elif fault & 0x10:  # 过温
                print("温度过高，等待冷却...")
                await asyncio.sleep(5.0)

            elif fault & 0x20:  # 编码器故障
                print("编码器故障，停止运动")
                await controller.set_stop()
                break  # 需要人工检查

        await asyncio.sleep(0.01)

asyncio.run(safe_fault_handler(c))
```

```python
class TrotGait:
    def __init__(self, transport):
        self.transport = transport
        self.controllers = [
            moteus.Controller(id=i, transport=transport)
            for i in range(1, 13)  # 12个电机
        ]

    async def step(self, t):
        """
        执行 trot 步态

        步态周期：
        0.0-0.5s：前左+后右腿摆动
        0.5-1.0s：前右+后左腿摆动
        """
        # 定义腿部索引
        FL = [1, 2, 3]   # 前左腿
        FR = [4, 5, 6]   # 前右腿
        BL = [7, 8, 9]   # 后左腿
        BR = [10, 11, 12] # 后右腿

        if (t % 1.0) < 0.5:
            # 前左 + 后右摆动
            swing_legs = FL + BR
            stance_legs = FR + BL
        else:
            # 前右 + 后左摆动
            swing_legs = FR + BL
            stance_legs = BR + FL

        # 准备命令
        commands = []

        for i in range(1, 13):
            if i in swing_legs:
                # 摆动腿：抬起并向前
                commands.append(moteus.Command(
                    position=0.5,  # 抬起
                    kp=50,
                    kd=1.0,
                    maximum_torque=10.0,
                    query=False
                ))
            else:
                # 支撑腿：接地支撑
                commands.append(moteus.Command(
                    position=0.0,  # 接地
                    kp=100,       # 高刚度
                    kd=2.0,
                    maximum_torque=10.0,
                    query=False
                ))

        # 发送同步命令
        await self.transport.cycle(commands)
```

```python
import asyncio
import moteus
import math

class QuadrupedController:
    def __init__(self):
        self.transport = moteus.Pi3Hat(
            cpu=1,
            spi_speed=3000000
        )

        # 12个关节控制器（ID 1-12）
        self.controllers = [
            moteus.Controller(id=i, transport=self.transport)
            for i in range(1, 13)
        ]

        # 腿部运动学参数
        self.leg_config = {
            'l1': 0.08,  # 上腿长度 (m)
            'l2': 0.12,  # 下腿长度 (m)
        }

    async def trot_gait(self, forward_speed=0.1, step_height=0.05):
        """Trot步态生成器"""
        t = 0
        dt = 0.01  # 100Hz控制频率
        period = 0.5  # 步态周期 (s)

        while True:
            t += dt
            phase = (t % period) / period

            # 四条腿的相位偏移
            leg_phases = [0, 0.5, 0.5, 0]  # FL, FR, RL, RR

            # 计算每条腿的足端位置
            foot_positions = []
            for i, leg_phase in enumerate(leg_phases):
                effective_phase = (phase + leg_phase) % 1.0

                # 轨迹生成（摆线轨迹）
                if effective_phase < 0.5:
                    # 支撑相：向后移动
                    x = forward_speed * (effective_phase - 0.25) * period
                    y = 0
                    z = 0
                else:
                    # 摆动相：抬起向前
                    swing_phase = (effective_phase - 0.5) * 2
                    x = forward_speed * (swing_phase - 0.5) * period * 0.5
                    y = 0
                    z = step_height * math.sin(swing_phase * math.pi)

                foot_positions.append((x, y, z))

            # 逆运动学求解关节角度
            joint_commands = self.ik_solve(foot_positions)

            # 发送命令到所有关节
            await self.transport.cycle(joint_commands)

            await asyncio.sleep(dt)

    def ik_solve(self, foot_positions):
        """逆运动学求解（简化版）"""
        commands = []

        for leg_id, (x, y, z) in enumerate(foot_positions):
            # 简化的2DOF逆运动学（仅示例）
            # 实际Quad A1使用3DOF腿部

            hip_angle = math.atan2(y, x)
            knee_angle = math.acos(
                (x**2 + y**2 + z**2 - self.leg_config['l1']**2 - self.leg_config['l2']**2) /
                (2 * self.leg_config['l1'] * self.leg_config['l2'])
            )

            # 每条腿3个关节，ID偏移
            base_id = leg_id * 3 + 1
            commands.extend([
                moteus.Command(position=hip_angle, kp=50, kd=1.0, maximum_torque=5.0, query=True),
                moteus.Command(position=knee_angle, kp=50, kd=1.0, maximum_torque=5.0, query=True),
                moteus.Command(position=0.0, kp=50, kd=1.0, maximum_torque=5.0, query=True),
            ])

        return commands

# 使用示例
async def main():
    robot = QuadrupedController()
    await robot.trot_gait(forward_speed=0.15, step_height=0.04)

asyncio.run(main())
```

### 7.3 性能调优指南

#### 7.3.1 PID参数调优

**系统化调优流程**：

1. **确定控制模式需求**：
   - 位置精度优先：提高kp
   - 快速响应：提高kd
   - 柔顺控制：降低kp和kd

2. **基础调优步骤**：
   ```python
   # Step 1: 从保守参数开始
   kp = 10
   kd = 0.1
   ki = 0  # 通常不需要积分项

   # Step 2: 逐步增加kp，直到出现轻微震荡
   # 例如: 10 -> 20 -> 40 -> 80 (震荡)
   # 最终选择: kp = 60

   # Step 3: 增加kd以抑制震荡
   # 例如: 0.1 -> 0.5 -> 1.0 -> 2.0 (稳定)
   # 最终选择: kd = 1.5

   # Step 4: 微调平衡位置误差和响应速度
   # 最终参数: kp=60, kd=1.5
   ```

3. **Ziegler-Nichols法（适用于困难系统）**：
   ```python
   # 1. 设kd=0, ki=0，逐步增加kp直到持续震荡
   #    记录临界kp_krit和震荡周期T_krit

   kp_krit = 120  # 临界增益
   T_krit = 0.5   # 震荡周期 (s)

   # 2. 根据公式计算初始参数
   kp = 0.6 * kp_krit      # = 72
   kd = 0.075 * kp_krit * T_krit  # = 4.5
   ki = 1.2 * kp_krit / T_krit    # = 288

   # 3. 微调优化
   ```

4. **负载自适应调优**：
   ```python
   # 不同负载范围使用不同参数
   def adaptive_pid(load_mass):
       if load_mass < 0.5:
           return {'kp': 80, 'kd': 2.0}  # 轻负载
       elif load_mass < 1.5:
           return {'kp': 50, 'kd': 1.0}  # 中等负载
       else:
           return {'kp': 30, 'kd': 0.5}  # 重负载
   ```

#### 7.3.2 轨迹优化

**平滑轨迹生成**：

```python
import numpy as np
import math

def minimum_jerk_trajectory(start, end, duration, dt=0.01):
    """最小加速度轨迹（5次多项式）"""
    t = np.arange(0, duration, dt)

    # 最小jerk轨迹系数
    T = duration
    a0 = start
    a1 = 0
    a2 = 0
    a3 = 10 / T**3
    a4 = -15 / T**4
    a5 = 6 / T**5

    trajectory = (a0 +
                  a1 * t +
                  a2 * t**2 +
                  a3 * (end - start) * t**3 +
                  a4 * (end - start) * t**4 +
                  a5 * (end - start) * t**5)

    return list(trajectory)

# 使用示例
waypoints = [0.0, 1.0, 0.5, 1.5, 0.0]
full_trajectory = []

for i in range(len(waypoints) - 1):
    segment = minimum_jerk_trajectory(
        waypoints[i],
        waypoints[i+1],
        duration=1.0,
        dt=0.01
    )
    full_trajectory.extend(segment)

# 发送到moteus
for target in full_trajectory:
    await controller.set_position(
        position=target,
        kp=50,
        kd=1.0,
        maximum_torque=5.0,
        query=True
    )
    await asyncio.sleep(0.01)
```

**节能策略**：

1. **零扭矩保持**（利用制动）：
   ```python
   # 到达目标后，降低增益减少功耗
   ```

   ```python
   # 根据速度动态调整扭矩限制
   ```

```python
async def hybrid_control():
    while True:
        # 同步控制关节和轮子
        # 1. 更新关节位置
        joint_commands = [moteus.Command(position=target[i], kp=50, kd=1.0) for i in range(12)]
```
