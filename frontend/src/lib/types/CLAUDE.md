# `@/lib/types` Module

## Layer 1: Module Overview

### Purpose
TypeScript type definitions and interfaces for API contracts, domain models, and data structures.

### Scope
- Backend API types (requests/responses)
- Blog content types
- User and authentication types
- Admin types
- Table of contents types
- Visitor tracking types

## Layer 2: Architecture

### Files
- **backend.ts**: Backend API types (largest file, ~9000 bytes)
- **api.ts**: API client types
- **toc.ts**: Table of contents types
- **visitor.ts**: Visitor tracking types

## Layer 3: Implementation Details

### Type Generation

**OpenAPI Code Generation** (openapi-generated.ts):
```typescript
// Generated from backend OpenAPI spec
// Command: openapi-typescript http://api.example.com/openapi.json -o src/lib/types/openapi-generated.ts

/**
 * This file was auto-generated by openapi-typescript.
 * Do not modify it manually.
 */
export interface paths {
  '/auth/login': {
    post: operations['login']
  }
  // ... more paths
}
```

**Generated Types**:
- Ensures frontend types match backend exactly
- Automatically updated when backend API changes
- Includes all endpoints, request bodies, responses

### Type Guards

**Runtime Type Checking**:
```typescript
function isUserInfo(obj: unknown): obj is UserInfo {
  return (
    typeof obj === 'object' &&
    obj !== null &&
    'id' in obj &&
    'username' in obj &&
    'email' in obj &&
    'role' in obj
  )
}

function safeParseUser(data: unknown): UserInfo | null {
  if (isUserInfo(data)) {
    return data
  }
  return null
}
```

### Discriminated Unions

**API Response Types**:
```typescript
type ApiResponse<T> =
  | { success: true; data: T }
  | { success: false; error: string }

function handleResponse<T>(response: ApiResponse<T>) {
  if (response.success) {
    console.log(response.data)  // T
  } else {
    console.log(response.error)  // string
  }
}
```

### Utility Types

**Partial**: Make all properties optional
```typescript
type UpdatePost = Partial<PostDetail>

// Equivalent to:
interface UpdatePost {
  id?: string
  slug?: string
  title?: string
  // ... all optional
}
```

**Pick**: Select specific properties
```typescript
type PostSummary = Pick<PostDetail, 'id' | 'slug' | 'title'>

// Results in:
interface PostSummary {
  id: string
  slug: string
  title: string
}
```

**Omit**: Exclude specific properties
```typescript
type CreatePostRequest = Omit<PostDetail, 'id' | 'created_at' | 'author'>

// Results in:
interface CreatePostRequest {
  slug: string
  title: string
  content: string
  // ... but no id, created_at, or author
}
```

### Usage Examples

**Typed API Calls**:
```typescript
import { LoginRequest, AuthResponse } from '@/lib/types/backend'

async function login(data: LoginRequest): Promise<AuthResponse> {
  const response = await fetch('/api/auth/login', {
    method: 'POST',
    body: JSON.stringify(data)
  })

  return response.json()
}
```

**Type-Safe Components**:
```typescript
import { PostDetail } from '@/lib/types/backend'

function PostCard({ post }: { post: PostDetail }) {
  return (
    <article>
      <h2>{post.title}</h2>
      <p>{post.excerpt}</p>
      <span>{post.category.name}</span>
    </article>
  )
}
```

**Generic Type Constraints**:
```typescript
function fetchById<T extends { id: string }>(
  collection: T[],
  id: string
): T | undefined {
  return collection.find(item => item.id === id)
}

const post = fetchById(posts, 'post-123')
const user = fetchById(users, 'user-456')
```

## Type Safety Benefits

**Compile-Time Error Detection**:
```typescript
// ERROR: Property 'password' does not exist on type 'UserInfo'
function sendEmail(user: UserInfo) {
  console.log(user.password)  // TypeScript error!
}
```

**Autocomplete**:
```typescript
const post: PostDetail = {
  id: '...',
  title: '...',  // IDE suggests all required fields
  // ...
}
```

**Refactoring Safety**:
```typescript
// Change interface definition
interface PostDetail {
  title: string
  slug: string  // Changed from 'id'
}

// TypeScript shows all places that need updating
post.id  // Error: Property 'id' does not exist
post.slug  // Correct
```

## Best Practices

1. **Use Generated Types**: Don't duplicate backend types
2. **Strict Mode**: Enable `strict: true` in tsconfig.json
3. **No `any`**: Avoid `any` type, use `unknown` instead
4. **Readonly**: Use `readonly` for immutable data
5. **Branding**: Use branded types for IDs

```typescript
// Branded types
type PostId = string & { readonly __brand: unique symbol }
type UserId = string & { readonly __brand: unique symbol }

// Prevents mixing IDs
function getPost(id: PostId) { }
function getUser(id: UserId) { }

const postId: PostId = 'post-123' as PostId
const userId: UserId = 'user-456' as UserId

getPost(postId)  // OK
getPost(userId)  // Type error!
```
