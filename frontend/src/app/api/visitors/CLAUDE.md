# Visitors API Module

## Overview

This module provides a REST API endpoint for retrieving visitor statistics data. It serves as a simple data access layer that reads visitor information from a JSON file and returns it in a structured format.

**Purpose**: Expose visitor analytics data to the frontend application
**Pattern**: Static API Route with file-based data storage
**Layer**: Layer 2 - API Route Handler

## Module Structure

```
frontend/src/app/api/visitors/
└── route.ts              # API route handler (GET /api/visitors)
```

## Architecture

### Data Flow

```
Frontend Request
    ↓
GET /api/visitors
    ↓
Read visitors.json from filesystem
    ↓
Parse JSON data
    ↓
Return NextResponse with visitor array
```

### Components

**route.ts** (25 lines)
- **Handler**: `GET()` - Retrieves all visitor data
- **Data Source**: `visitors.json` (root directory)
- **Error Handling**: Returns empty array on file read errors
- **Static Export**: `dynamic = 'force-static'` - Pre-rendered at build time

## API Specification

### GET /api/visitors

**Response** (Success - 200):
```json
{
  "visitors": [
    {
      "id": string,
      "timestamp": number,
      "path": string,
      "referrer": string | null,
      "userAgent": string,
      "metadata": Record<string, any>
    }
  ]
}
```

**Response** (Error - 200):
```json
{
  "visitors": []
}
```

**Note**: Errors are handled gracefully - file not found or parse errors return empty array instead of throwing

## Data Models

### VisitorData

```typescript
interface VisitorData {
  id: string
  timestamp: number
  path: string
  referrer: string | null
  userAgent: string
  metadata: Record<string, any>
}
```

Location: `@/lib/types/visitor`

## Error Handling

1. **File Not Found**: Returns `{ visitors: [] }` instead of 404
2. **Parse Error**: Returns `{ visitors: [] }` on invalid JSON
3. **No Exceptions**: All errors caught and converted to safe responses

## Constraints & Limitations

- **Static Only**: `force-static` means no server-side execution at runtime
- **File-based**: Data must exist at build time in `visitors.json`
- **No Real-time**: Data is static as of build time
- **Write Operations**: Not supported (GET only)
- **Size Limit**: Entire file loaded into memory (no pagination)

### Static Export Support

```typescript
export const dynamic = 'force-static'
```

This configuration:
- Pre-renders the route at build time
- Generates static JSON response
- Disables server-side execution
- Required for `output: 'export'` in Next.js config

## Usage Examples

### Frontend Integration

```typescript
// Fetch visitor data
async function getVisitorData() {
  const response = await fetch('/api/visitors')
  const data = await response.json()
  return data.visitors as VisitorData[]
}

// Usage in component
const visitors = await getVisitorData()
console.log(`Total visitors: ${visitors.length}`)
```

### Data Source

The `visitors.json` file should be generated by a build step or external service:

```json
{
  "visitors": [
    {
      "id": "uuid-1",
      "timestamp": 1735929600000,
      "path": "/",
      "referrer": "https://google.com",
      "userAgent": "Mozilla/5.0...",
      "metadata": {
        "country": "US",
        "device": "desktop"
      }
    }
  ]
}
```
